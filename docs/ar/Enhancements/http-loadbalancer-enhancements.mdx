بيانات تعريفية للإثراء لمخططات موازن حمل HTTP. راجع [امتدادات OpenAPI](../development/#openapi-extensions) لتعريفات الامتدادات.

## المخططات المُثراة

### النمط

جميع المخططات المطابقة لـ `viewshttp_loadbalancer.*SpecType` تتلقى إثراءات:

- `viewshttp_loadbalancerCreateSpecType`
- `viewshttp_loadbalancerReplaceSpecType`
- `viewshttp_loadbalancerGetSpecType`

## البيانات التعريفية للتكوين الأدنى

يوفر امتداد `x-f5xc-minimum-configuration` بيانات تعريفية شاملة لواجهة سطر الأوامر لإنشاء موازنات حمل HTTP بالحد الأدنى من التكوين القابل للتشغيل.

### الحقول المطلوبة

| الحقل | القيد | الوصف |
|-------|-------|-------|
| `metadata.name` | تنسيق تسمية DNS: `[a-z0-9]([-a-z0-9]*[a-z0-9])?` | اسم المورد |
| `metadata.namespace` | تنسيق تسمية DNS | مساحة الاسم |
| `spec.domains` | مصفوفة، الحد الأدنى للعناصر: 1 | النطاقات المراد خدمتها |

### نوع موازن الحمل (OneOf مطلوب)

يجب تحديد أحد أنواع موازن الحمل التالية:

| المتغير | الوصف |
|---------|-------|
| `spec.http` | HTTP فقط |
| `spec.https` | HTTPS مع شهادة يدوية |
| `spec.https_auto_cert` | HTTPS مع شهادة تلقائية |
| `spec.http_https` | كلاً من HTTP و HTTPS |

**ملاحظة**: تتم الإشارة إلى مجموعة OneOf لنوع موازن الحمل بالأسماء التالية:
- `lb_type` في ملفات التكوين (الشكل المختصر في `config/minimum_configs.yaml`)
- `loadbalancer_type` في الامتدادات الأصلية لمواصفات OpenAPI (اسم الحقل الأصلي)

**مرجع مخطط API**: `x-ves-oneof-field-loadbalancer_type: ["http", "https", "https_auto_cert", "http_https"]`

### الحد الأدنى من التكوين القابل للتشغيل

```json
{
  "metadata": {
    "name": "example-app",
    "namespace": "default"
  },
  "spec": {
    "domains": ["example.com"],
    "https_auto_cert": {
      "port": 443,
      "tls_config": {"default_security": {}}
    },
    "advertise_on_public_default_vip": {},
    "routes": [{"prefix": "/", "origin_pool": {"pool_name": "backend-pool"}}]
  }
}
```

## القيم الافتراضية المطبقة من الخادم

الحقول المميزة بـ `x-f5xc-server-default: true` يتم تطبيق قيمتها `default` بواسطة خادم F5 XC API عند حذفها من الطلبات. يحتوي موازن حمل HTTP على عدد كبير من القيم الافتراضية المطبقة من الخادم لأن معظم ميزات الأمان وخيارات البروتوكول تكون افتراضياً في حالة معطلة أو آمنة.

### القيم الافتراضية لـ HTTPS Auto-Cert

عند تحديد `https_auto_cert`، يطبق الخادم هذه القيم الافتراضية للحقول المحذوفة:

| `add_hsts` | `false` | منطقي | رأس HTTP Strict Transport Security |
| `tls_config.default_security` | `{}` | كائن | TLS 1.2+ مع تشفيرات قوية |
| `no_mtls` | `{}` | كائن | TLS المتبادل معطل |
| `default_header` | `{}` | كائن | معالجة رأس اسم الخادم الافتراضي |

## توصيات متغيرات OneOf (امتداد مستقبلي)

**الحالة**: لم يتم تطبيقها بعد على مخططات http_loadbalancer.

على عكس healthcheck الذي يتضمن `x-f5xc-recommended-oneof-variant` للإشارة إلى الخيار الأكثر شيوعاً لمجموعات OneOf، فإن http_loadbalancer يفتقر حالياً إلى هذا الامتداد.

### المقارنة مع Healthcheck

**Healthcheck** (مُنفَّذ):
```yaml
viewshealthcheckCreateSpecType:
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

**HTTP LoadBalancer** (غير مُنفَّذ بعد):
```yaml
viewshttp_loadbalancerCreateSpecType:
  x-f5xc-recommended-oneof-variant: null  # Would indicate recommended lb_type variant
```

### التنفيذ المستقبلي

لإضافة هذا الامتداد لـ http_loadbalancer:
1. مراقبة الاختيارات الافتراضية في وحدة تحكم F5 XC (مثلاً، أي lb_type يتم تحديده مسبقاً في واجهة المستخدم)
2. إضافة التكوين إلى `config/discovered_defaults.yaml`
3. إعادة تشغيل خط أنابيب الإثراء لتطبيق الامتداد

**الهيكل المتوقع عند الإضافة**:
```yaml
viewshttp_loadbalancerCreateSpecType:
  x-f5xc-recommended-oneof-variant:
    loadbalancer_type: "https_auto_cert"  # Example - requires verification
```

سيمكّن هذا الامتداد الأدوات اللاحقة من التحديد المسبق للمتغير الأكثر استخداماً عند عرض خيارات التكوين للمستخدمين.

## تكامل سير العمل الموجّه

يضيف خط أنابيب الإثراء بيانات تعريفية `x-f5xc-guided-workflows` على مستوى المواصفات. توفر مسارات العمل هذه إرشادات نشر خطوة بخطوة تستهلكها المساعدات الذكية وأدوات سطر الأوامر.

### نشر موازن حمل HTTP

سير عمل من 5 خطوات لإنشاء موازن حمل HTTP مكوّن بالكامل مع مجمع أصل خلفي (مُعرّف في `config/guided_workflows.yaml`):

| الخطوة | الإجراء | المورد | الحقول المطلوبة | اختياري |
|--------|---------|--------|-----------------|---------|
| 1 | إنشاء مجمع الأصل | `origin_pool` | `name`, `origin_servers`, `port` | لا |
| 2 | تكوين فحص الصحة | `healthcheck` | `name`, `http_health_check` | نعم |
| 3 | ربط فحص الصحة | - | - | نعم |
| 4 | إنشاء موازن حمل HTTP | `http_loadbalancer` | `name`, `domains`, `http.port` | لا |
| 5 | التحقق من النشر | - | - | لا |

**المتطلبات المسبقة**: مساحة اسم صالحة في المستأجر المستهدف، تطبيق خلفي يمكن الوصول إليه عبر IP أو DNS، شهادة SSL (اختيارية لـ HTTPS).

### نشر موازن حمل HTTPS مع TLS

سير عمل من 7 خطوات لإنشاء موازن حمل HTTPS مع إنهاء SSL/TLS:

| الخطوة | الإجراء | المورد | الحقول المطلوبة | اختياري |
|--------|---------|--------|-----------------|---------|
| 1 | رفع شهادة SSL | `certificate` | `name`, `certificate_chain`, `private_key` | لا |
| 2 | إنشاء مجمع الأصل | `origin_pool` | `name`, `origin_servers` | لا |
| 3 | تكوين فحص الصحة | `healthcheck` | - | نعم |
| 4 | تكوين سياسة WAF | `app_firewall` | - | نعم |
| 5 | إنشاء موازن حمل HTTPS | `http_loadbalancer` | `name`, `domains`, `https.tls_parameters` | لا |
| 6 | ربط سياسة WAF | - | - | نعم |
| 7 | التحقق من نشر HTTPS | - | - | لا |

**المتطلبات المسبقة**: مساحة اسم صالحة في المستأجر المستهدف، شهادة SSL ومفتاح خاص، تطبيق خلفي يمكن الوصول إليه.

### استهلاك سير العمل

تستخدم الأدوات اللاحقة مسارات العمل هذه لتوجيه المستخدمين خلال عمليات النشر متعددة الخطوات:

- **واجهة سطر أوامر xcsh**: تعرض مسارات العمل كأوامر موجّهة تفاعلية (مثلاً `xcsh deploy http-lb`)
- **امتداد VS Code**: يعرض خطوات سير العمل في لوحة جانبية مع تتبع التقدم
- **المساعدات الذكية**: تتبع خطوات سير العمل بالتتابع، وتنشئ كل مورد وتتحقق من التبعيات

## مرجع امتدادات OpenAPI

تُضاف هذه الامتدادات الخاصة بالمورّد إلى مخطط OpenAPI القياسي لنقل البيانات التعريفية الخاصة بـ F5 XC.

### x-f5xc-cli-domain

**النوع**: `string`

يشير إلى تصنيف النطاق لتنظيم واجهة سطر الأوامر والأدوات.

```yaml
viewshttp_loadbalancerCreateSpecType:
  type: object
  x-f5xc-cli-domain: "virtual"
```

### x-f5xc-minimum-configuration

**النوع**: `object`

يوفر بيانات تعريفية شاملة لإنشاء تكوينات بالحد الأدنى القابل للتشغيل. يتضمن:
- الوصف
- الحقول المطلوبة مع القيود
- المجموعات المتعارضة
- أمثلة التكوين (YAML، JSON)
- أمثلة أوامر curl

```yaml
viewshttp_loadbalancerCreateSpecType:
  type: object
  x-f5xc-minimum-configuration:
    description: "HTTP/HTTPS load balancer for distributing traffic across origin pools"
    required_fields:
      - "metadata.name"
      - "metadata.namespace"
      - "spec.domains"
    mutually_exclusive_groups:
      - name: "lb_type"
        fields: ["spec.http", "spec.https", "spec.https_auto_cert", "spec.http_https"]
        reason: "Choose exactly one load balancer type"
    example_yaml: |
      ...
    example_json: |
      ...
    example_curl: |
      ...
```

### x-f5xc-server-default

**النوع**: `boolean`

عندما تكون القيمة `true`، يشير إلى أن قيمة `default` المرافقة يتم فرضها بواسطة خادم F5 XC API. يمكن حذف الحقول التي تحمل هذا الامتداد بأمان من طلبات API -- حيث يطبق الخادم القيمة الافتراضية تلقائياً.

```yaml
disable_waf:
  type: object
  default: {}
  x-f5xc-server-default: true
```

### x-f5xc-conflicts-with

**النوع**: `array of strings`

يسرد أسماء الحقول المتعارضة مع الحقل الحالي، مما يشير إلى أنماط OneOf.

```yaml
advertise_custom:
  type: object
  x-f5xc-conflicts-with:
    - advertise_on_public
    - advertise_on_public_default_vip
    - do_not_advertise
```

### x-f5xc-constraints

**النوع**: `object`

يوفر قيود التحقق بما في ذلك حدود حجم المصفوفات، ونطاقات الأعداد الصحيحة، ومتطلبات التفرد، والبيانات التعريفية للاكتشاف.

```yaml
routes:
  type: array
  x-f5xc-constraints:
    constraintType: "array"
    minItems: 1
    maxItems: 256
    uniqueItems: false
    metadata:
      source: "discovery"
      confidence: 0.99
      validatedAt: "2026-01-19T12:00:00Z"
```

```yaml
port:
  type: integer
  x-f5xc-constraints:
    constraintType: "number"
    minimum: 1
    maximum: 65535
    metadata:
      source: "discovery"
      confidence: 0.99
```

### x-f5xc-description-short و x-f5xc-description-medium

**النوع**: `string`

يوفر أطوال وصف بديلة لحالات استخدام مختلفة:
- `x-f5xc-description-short`: بحد أقصى 60 حرفاً (أعمدة سطر الأوامر، الشارات)
- `x-f5xc-description-medium`: بحد أقصى 150 حرفاً (التلميحات، الملخصات)

```yaml
domains:
  type: array
  description: "List of domains the load balancer will serve"
  x-f5xc-description-short: "Domains to serve"
  x-f5xc-description-medium: "List of domains for which the load balancer accepts traffic"
```

### x-f5xc-example

**النوع**: `any` (يطابق نوع الحقل)

يوفر أمثلة ملموسة للحقول للتوثيق والأدوات.

```yaml
name:
  type: string
  x-f5xc-example: "example-app"

domains:
  type: array
  x-f5xc-example: ["example.com", "www.example.com"]
```

### x-f5xc-required-for

**النوع**: `array of strings`

يشير إلى المتطلبات الخاصة بالسياق باستخدام علامات: `minimum_config`, `create`, `update`, `read`.

```yaml
metadata.name:
  type: string
  x-f5xc-required-for: ["minimum_config", "create"]

metadata.uid:
  type: string
  x-f5xc-required-for: ["read"]
```
