بيانات وصفية للإثراء لمخططات موازن حمل HTTP. راجع [امتدادات OpenAPI](../development/#openapi-extensions) لتعريفات الامتدادات.

## المخططات المُثراة

### النمط

جميع المخططات المطابقة لـ `viewshttp_loadbalancer.*SpecType` تتلقى إثراءات:

- `viewshttp_loadbalancerCreateSpecType`
- `viewshttp_loadbalancerReplaceSpecType`
- `viewshttp_loadbalancerGetSpecType`

## بيانات وصفية للحد الأدنى من التكوين

يوفر امتداد `x-f5xc-minimum-configuration` بيانات وصفية شاملة لواجهة سطر الأوامر لإنشاء موازنات حمل HTTP بالحد الأدنى القابل للتشغيل.

### الحقول المطلوبة

| الحقل | القيد | الوصف |
|-------|-------|-------|
| `metadata.name` | تنسيق تسمية DNS: `[a-z0-9]([-a-z0-9]*[a-z0-9])?` | اسم المورد |
| `metadata.namespace` | تنسيق تسمية DNS | نطاق الأسماء |
| `spec.domains` | مصفوفة، الحد الأدنى للعناصر: 1 | النطاقات المراد خدمتها |

### نوع موازن الحمل (يجب تحديد واحد - OneOf)

يجب تحديد أحد أنواع موازن الحمل التالية:

| البديل | الوصف |
|--------|-------|
| `spec.http` | HTTP فقط |
| `spec.https` | HTTPS مع شهادة يدوية |
| `spec.https_auto_cert` | HTTPS مع شهادة تلقائية |
| `spec.http_https` | كل من HTTP و HTTPS |

**ملاحظة**: يُشار إلى مجموعة OneOf لنوع موازن الحمل بالأسماء التالية:
- `lb_type` في ملفات التكوين (الشكل المختصر في `config/minimum_configs.yaml`)
- `loadbalancer_type` في امتدادات OpenAPI الأصلية (اسم الحقل الأصلي)

**مرجع مخطط API**: `x-ves-oneof-field-loadbalancer_type: ["http", "https", "https_auto_cert", "http_https"]`

### الحد الأدنى من التكوين القابل للتشغيل

```json
{
  "metadata": {
    "name": "example-app",
    "namespace": "default"
  },
  "spec": {
    "domains": ["example.com"],
    "https_auto_cert": {
      "port": 443,
      "tls_config": {"default_security": {}}
    },
    "advertise_on_public_default_vip": {},
    "routes": [{"prefix": "/", "origin_pool": {"pool_name": "backend-pool"}}]
  }
}
```

## القيم الافتراضية المُطبقة من الخادم

الحقول المعلّمة بـ `x-f5xc-server-default: true` يتم تطبيق قيمتها `default` بواسطة خادم F5 XC API عند حذفها من الطلبات. يحتوي موازن حمل HTTP على عدد كبير من القيم الافتراضية المُطبقة من الخادم لأن معظم ميزات الأمان وخيارات البروتوكول تكون في حالة معطلة أو آمنة افتراضياً.

## توصيات بدائل OneOf (امتداد مستقبلي)

**الحالة**: لم يتم تطبيقه بعد على مخططات http_loadbalancer.

على عكس healthcheck، الذي يتضمن `x-f5xc-recommended-oneof-variant` للإشارة إلى الخيار الأكثر شيوعاً لمجموعات OneOf، فإن http_loadbalancer لا يحتوي حالياً على هذا الامتداد.

### المقارنة مع Healthcheck

**Healthcheck** (مُطبق):
```yaml
viewshealthcheckCreateSpecType:
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

**HTTP LoadBalancer** (لم يُطبق بعد):
```yaml
viewshttp_loadbalancerCreateSpecType:
  x-f5xc-recommended-oneof-variant: null  # Would indicate recommended lb_type variant
```

### التطبيق المستقبلي

لإضافة هذا الامتداد لـ http_loadbalancer:
1. مراقبة اختيارات وحدة تحكم F5 XC الافتراضية (مثلاً، أي نوع lb_type يكون محدداً مسبقاً في واجهة المستخدم)
2. إضافة التكوين إلى `config/discovered_defaults.yaml`
3. إعادة تشغيل خط أنابيب الإثراء لتطبيق الامتداد

**الهيكل المتوقع عند الإضافة**:
```yaml
viewshttp_loadbalancerCreateSpecType:
  x-f5xc-recommended-oneof-variant:
    loadbalancer_type: "https_auto_cert"  # Example - requires verification
```

سيمكن هذا الامتداد الأدوات المتلقية من تحديد البديل الأكثر استخداماً مسبقاً عند عرض خيارات التكوين للمستخدمين.

## تكامل سير العمل الموجّه

يضيف خط أنابيب الإثراء بيانات وصفية `x-f5xc-guided-workflows` على مستوى المواصفات. توفر سيرورات العمل هذه إرشادات نشر خطوة بخطوة تستهلكها مساعدات الذكاء الاصطناعي وأدوات سطر الأوامر.

### نشر موازن حمل HTTP

سير عمل من 5 خطوات لإنشاء موازن حمل HTTP مكوّن بالكامل مع مجمع أصل خلفي (معرّف في `config/guided_workflows.yaml`):

| الخطوة | الإجراء | المورد | الحقول المطلوبة | اختياري |
|--------|---------|--------|----------------|---------|
| 1 | إنشاء مجمع الأصل | `origin_pool` | `name`, `origin_servers`, `port` | لا |
| 2 | تكوين فحص الصحة | `healthcheck` | `name`, `http_health_check` | نعم |
| 3 | ربط فحص الصحة | - | - | نعم |
| 4 | إنشاء موازن حمل HTTP | `http_loadbalancer` | `name`, `domains`, `http.port` | لا |
| 5 | التحقق من النشر | - | - | لا |

**المتطلبات الأساسية**: نطاق أسماء صالح في المستأجر المستهدف، تطبيق خلفي قابل للوصول عبر IP أو DNS، شهادة SSL (اختيارية لـ HTTPS).

### نشر موازن حمل HTTPS مع TLS

سير عمل من 7 خطوات لإنشاء موازن حمل HTTPS مع إنهاء SSL/TLS:

| الخطوة | الإجراء | المورد | الحقول المطلوبة | اختياري |
|--------|---------|--------|----------------|---------|
| 1 | رفع شهادة SSL | `certificate` | `name`, `certificate_chain`, `private_key` | لا |
| 2 | إنشاء مجمع الأصل | `origin_pool` | `name`, `origin_servers` | لا |
| 3 | تكوين فحص الصحة | `healthcheck` | - | نعم |
| 4 | تكوين سياسة WAF | `app_firewall` | - | نعم |
| 5 | إنشاء موازن حمل HTTPS | `http_loadbalancer` | `name`, `domains`, `https.tls_parameters` | لا |
| 6 | ربط سياسة WAF | - | - | نعم |
| 7 | التحقق من نشر HTTPS | - | - | لا |

**المتطلبات الأساسية**: نطاق أسماء صالح في المستأجر المستهدف، شهادة SSL ومفتاح خاص، تطبيق خلفي قابل للوصول.

### استهلاك سير العمل

تستخدم الأدوات المتلقية سيرورات العمل هذه لتوجيه المستخدمين خلال عمليات النشر متعددة الخطوات:

- **واجهة سطر الأوامر xcsh**: تعرض سيرورات العمل كأوامر تفاعلية موجّهة (مثلاً `xcsh deploy http-lb`)
- **إضافة VS Code**: تعرض خطوات سير العمل في لوحة جانبية مع تتبع التقدم
- **مساعدات الذكاء الاصطناعي**: تتبع خطوات سير العمل بالتتابع، وتنشئ كل مورد وتتحقق من التبعيات

## مرجع امتدادات OpenAPI

تُضاف هذه الامتدادات المخصصة إلى مخطط OpenAPI القياسي لنقل بيانات وصفية خاصة بـ F5 XC.

### x-f5xc-cli-domain

**النوع**: `string`

يشير إلى تصنيف النطاق لتنظيم واجهة سطر الأوامر والأدوات.

```yaml
viewshttp_loadbalancerCreateSpecType:
  type: object
  x-f5xc-cli-domain: "virtual"
```

### x-f5xc-minimum-configuration

**النوع**: `object`

يوفر بيانات وصفية شاملة لإنشاء تكوينات بالحد الأدنى القابل للتشغيل. يتضمن:
- الوصف
- الحقول المطلوبة مع القيود
- المجموعات المتعارضة
- أمثلة تكوين (YAML, JSON)
- أمثلة أوامر curl

```yaml
viewshttp_loadbalancerCreateSpecType:
  type: object
  x-f5xc-minimum-configuration:
    description: "HTTP/HTTPS load balancer for distributing traffic across origin pools"
    required_fields:
      - "metadata.name"
      - "metadata.namespace"
      - "spec.domains"
    mutually_exclusive_groups:
      - name: "lb_type"
        fields: ["spec.http", "spec.https", "spec.https_auto_cert", "spec.http_https"]
        reason: "Choose exactly one load balancer type"
    example_yaml: |
      ...
    example_json: |
      ...
    example_curl: |
      ...
```

### x-f5xc-server-default

**النوع**: `boolean`

عندما تكون القيمة `true`، يشير إلى أن قيمة `default` المرافقة يتم فرضها بواسطة خادم F5 XC API. يمكن حذف الحقول التي تحتوي على هذا الامتداد من طلبات API بأمان -- حيث يطبق الخادم القيمة الافتراضية تلقائياً.

```yaml
disable_waf:
  type: object
  default: {}
  x-f5xc-server-default: true
```

### x-f5xc-conflicts-with

**النوع**: `array of strings`

يسرد أسماء الحقول المتعارضة مع الحقل الحالي، مما يشير إلى أنماط OneOf.

```yaml
advertise_custom:
  type: object
  x-f5xc-conflicts-with:
    - advertise_on_public
    - advertise_on_public_default_vip
    - do_not_advertise
```

### x-f5xc-constraints

**النوع**: `object`

يوفر قيود التحقق بما في ذلك حدود حجم المصفوفات، ونطاقات الأعداد الصحيحة، ومتطلبات التفرد، وبيانات الاكتشاف الوصفية.

```yaml
routes:
  type: array
  x-f5xc-constraints:
    constraintType: "array"
    minItems: 1
    maxItems: 256
    uniqueItems: false
    metadata:
      source: "discovery"
      confidence: 0.99
      validatedAt: "2026-01-19T12:00:00Z"
```

```yaml
port:
  type: integer
  x-f5xc-constraints:
    constraintType: "number"
    minimum: 1
    maximum: 65535
    metadata:
      source: "discovery"
      confidence: 0.99
```

### x-f5xc-description-short و x-f5xc-description-medium

**النوع**: `string`

يوفر أطوال وصف بديلة لحالات استخدام مختلفة:
- `x-f5xc-description-short`: 60 حرفاً كحد أقصى (أعمدة واجهة سطر الأوامر، الشارات)
- `x-f5xc-description-medium`: 150 حرفاً كحد أقصى (تلميحات الأدوات، الملخصات)

```yaml
domains:
  type: array
  description: "List of domains the load balancer will serve"
  x-f5xc-description-short: "Domains to serve"
  x-f5xc-description-medium: "List of domains for which the load balancer accepts traffic"
```

### x-f5xc-example

**النوع**: `any` (يطابق نوع الحقل)

يوفر أمثلة ملموسة للحقول للتوثيق والأدوات.

```yaml
name:
  type: string
  x-f5xc-example: "example-app"

domains:
  type: array
  x-f5xc-example: ["example.com", "www.example.com"]
```

### x-f5xc-required-for

**النوع**: `array of strings`

يشير إلى المتطلبات الخاصة بالسياق باستخدام علامات: `minimum_config`, `create`, `update`, `read`.

```yaml
metadata.name:
  type: string
  x-f5xc-required-for: ["minimum_config", "create"]

metadata.uid:
  type: string
  x-f5xc-required-for: ["read"]
```
