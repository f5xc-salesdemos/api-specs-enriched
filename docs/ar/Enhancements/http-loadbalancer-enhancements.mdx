بيانات وصفية للإثراء لمخططات موازن حمل HTTP. راجع [امتدادات OpenAPI](../development/#openapi-extensions) للاطلاع على تعريفات الامتدادات.

## المخططات المُثراة

### النمط

جميع المخططات المطابقة لـ `viewshttp_loadbalancer.*SpecType` تتلقى إثراءات:

- `viewshttp_loadbalancerCreateSpecType`
- `viewshttp_loadbalancerReplaceSpecType`
- `viewshttp_loadbalancerGetSpecType`

## بيانات وصفية للحد الأدنى من التكوين

يوفر امتداد `x-f5xc-minimum-configuration` بيانات وصفية شاملة لواجهة سطر الأوامر لإنشاء موازنات حمل HTTP بأقل تكوين قابل للتشغيل.

### الحقول المطلوبة

| الحقل | القيد | الوصف |
|-------|-------|-------|
| `metadata.name` | تنسيق تسمية DNS: `[a-z0-9]([-a-z0-9]*[a-z0-9])?` | اسم المورد |
| `metadata.namespace` | تنسيق تسمية DNS | مساحة الاسم |
| `spec.domains` | مصفوفة، الحد الأدنى للعناصر: 1 | النطاقات المراد خدمتها |

### نوع موازن الحمل (مطلوب واحد من عدة خيارات)

يجب تحديد أحد أنواع موازن الحمل التالية:

| الخيار | الوصف |
|--------|-------|
| `spec.http` | HTTP فقط |
| `spec.https` | HTTPS مع شهادة يدوية |
| `spec.https_auto_cert` | HTTPS مع شهادة تلقائية |
| `spec.http_https` | HTTP و HTTPS معًا |

**ملاحظة**: يُشار إلى مجموعة OneOf لنوع موازن الحمل بالأسماء التالية:
- `lb_type` في ملفات التكوين (الصيغة المختصرة في `config/minimum_configs.yaml`)
- `loadbalancer_type` في الامتدادات الأصلية لمواصفات OpenAPI (اسم الحقل الأصلي)

**مرجع مخطط API**: `x-ves-oneof-field-loadbalancer_type: ["http", "https", "https_auto_cert", "http_https"]`

### الحد الأدنى من التكوين القابل للتشغيل

```json
{
  "metadata": {
    "name": "example-app",
    "namespace": "default"
  },
  "spec": {
    "domains": ["example.com"],
    "https_auto_cert": {
      "port": 443,
      "tls_config": {"default_security": {}}
    },
    "advertise_on_public_default_vip": {},
    "routes": [{"prefix": "/", "origin_pool": {"pool_name": "backend-pool"}}]
  }
}
```

## القيم الافتراضية المُطبقة من الخادم

الحقول المعلّمة بـ `x-f5xc-server-default: true` يتم تطبيق قيمتها الافتراضية (`default`) بواسطة خادم F5 XC API عند حذفها من الطلبات. يحتوي موازن حمل HTTP على عدد كبير من القيم الافتراضية المُطبقة من الخادم لأن معظم ميزات الأمان وخيارات البروتوكول تكون معطلة أو في حالة آمنة افتراضيًا.

### القيم الافتراضية لشهادة HTTPS التلقائية

عند تحديد `https_auto_cert`، يُطبق الخادم هذه القيم الافتراضية للحقول المحذوفة:

| الحقل | القيمة الافتراضية | النوع | الوصف |
|-------|-------------------|-------|-------|
| `port` | `443` | عدد صحيح | منفذ استماع HTTPS |
| `http_redirect` | `false` | قيمة منطقية | إعادة توجيه HTTP إلى HTTPS |
| `add_hsts` | `false` | قيمة منطقية | ترويسة أمان النقل الصارم HTTP |
| `tls_config.default_security` | `{}` | كائن | TLS 1.2+ مع تشفيرات قوية |
| `no_mtls` | `{}` | كائن | TLS المتبادل معطل |
| `default_header` | `{}` | كائن | معالجة ترويسة اسم الخادم الافتراضية |
| `enable_path_normalize` | `{}` | كائن | تطبيع المسار مُفعّل |
| `default_loadbalancer` | `{}` | كائن | إعدادات موازن الحمل الافتراضية |
| `header_transformation_type.legacy_header_transformation` | `{}` | كائن | تحويل الترويسات القديم |
| `connection_idle_timeout` | `120000` | عدد صحيح | مهلة خمول الاتصال بالميلي ثانية (دقيقتان) |

## توصيات خيارات OneOf (امتداد مستقبلي)

**الحالة**: لم يُطبق بعد على مخططات http_loadbalancer.

على عكس فحص الصحة (healthcheck) الذي يتضمن `x-f5xc-recommended-oneof-variant` للإشارة إلى الخيار الأكثر شيوعًا لمجموعات OneOf، فإن http_loadbalancer لا يحتوي حاليًا على هذا الامتداد.

### المقارنة مع فحص الصحة

**فحص الصحة** (مُطبق):
```yaml
viewshealthcheckCreateSpecType:
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

**موازن حمل HTTP** (لم يُطبق بعد):
```yaml
viewshttp_loadbalancerCreateSpecType:
  x-f5xc-recommended-oneof-variant: null  # Would indicate recommended lb_type variant
```

### التطبيق المستقبلي

لإضافة هذا الامتداد لـ http_loadbalancer:
1. مراقبة التحديدات الافتراضية في وحدة تحكم F5 XC (مثلاً، أي نوع lb_type يتم تحديده مسبقًا في واجهة المستخدم)
2. إضافة التكوين إلى `config/discovered_defaults.yaml`
3. إعادة تشغيل خط أنابيب الإثراء لتطبيق الامتداد

**البنية المتوقعة عند الإضافة**:
```yaml
viewshttp_loadbalancerCreateSpecType:
  x-f5xc-recommended-oneof-variant:
    loadbalancer_type: "https_auto_cert"  # Example - requires verification
```

سيمكّن هذا الامتداد الأدوات التابعة من التحديد المسبق للخيار الأكثر استخدامًا عند تقديم خيارات التكوين للمستخدمين.

## تكامل سير العمل الموجّه

يضيف خط أنابيب الإثراء بيانات وصفية `x-f5xc-guided-workflows` على مستوى المواصفة. توفر مسارات العمل هذه إرشادات نشر خطوة بخطوة تستهلكها مساعدات الذكاء الاصطناعي وأدوات سطر الأوامر.

### نشر موازن حمل HTTP

سير عمل من 5 خطوات لإنشاء موازن حمل HTTP مُكوّن بالكامل مع مجمع أصل خلفي (مُعرّف في `config/guided_workflows.yaml`):

| الخطوة | الإجراء | المورد | الحقول المطلوبة | اختياري |
|--------|---------|--------|----------------|---------|
| 1 | إنشاء مجمع الأصل | `origin_pool` | `name`, `origin_servers`, `port` | لا |
| 2 | تكوين فحص الصحة | `healthcheck` | `name`, `http_health_check` | نعم |
| 3 | ربط فحص الصحة | - | - | نعم |
| 4 | إنشاء موازن حمل HTTP | `http_loadbalancer` | `name`, `domains`, `http.port` | لا |
| 5 | التحقق من النشر | - | - | لا |

**المتطلبات المسبقة**: مساحة اسم صالحة في المستأجر الهدف، تطبيق خلفي يمكن الوصول إليه عبر IP أو DNS، شهادة SSL (اختيارية لـ HTTPS).

### نشر موازن حمل HTTPS مع TLS

سير عمل من 7 خطوات لإنشاء موازن حمل HTTPS مع إنهاء SSL/TLS:

| الخطوة | الإجراء | المورد | الحقول المطلوبة | اختياري |
|--------|---------|--------|----------------|---------|
| 1 | رفع شهادة SSL | `certificate` | `name`, `certificate_chain`, `private_key` | لا |
| 2 | إنشاء مجمع الأصل | `origin_pool` | `name`, `origin_servers` | لا |
| 3 | تكوين فحص الصحة | `healthcheck` | - | نعم |
| 4 | تكوين سياسة WAF | `app_firewall` | - | نعم |
| 5 | إنشاء موازن حمل HTTPS | `http_loadbalancer` | `name`, `domains`, `https.tls_parameters` | لا |
| 6 | ربط سياسة WAF | - | - | نعم |
| 7 | التحقق من نشر HTTPS | - | - | لا |

**المتطلبات المسبقة**: مساحة اسم صالحة في المستأجر الهدف، شهادة SSL ومفتاح خاص، تطبيق خلفي يمكن الوصول إليه.

### استهلاك سير العمل

تستخدم الأدوات التابعة مسارات العمل هذه لتوجيه المستخدمين عبر عمليات النشر متعددة الخطوات:

- **واجهة سطر الأوامر xcsh**: تعرض مسارات العمل كأوامر تفاعلية موجّهة (مثلاً، `xcsh deploy http-lb`)
- **امتداد VS Code**: يعرض خطوات سير العمل في لوحة جانبية مع تتبع التقدم
- **مساعدات الذكاء الاصطناعي**: تتبع خطوات سير العمل بالتسلسل، وتنشئ كل مورد وتتحقق من التبعيات

## مرجع امتدادات OpenAPI

تُضاف امتدادات المورّد هذه إلى مخطط OpenAPI القياسي لنقل البيانات الوصفية الخاصة بـ F5 XC.

### x-f5xc-cli-domain

**النوع**: `string`

يشير إلى تصنيف النطاق لتنظيم واجهة سطر الأوامر والأدوات.

```yaml
viewshttp_loadbalancerCreateSpecType:
  type: object
  x-f5xc-cli-domain: "virtual"
```

### x-f5xc-minimum-configuration

**النوع**: `object`

يوفر بيانات وصفية شاملة لإنشاء الحد الأدنى من التكوينات القابلة للتشغيل. يتضمن:
- الوصف
- الحقول المطلوبة مع القيود
- المجموعات المتنافية
- أمثلة تكوين (YAML، JSON)
- أمثلة أوامر curl

```yaml
viewshttp_loadbalancerCreateSpecType:
  type: object
  x-f5xc-minimum-configuration:
    description: "HTTP/HTTPS load balancer for distributing traffic across origin pools"
    required_fields:
      - "metadata.name"
      - "metadata.namespace"
      - "spec.domains"
    mutually_exclusive_groups:
      - name: "lb_type"
        fields: ["spec.http", "spec.https", "spec.https_auto_cert", "spec.http_https"]
        reason: "Choose exactly one load balancer type"
    example_yaml: |
      ...
    example_json: |
      ...
    example_curl: |
      ...
```

### x-f5xc-server-default

**النوع**: `boolean`

عندما تكون القيمة `true`، يشير إلى أن القيمة الافتراضية (`default`) المصاحبة يتم فرضها بواسطة خادم F5 XC API. يمكن حذف الحقول التي تحمل هذا الامتداد بأمان من طلبات API -- حيث يطبق الخادم القيمة الافتراضية تلقائيًا.

```yaml
disable_waf:
  type: object
  default: {}
  x-f5xc-server-default: true
```

### x-f5xc-conflicts-with

**النوع**: `array of strings`

يسرد أسماء الحقول المتنافية مع الحقل الحالي، مما يشير إلى أنماط OneOf.

```yaml
advertise_custom:
  type: object
  x-f5xc-conflicts-with:
    - advertise_on_public
    - advertise_on_public_default_vip
    - do_not_advertise
```

### x-f5xc-constraints

**النوع**: `object`

يوفر قيود التحقق بما في ذلك حدود حجم المصفوفات، نطاقات الأعداد الصحيحة، متطلبات التفرد، وبيانات الاكتشاف الوصفية.

```yaml
routes:
  type: array
  x-f5xc-constraints:
    constraintType: "array"
    minItems: 1
    maxItems: 256
    uniqueItems: false
    metadata:
      source: "discovery"
      confidence: 0.99
      validatedAt: "2026-01-19T12:00:00Z"
```

```yaml
port:
  type: integer
  x-f5xc-constraints:
    constraintType: "number"
    minimum: 1
    maximum: 65535
    metadata:
      source: "discovery"
      confidence: 0.99
```

### x-f5xc-description-short و x-f5xc-description-medium

**النوع**: `string`

يوفر أطوال وصف بديلة لحالات الاستخدام المختلفة:
- `x-f5xc-description-short`: 60 حرفًا كحد أقصى (أعمدة واجهة سطر الأوامر، الشارات)
- `x-f5xc-description-medium`: 150 حرفًا كحد أقصى (التلميحات، الملخصات)

```yaml
domains:
  type: array
  description: "List of domains the load balancer will serve"
  x-f5xc-description-short: "Domains to serve"
  x-f5xc-description-medium: "List of domains for which the load balancer accepts traffic"
```

### x-f5xc-example

**النوع**: `any` (يطابق نوع الحقل)

يوفر أمثلة ملموسة للحقول للتوثيق والأدوات.

```yaml
name:
  type: string
  x-f5xc-example: "example-app"

domains:
  type: array
  x-f5xc-example: ["example.com", "www.example.com"]
```

### x-f5xc-required-for

**النوع**: `array of strings`

يشير إلى المتطلبات الخاصة بالسياق باستخدام العلامات: `minimum_config`, `create`, `update`, `read`.

```yaml
metadata.name:
  type: string
  x-f5xc-required-for: ["minimum_config", "create"]

metadata.uid:
  type: string
  x-f5xc-required-for: ["read"]
```

## الوصول إلى البيانات

### مواصفات OpenAPI

| الملف | المحتوى |
|-------|---------|
| `docs/specifications/api/virtual.json` | جميع مخططات http_loadbalancer مع الإثراءات |
| `docs/specifications/api/openapi.json` | المواصفة المدمجة مع جميع المخططات |

### ملفات التكوين

| الملف | الغرض |
|-------|-------|
| `config/minimum_configs.yaml` | مصدر البيانات الوصفية للحد الأدنى من التكوين |
| `config/constraint_patterns.yaml` | تعريفات أنماط القيود |
| `config/guided_workflows.yaml` | تعريفات خطوات سير العمل الموجّه |
| `config/domain_descriptions.yaml` | البيانات الوصفية لوصف النطاق |

### المخططات المُثراة

- `viewshttp_loadbalancerCreateSpecType` - مخطط عملية الإنشاء
- `viewshttp_loadbalancerReplaceSpecType` - مخطط عملية التحديث
- `viewshttp_loadbalancerGetSpecType` - مخطط عملية القراءة
