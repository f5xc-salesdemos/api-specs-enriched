بيانات وصفية إثرائية للمخططات المتعلقة بـ origin_pool. انظر [امتدادات OpenAPI](../development/#openapi-extensions) لتعريفات الامتدادات.

## المخططات المُثراة

### النمط

جميع المخططات المطابقة لـ `origin_pool.*SpecType` تتلقى إثراءات:

- `origin_poolCreateSpecType`
- `origin_poolReplaceSpecType`
- `origin_poolGetSpecType`

## الحقول المطلوبة

| الحقل | مطلوب لـ | ملاحظات |
|-------|----------|---------|
| `metadata.name` | جميع العمليات | 1-63 حرف، أحرف صغيرة أبجدية رقمية، النمط: `^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |
| `metadata.namespace` | جميع العمليات | معرّف مساحة الاسم |
| `spec.origin_servers` | الإنشاء | عنصر واحد كحد أدنى |
| `spec.port` | الإنشاء | 1-65535، بدون قيمة افتراضية من الخادم |

## الحد الأدنى للتكوين القابل للتشغيل

```json
{
  "metadata": {
    "name": "my-origin-pool",
    "namespace": "default"
  },
  "spec": {
    "origin_servers": [
      {
        "public_name": {
          "dns_name": "backend.example.com"
        }
      }
    ],
    "port": 443
  }
}
```

## القيم الافتراضية المُطبَّقة من الخادم

الحقول المُعلَّمة بـ `x-f5xc-server-default: true` يتم تطبيق قيمتها الافتراضية (`default`) بواسطة خادم F5 XC API عند حذفها من الطلبات.

### حقول المستوى الأعلى

| الحقل | القيمة الافتراضية | النوع | الوصف |
|-------|-------------------|-------|-------|
| `no_tls` | `{}` | كائن | تعطيل TLS إلى الأصل |
| `healthcheck` | `[]` | مصفوفة | لا توجد فحوصات صحية مُكوَّنة |
| `loadbalancer_algorithm` | `ROUND_ROBIN` | تعداد | موازنة حمل بنظام الدوران |
| `endpoint_selection` | `DISTRIBUTED` | تعداد | استخدام جميع نقاط النهاية (محلية + بعيدة) |

### القيم الافتراضية لـ advanced_options

عندما لا يتم تحديد `advanced_options`، يتصرف الخادم كما لو أن هذه القيم كانت مُعيَّنة:

| الحقل | القيمة الافتراضية | النوع | الوصف |
|-------|-------------------|-------|-------|
| `connection_timeout` | `2000` | عدد صحيح | مهلة الاتصال بالمللي ثانية |
| `http_idle_timeout` | `300000` | عدد صحيح | مهلة خمول HTTP بالمللي ثانية (5 دقائق) |
| `same_as_endpoint_port` | `{}` | كائن | فحص الصحة يستخدم منفذ نقطة النهاية |
| `default_circuit_breaker` | `{}` | كائن | إعدادات قاطع الدائرة الافتراضية |
| `disable_outlier_detection` | `{}` | كائن | كشف القيم الشاذة معطّل |
| `no_panic_threshold` | `{}` | كائن | بدون عتبة ذعر |
| `disable_subsets` | `{}` | كائن | موازنة حمل المجموعات الفرعية معطّلة |
| `auto_http_config` | `{}` | كائن | تفاوض تلقائي لبروتوكول HTTP |
| `disable_proxy_protocol` | `{}` | كائن | بروتوكول الوكيل معطّل |
| `disable_lb_source_ip_persistance` | `{}` | كائن | ثبات IP المصدر لموازن الحمل معطّل |

### القيم الافتراضية للكائنات المُتداخلة

| المسار | القيمة الافتراضية | الوصف |
|--------|-------------------|-------|
| `origin_servers[].labels` | `{}` | كائن تسميات فارغ |
| `origin_servers[].public_name.refresh_interval` | `0` | استخدام الفترة الافتراضية للنظام لتحديث DNS |

## التباينات بين واجهة المستخدم والقيم الافتراضية للخادم

تقوم واجهة F5 XC الإلكترونية بتحديد قيم مسبقة مختلفة عما يُطبِّقه الخادم (API) عند حذف الحقول.

| الحقل | القيمة المُحدَّدة مسبقاً في الواجهة | القيمة المُطبَّقة من الخادم | ملاحظة |
|-------|--------------------------------------|----------------------------|--------|
| `loadbalancer_algorithm` | `LB_OVERRIDE` | `ROUND_ROBIN` | الموارد المُنشأة عبر الواجهة والمُنشأة عبر API تختلف عند حذف الحقل |

## قيم التعداد

### loadbalancer_algorithm

| القيمة | الوصف | ملاحظات |
|--------|-------|---------|
| `ROUND_ROBIN` | يتم اختيار كل نقطة نهاية سليمة بنظام الدوران | القيمة الافتراضية للخادم |
| `LEAST_REQUEST` | يتم اختيار نقطة النهاية ذات أقل عدد من الطلبات النشطة | |
| `RING_HASH` | تجزئة متسقة باستخدام تجزئة حلقية لأسماء نقاط النهاية | |
| `RANDOM` | اختيار عشوائي لنقطة نهاية سليمة | |
| `LB_OVERRIDE` | سياسة التجزئة موروثة من موازن الحمل الأصلي | القيمة الافتراضية للواجهة |

### endpoint_selection

| القيمة | الوصف | ملاحظات |
|--------|-------|---------|
| `DISTRIBUTED` | مراعاة نقاط النهاية البعيدة والمحلية معاً | القيمة الافتراضية للخادم |
| `LOCAL_ONLY` | استخدام نقاط النهاية المحلية فقط | |
| `LOCAL_PREFERRED` | تفضيل المحلية، مع الرجوع إلى البعيدة عند عدم التوفر | |

## مجموعات حقول OneOf

مجموعات حقول متبادلة الاستبعاد. يمكن تحديد حقل واحد فقط من كل مجموعة:

| المجموعة | الحقول | القيمة الافتراضية للخادم |
|----------|--------|--------------------------|
| تكوين المنفذ | `port`, `automatic_port`, `lb_port` | `port` (صريح) |
| TLS إلى الأصل | `no_tls`, `use_tls` | `no_tls` |
| منفذ فحص الصحة | `same_as_endpoint_port`, `health_check_port` | `same_as_endpoint_port` |
| قاطع الدائرة | `default_circuit_breaker`, `disable_circuit_breaker`, `circuit_breaker` | `default_circuit_breaker` |
| كشف القيم الشاذة | `disable_outlier_detection`, `outlier_detection` | `disable_outlier_detection` |
| عتبة الذعر | `no_panic_threshold`, `panic_threshold` | `no_panic_threshold` |
| موازنة المجموعات الفرعية | `disable_subsets`, `enable_subsets` | `disable_subsets` |
| بروتوكول HTTP | `auto_http_config`, `http1_config`, `http2_options` | `auto_http_config` |
| بروتوكول الوكيل | `disable_proxy_protocol`, `proxy_protocol_v1`, `proxy_protocol_v2` | `disable_proxy_protocol` |
| IP المصدر لموازن الحمل | `disable_lb_source_ip_persistance`, `enable_lb_source_ip_persistance` | `disable_lb_source_ip_persistance` |
| مجمع الاتصالات | `enable_conn_pool_reuse`, `disable_conn_pool_reuse` | `enable_conn_pool_reuse` |

## قيود الحقول

| الحقل | النوع | القيد |
|-------|-------|-------|
| `spec.port` | عدد صحيح | 1-65535 |
| `spec.advanced_options.connection_timeout` | عدد صحيح | 0-1,800,000 مللي ثانية |
| `spec.advanced_options.http_idle_timeout` | عدد صحيح | 0-600,000 مللي ثانية |
| `spec.advanced_options.panic_threshold` | عدد صحيح | 0-100 (نسبة مئوية) |
| `metadata.name` | سلسلة نصية | 1-63 حرف، النمط: `^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |

## أنواع خوادم الأصل

يمكن تحديد خوادم الأصل باستخدام متغيرات أنواع مختلفة. يحدد النوع الحقول الإضافية المطلوبة.

### الأنواع المُنفَّذة

| النوع | الوصف | الحقول المطلوبة | الحالة |
|-------|-------|-----------------|--------|
| `public_name` | خادم أصل باسم DNS عام | `dns_name` | ✅ مكتمل |
| `public_ip` | خادم أصل بعنوان IP عام | `ip` (IPv4) | ✅ مكتمل |

### مثال: public_name

```json
{
  "origin_servers": [
    {
      "public_name": {
        "dns_name": "backend.example.com"
      }
    }
  ]
}
```

### مثال: public_ip

```json
{
  "origin_servers": [
    {
      "public_ip": {
        "ip": "8.8.8.8"
      }
    }
  ]
}
```

## مهام مُعلَّقة: أنواع خوادم الأصل المعتمدة على الموقع

تتطلب أنواع خوادم الأصل التالية تطوير منطق مورد "الموقع" قبل التنفيذ:

| النوع | الوصف | الحقول المطلوبة | الحالة |
|-------|-------|-----------------|--------|
| `private_ip` | خادم أصل بعنوان IP خاص/عام ومعلومات الموقع | `ip`, `site_locator`, `network_choice`, `snat_pool` | 🔲 بانتظار منطق الموقع |
| `private_name` | خادم أصل باسم DNS ومعلومات الموقع | `dns_name`, `site_locator`, `network_choice`, `snat_pool` | 🔲 بانتظار منطق الموقع |
| `k8s_service` | خدمة Kubernetes مع معلومات الموقع | `service_name`, `site_locator`, `network_choice` | 🔲 بانتظار منطق الموقع |
| `consul_service` | خدمة HashiCorp Consul مع معلومات الموقع | `service_name`, `site_locator`, `network_choice` | 🔲 بانتظار منطق الموقع |

### متطلبات التنفيذ

قبل أن يمكن توثيق هذه الأنواع بالكامل:

1. **اكتشاف مورد الموقع** - توثيق مخطط مورد `site` وأنماط site_locator
2. **تعداد اختيار الشبكة** - توثيق القيم المُقيَّدة لـ `network_choice` (site_local_inside, site_local_outside, إلخ.)
3. **خيارات مجمع SNAT** - توثيق أنماط تكوين مجمع SNAT
4. **المراجع المتقاطعة بين الموارد** - تحديد كيفية إشارة origin_pool إلى موارد الموقع

### التتبع

- عنصر نائب للتكوين: `config/discovered_defaults.yaml` → `origin_pool.origin_server_types.enums`
- السبرنت المرتبط: إثراءات مخطط مورد الموقع
