تخفيف تأثير المستخدمين الضارين مع تصنيف مستوى التهديد. إجراءات استجابة آلية لأنماط السلوك المشبوه.

- **الفئة**: الأمان
- **التعقيد**: بسيط
- **المسارات**: 4 | **المخططات**: 32
- **المستوى**: متقدم
- **المجالات ذات الصلة**: `bot_defense`, `virtual`, `network_security`

## حالات الاستخدام

- تكوين سياسات تخفيف التهديدات الآلية
- تحديد قواعد اكتشاف المستخدمين الضارين
- إدارة سير عمل الاستجابة للحوادث
- تطبيق الحظر أو تحديد معدل الاستخدام على التهديدات

## الموارد الأساسية

- **mitigation_policy**: سياسة التخفيف للاستجابة للحوادث
- **malicious_user_rule**: قاعدة المستخدم الضار لتخفيف التهديدات

## نقاط النهاية

| الطريقة | المسار | الوصف |
|--------|------|-------------|
| POST | `/api/config/namespaces/&#123;metadata.namespace&#125;/malicious_user_mitigations` | إنشاء تخفيف للمستخدم الضار. |
| PUT | `/api/config/namespaces/&#123;metadata.namespace&#125;/malicious_user_mitigations/&#123;metadata.name&#125;` | استبدال تخفيف للمستخدم الضار. |
| GET | `/api/config/namespaces/&#123;namespace&#125;/malicious_user_mitigations` | سرد تخفيفات المستخدم الضار. |
| GET | `/api/config/namespaces/&#123;namespace&#125;/malicious_user_mitigations/&#123;name&#125;` | الحصول على تخفيف للمستخدم الضار. |
| DELETE | `/api/config/namespaces/&#123;namespace&#125;/malicious_user_mitigations/&#123;name&#125;` | حذف تخفيف للمستخدم الضار. |
