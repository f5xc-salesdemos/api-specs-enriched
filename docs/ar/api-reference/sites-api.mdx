تكامل AWS وAzure وGCP VPC مع بوابات العبور. اختيار قائم على التسميات لتطبيق السياسات عبر المناطق.

- **الفئة**: البنية التحتية
- **التعقيد**: متقدم
- **المسارات**: 133 | **المخططات**: 1015
- **المستوى**: قياسي
- **النطاقات ذات الصلة**: `cloud_infrastructure`، `customer_edge`، `managed_kubernetes`

## حالات الاستخدام

- نشر F5 XC عبر مزودي الخدمات السحابية (AWS، Azure، GCP)
- إدارة عمليات نشر مواقع XCKS (Kubernetes المُدارة) (المعروفة سابقاً بـ AppStack)
- نشر مواقع Secure Mesh لعمليات نشر الحافة المركزة على الشبكات
- دمج مجموعات Kubernetes الخارجية كحافة العميل
- تكوين مواقع AWS VPC وAzure VNet وGCP VPC
- إدارة المواقع الافتراضية وسياسات المواقع

## الموارد الأساسية

- **site**: موقع نشر فعلي أو سحابي لخدمات الحافة
- **virtual_site**: تجميع منطقي للمواقع باستخدام محددات التسميات
- **site_mesh_group**: تكوين اتصال الشبكة بين مواقع متعددة
