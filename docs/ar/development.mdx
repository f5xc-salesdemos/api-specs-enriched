يصف هذا الدليل سير عمل التطوير لخط أنابيب إثراء API الخاص بـ F5 XC.

## البدء السريع

### الإعداد للمرة الأولى

```bash
# Clone the repository
git clone https://github.com/f5xc-salesdemos/api-specs-enriched.git
cd api-specs-enriched

# Create virtual environment and install dependencies
make install

# Install pre-commit hooks
make pre-commit-install

# Download specs and run pipeline
make build

# Preview documentation locally
make serve
```

### التطوير اليومي

```bash
# Quick rebuild (uses cached specs)
make rebuild

# Run all quality checks
make pre-commit-run

# Preview documentation
make serve
```

## أنماط سير العمل

### 1. سير عمل الاكتشاف

**المتطلبات الأساسية**:
- اتصال VPN ببيئة F5 XC
- بيانات اعتماد API صالحة

```bash
# Set credentials
export F5XC_API_URL="https://your-tenant.console.ves.volterra.io/api"
export F5XC_API_TOKEN="your-api-token"

# Run discovery
make discover

# View results
jq '.statistics' specs/discovered/session.json

# Commit for CI/CD
make push-discovery
```

**يلتقط الاكتشاف**:
- الحقول المطلوبة/الاختيارية الفعلية
- قيم Enum من الاستجابات الحية
- القيم الافتراضية
- عمليات التحقق من النمط
- أمثلة الاستجابة

### 2. سير عمل الإصدار

الإصدارات مؤتمتة:

1. الجدول الزمني اليومي (الساعة 6 صباحاً UTC) أو الدفع إلى الفرع الرئيسي يُشغّل سير العمل
2. يحدد فحص ETag ما إذا كانت مواصفات F5 قد تغيرت
3. يعالج خط الأنابيب المواصفات ويثريها
4. يُحسب الإصدار من علامات git: `git describe --tags --abbrev=0`
5. يُنشأ إيداع مباشر + علامة (بدون طلب سحب لرفع الإصدار)
6. يُنشأ إصدار GitHub مع سجل التغييرات
7. يُنشر التوثيق على GitHub Pages

**قواعد رفع رقم الإصدار**:

| الشرط | نوع الرفع | المثال |
|-----------|-----------|---------|
| `[major]` في الإيداع | رئيسي | 1.0.0 → 2.0.0 |
| `BREAKING CHANGE` في الإيداع | رئيسي | 1.0.0 → 2.0.0 |
| إضافة مواصفة نطاق جديدة | ثانوي | 1.0.0 → 1.1.0 |
| أي تغيير آخر | تصحيح | 1.0.0 → 1.0.1 |

### 3. سير عمل التطوير

```bash
# Create feature branch
git checkout -b feature/my-change

# Make changes to config or scripts

# Test locally
make pipeline
make lint

# Commit (pre-commit hooks run automatically)
git add .
git commit -m "feat: add new enrichment rule"

# Push and create PR
git push -u origin feature/my-change
gh pr create
```

## التعمق في الاكتشاف

### ما هو الاكتشاف؟

يستكشف الاكتشاف واجهة برمجة التطبيقات F5 XC الحية للعثور على:

- **القيود غير الموثقة**: الحقول المطلوبة غير المُشار إليها في OpenAPI
- **قيم Enum الفعلية**: القيم الحقيقية المرصودة في الإنتاج
- **السلوكيات الافتراضية**: القيم التي يطبقها الخادم عند حذف الحقول
- **أنماط الاستجابة**: أشكال البيانات الفعلية

### إعداد الاكتشاف

```yaml
# config/discovery.yaml
discovery:
  exploration:
    namespaces:
      - "system"
      - "shared"
    methods:
      - "GET"
      - "OPTIONS"
    max_endpoints_per_run: 500

  schema_inference:
    sample_size: 3
    detect_patterns: true
    detect_constraints: true
```

### امتدادات الاكتشاف

يضيف الاكتشاف امتدادات `x-discovered-*` إلى المواصفات:

```json
{
  "properties": {
    "name": {
      "type": "string",
      "x-discovered-required": true,
      "x-discovered-pattern": "^[a-z][a-z0-9-]*$",
      "x-discovered-examples": ["my-app", "prod-lb"]
    }
  }
}
```

### تقرير تحليل القيود

```bash
make constraint-report
# Output: reports/constraint-analysis.md
```

## امتدادات OpenAPI

يستخدم خط أنابيب الإثراء امتدادات الموردين لتضمين بيانات التحقق والقيم الافتراضية.

### فئات الامتدادات

| الامتداد | المعنى | الدلالة |
|-----------|---------|-------------|
| `x-f5xc-server-default` | يطبق خادم F5 XC API هذه القيمة عند حذف الحقل | الحقل اختياري؛ حذفه يُنتج السلوك الافتراضي الموثق |
| `x-f5xc-recommended-value` | تُعبّئ وحدة تحكم الويب F5 XC هذه القيمة مسبقاً للموارد الجديدة | لا يوجد افتراضي للخادم لهذا الحقل لكن هذه القيمة تمثل الإعداد النموذجي |
| `x-f5xc-recommended-oneof-variant` | تختار وحدة تحكم F5 XC هذا الخيار المسبق من OneOf | يحدد الخيار النموذجي عند وجود خيارات متنافية متعددة |
| `x-f5xc-conflicts-with` | يسرد الخصائص الأخرى التي لا يمكن استخدامها مع هذا الحقل | الخاصية جزء من مجموعة OneOf؛ يمكن تحديد خاصية واحدة فقط من الخصائص المتعارضة |

### امتدادات الحقول المطلوبة

| الامتداد | المصدر | الغرض |
|-----------|--------|---------|
| `x-ves-required: "true"` | المواصفة الأصلية لـ F5 XC | الحقل يتطلب قيمة غير صفرية/غير فارغة |
| `x-f5xc-required-for` | خط أنابيب الإثراء | الحالة المطلوبة الخاصة بالسياق |

**سياقات x-f5xc-required-for**:
- `create`: مطلوب عند إنشاء المورد
- `update`: مطلوب عند تحديث المورد
- `minimum_config`: مطلوب للحد الأدنى من الإعداد القابل للتطبيق

### امتدادات القيمة الافتراضية وOneOf

| الامتداد | الغرض |
|-----------|---------|
| `x-f5xc-server-default: true` | يُحدد الإعدادات الافتراضية المُطبَّقة من الخادم |
| `x-f5xc-recommended-value` | القيمة المُعبَّأة مسبقاً في وحدة تحكم F5 XC |
| `x-f5xc-recommended-oneof-variant` | خيار OneOf الموصى به |
| `x-f5xc-conflicts-with` | التنافي المتبادل مع خصائص OneOf الأخرى |

#### x-f5xc-server-default

**النوع**: `boolean`

عندما تكون `true`، يشير ذلك إلى أن القيمة `default` المرافقة يُطبّقها خادم F5 XC API. يمكن حذف الحقول التي تحمل هذا الامتداد بأمان من طلبات API — يطبق الخادم الافتراضي تلقائياً.

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

#### x-f5xc-recommended-value

**النوع**: `any` (يتطابق مع نوع الحقل)

يحدد قيمة تستخدمها وحدة تحكم الويب F5 XC كافتراضية مُعبَّأة مسبقاً. هذه القيمة غير مُفرَضة من الخادم لكنها تمثل الإعداد الابتدائي النموذجي للموارد الجديدة المُنشأة عبر وحدة التحكم.

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

#### x-f5xc-recommended-oneof-variant

**النوع**: `object` (خريطة من اسم المجموعة إلى اسم المتغير)

بالنسبة للمخططات التي تحتوي على مجموعات حقول متنافية، يحدد أي متغير هو الخيار الافتراضي أو الأكثر شيوعاً. المفتاح هو اسم مجموعة OneOf والقيمة هي اسم حقل المتغير الموصى به.

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

#### x-f5xc-conflicts-with

**النوع**: `array` من السلاسل النصية

**أُضيف في**: v3.2.0 (العدد #494)

يُعلن عن علاقات التنافي المتبادل بين أعضاء مجموعة OneOf. يُشتق تلقائياً من امتدادات `x-ves-oneof-field-*`. يُمكّن هذا الأدوات الأدنى في السلسلة (Terraform وCLI وMCP) من التحقق من التعارضات على مستوى المخطط بدلاً من وقت التشغيل.

```yaml
host_header:
  type: string
  x-f5xc-conflicts-with: ["use_origin_server_name"]

use_origin_server_name:
  type: object
  x-f5xc-conflicts-with: ["host_header"]
```

**حالات الاستخدام**:
- يمكن لموفري Terraform توليد قواعد التحقق
- يمكن لأدوات CLI التحذير من تركيبات الحقول المتعارضة
- يمكن لمساعدي الذكاء الاصطناعي توليد إعدادات OneOf الصحيحة
- يمكن لإضافات IDE توفير إكمال تلقائي يراعي التعارضات

**المصدر**: يُشتق تلقائياً من امتدادات F5 الأصلية `x-ves-oneof-field-*` أثناء إثراء خط الأنابيب.

### دلالات قواعد التحقق

يفحص المُثري `x-ves-validation-rules` للاستنتاج حول الحالة المطلوبة:

| القاعدة | الدلالة |
|------|-------------|
| `ves.io.schema.rules.message.required: "true"` | الحقل مطلوب |
| `ves.io.schema.rules.uint32.gte: N` | إذا كانت N >= 1 ولا يوجد افتراضي للخادم، فالحقل مطلوب |
| `ves.io.schema.rules.repeated.min_items: N` | إذا كانت N >= 1، يتطلب المصفوفة N عنصراً على الأقل |
| `ves.io.schema.rules.string.min_bytes: N` | إذا كانت N >= 1، تتطلب السلسلة N بايت على الأقل |

### الموارد التي تحتوي على إعدادات افتراضية مُطبَّقة من الخادم

تقبل بعض الموارد مواصفات فارغة لأن الخادم يطبق الإعدادات الافتراضية:

| المورد | الإعدادات الافتراضية المُطبَّقة من الخادم |
|----------|------------------------|
| `app_firewall` | `monitoring: \{\}`, `default_detection_settings: \{\}` |
| `rate_limiter` | `limits: []`, `user_identification: []` |
| `api_definition` | `swagger_specs: []`, default `api_groups` |
| `healthcheck` | `jitter: 0`, `jitter_percent: 0`, nested http_health_check defaults |

لهذه الموارد، قد تكون `x-f5xc-required-for.create` بقيمة `false` حتى عندما تكون `x-ves-required` بقيمة `true`.

### أنماط القيمة الافتراضية

| النمط | النوع | المثال |
|---------|------|---------|
| `\{\}` | كائن فارغ (تحديد الخيار) | `monitoring: \{\}` |
| `[]` | مصفوفة فارغة | `expected_status_codes: []` |
| `0` | رقمي | `jitter: 0` |
| `""` | نص | `expected_response: ""` |
| `false` | منطقي | `use_http2: false` |

### إضافة الإعدادات الافتراضية المكتشفة

1. أنشئ المورد في F5 XC بمواصفة بسيطة عبر API
2. اقرأ المورد مجدداً لرؤية القيم التي طبّقها الخادم
3. وثّق الإعدادات الافتراضية في `config/discovered_defaults.yaml`
4. شغّل خط الأنابيب: `make pipeline`
5. تحقق باستخدام: `jq '.components.schemas | to_entries[] | select(.key | contains("resource_name"))'`

## استكشاف الأخطاء وإصلاحها

### pre-commit يستغرق وقتاً طويلاً

يعمل خط الأنابيب عند كل إيداع لضمان التناسق. للإيداعات الجارية قيد العمل:

```bash
git commit --no-verify -m "WIP: work in progress"
# Run before final commit: make pre-commit-run
```

### فشل الاكتشاف

```bash
# Check VPN
ping your-tenant.console.ves.volterra.io

# Check credentials
echo $F5XC_API_TOKEN | head -c 10

# Check API URL format
echo $F5XC_API_URL
# Format: https://tenant.console.ves.volterra.io/api
```

### أخطاء التدقيق في المواصفات المُولَّدة

```bash
# Check lint report
cat reports/lint-report.json | jq '.errors'
```

أصلح المشكلات في إعداد الإثراء/التطبيع، وليس في ملفات المخرجات.

### نظام الإصدار

يُشتق الإصدار من علامات git (مثل `v2.0.38`)، مما يُلغي حالات التعارض التي كانت تُسبب تعارضات الدمج في طلبات السحب المتزامنة.

```python
# Version is calculated dynamically from git tags
from scripts.utils.version_calculator import get_version_from_tags
version = get_version_from_tags()  # Returns "2.0.38"
```

### المواصفات مفقودة بعد الاستنساخ

المواصفات المُولَّدة مُستثناة من Git:

```bash
make build  # Downloads and generates everything
```

### ملف كبير محظور

أضف استثناءً إلى `.pre-commit-config.yaml`:

```yaml
- id: check-added-large-files
  exclude: ^path/to/large/file\.json$
```
