يصف هذا الدليل سير عمل التطوير لخط أنابيب إثراء API الخاص بـ F5 XC.

## البداية السريعة

### الإعداد لأول مرة

```bash
# Clone the repository
git clone https://github.com/f5xc-salesdemos/api-specs-enriched.git
cd api-specs-enriched

# Create virtual environment and install dependencies
make install

# Install pre-commit hooks
make pre-commit-install

# Download specs and run pipeline
make build

# Preview documentation locally
make serve
```

### التطوير اليومي

```bash
# Quick rebuild (uses cached specs)
make rebuild

# Run all quality checks
make pre-commit-run

# Preview documentation
make serve
```

## أنماط سير العمل

### 1. سير عمل الاكتشاف

**المتطلبات الأساسية**:
- اتصال VPN ببيئة F5 XC
- بيانات اعتماد API صالحة

```bash
# Set credentials
export F5XC_API_URL="https://your-tenant.console.ves.volterra.io/api"
export F5XC_API_TOKEN="your-api-token"

# Run discovery
make discover

# View results
jq '.statistics' specs/discovered/session.json

# Commit for CI/CD
make push-discovery
```

**يلتقط الاكتشاف**:
- الحقول المطلوبة/الاختيارية الفعلية
- قيم التعداد من الاستجابات الحية
- القيم الافتراضية
- عمليات التحقق من الأنماط
- أمثلة الاستجابات

### 2. سير عمل الإصدار

الإصدارات مؤتمتة:

1. الجدولة اليومية (6 صباحاً بتوقيت UTC) أو الدفع إلى main يُشغّل سير العمل
2. فحص ETag يحدد ما إذا كانت مواصفات F5 قد تغيرت
3. يعالج خط الأنابيب المواصفات ويُثريها
4. يُحسب الإصدار من وسوم git: `git describe --tags --abbrev=0`
5. يتم إنشاء التزام مباشر + وسم (بدون PR لرفع الإصدار)
6. يتم إنشاء إصدار GitHub مع سجل التغييرات
7. يتم نشر التوثيق على GitHub Pages

**قواعد رفع الإصدار**:

| الشرط | نوع الرفع | مثال |
|-----------|-----------|---------|
| `[major]` في الالتزام | رئيسي | 1.0.0 → 2.0.0 |
| `BREAKING CHANGE` في الالتزام | رئيسي | 1.0.0 → 2.0.0 |
| إضافة مواصفة نطاق جديدة | ثانوي | 1.0.0 → 1.1.0 |
| أي تغيير آخر | تصحيحي | 1.0.0 → 1.0.1 |

### 3. سير عمل التطوير

```bash
# Create feature branch
git checkout -b feature/my-change

# Make changes to config or scripts

# Test locally
make pipeline
make lint

# Commit (pre-commit hooks run automatically)
git add .
git commit -m "feat: add new enrichment rule"

# Push and create PR
git push -u origin feature/my-change
gh pr create
```

## الاكتشاف بالتفصيل

### ما هو الاكتشاف؟

يستكشف الاكتشاف API الحي لـ F5 XC للعثور على:

- **القيود غير الموثقة**: الحقول المطلوبة غير المُعلّمة في OpenAPI
- **قيم التعداد الفعلية**: القيم الحقيقية المُشاهدة في الإنتاج
- **السلوكيات الافتراضية**: القيم التي يطبقها الخادم عند حذف الحقول
- **أنماط الاستجابة**: أشكال البيانات الفعلية

### تكوين الاكتشاف

```yaml
# config/discovery.yaml
discovery:
  exploration:
    namespaces:
      - "system"
      - "shared"
    methods:
      - "GET"
      - "OPTIONS"
    max_endpoints_per_run: 500

  schema_inference:
    sample_size: 3
    detect_patterns: true
    detect_constraints: true
```

### ملحقات الاكتشاف

يضيف الاكتشاف ملحقات `x-discovered-*` إلى المواصفات:

```json
{
  "properties": {
    "name": {
      "type": "string",
      "x-discovered-required": true,
      "x-discovered-pattern": "^[a-z][a-z0-9-]*$",
      "x-discovered-examples": ["my-app", "prod-lb"]
    }
  }
}
```

### تقرير تحليل القيود

```bash
make constraint-report
# Output: reports/constraint-analysis.md
```

## ملحقات OpenAPI

يستخدم خط أنابيب الإثراء ملحقات المورّد لتضمين بيانات التحقق والقيم الافتراضية.

### فئات الملحقات

| الملحق | المعنى | الدلالة |
|-----------|---------|-------------|
| `x-f5xc-server-default` | خادم F5 XC API يطبق هذه القيمة عند حذف الحقل | الحقل اختياري؛ حذفه ينتج السلوك الافتراضي الموثق |
| `x-f5xc-recommended-value` | وحدة تحكم F5 XC الويب تملأ هذه القيمة مسبقاً للموارد الجديدة | الحقل ليس له قيمة افتراضية من الخادم لكن هذه القيمة تمثل التكوين النموذجي |
| `x-f5xc-recommended-oneof-variant` | وحدة تحكم F5 XC تحدد مسبقاً هذا المتغير OneOf | يحدد الخيار النموذجي عند وجود خيارات متعددة متنافية |
| `x-f5xc-conflicts-with` | يسرد الخصائص الأخرى التي لا يمكن استخدامها مع هذا الحقل | الخاصية جزء من مجموعة OneOf؛ يمكن تحديد واحدة فقط من الخصائص المتعارضة |

### ملحقات الحقول المطلوبة

| الملحق | المصدر | الغرض |
|-----------|--------|---------|
| `x-ves-required: "true"` | مواصفات F5 XC الأصلية | الحقل يتطلب قيمة غير صفرية/غير فارغة |
| `x-f5xc-required-for` | خط أنابيب الإثراء | حالة الطلب حسب السياق |

**سياقات x-f5xc-required-for**:
- `create`: مطلوب عند إنشاء المورد
- `update`: مطلوب عند تحديث المورد
- `minimum_config`: مطلوب للحد الأدنى من التكوين القابل للتطبيق

### ملحقات القيم الافتراضية و OneOf

| الملحق | الغرض |
|-----------|---------|
| `x-f5xc-server-default: true` | يُعلّم القيم الافتراضية المُطبّقة من الخادم |
| `x-f5xc-recommended-value` | القيمة المملوءة مسبقاً في وحدة تحكم F5 XC |
| `x-f5xc-recommended-oneof-variant` | متغير OneOf الموصى به |
| `x-f5xc-conflicts-with` | التنافي المتبادل مع خصائص OneOf الأخرى |

#### x-f5xc-server-default

**النوع**: `boolean`

عندما تكون القيمة `true`، يشير إلى أن قيمة `default` المرافقة يتم فرضها بواسطة خادم F5 XC API. يمكن حذف الحقول التي تحمل هذا الملحق بأمان من طلبات API — يطبق الخادم القيمة الافتراضية تلقائياً.

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

#### x-f5xc-recommended-value

**النوع**: `any` (يطابق نوع الحقل)

يحدد قيمة تستخدمها وحدة تحكم F5 XC الويب كقيمة افتراضية مملوءة مسبقاً. هذه القيمة ليست مفروضة من الخادم لكنها تمثل تكوين البداية النموذجي للموارد الجديدة المُنشأة عبر وحدة التحكم.

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

#### x-f5xc-recommended-oneof-variant

**النوع**: `object` (خريطة من اسم المجموعة إلى اسم المتغير)

للمخططات ذات مجموعات الحقول المتنافية، يحدد أي متغير هو الافتراضي أو الأكثر شيوعاً. المفتاح هو اسم مجموعة OneOf والقيمة هي اسم حقل المتغير الموصى به.

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

#### x-f5xc-conflicts-with

**النوع**: `array` من السلاسل النصية

**تمت الإضافة**: v3.2.0 (المشكلة #494)

يعلن عن علاقات التنافي المتبادل بين أعضاء مجموعة OneOf. يتم اشتقاقها تلقائياً من ملحقات `x-ves-oneof-field-*`. يمكّن هذا الأدوات المتلقية (Terraform، CLI، MCP) من التحقق من التعارضات على مستوى المخطط بدلاً من وقت التشغيل.

```yaml
host_header:
  type: string
  x-f5xc-conflicts-with: ["use_origin_server_name"]

use_origin_server_name:
  type: object
  x-f5xc-conflicts-with: ["host_header"]
```

**حالات الاستخدام**:
- يمكن لمزودي Terraform إنشاء قواعد تحقق
- يمكن لأدوات CLI التحذير من تركيبات الحقول المتعارضة
- يمكن لمساعدي الذكاء الاصطناعي إنشاء تكوينات OneOf صحيحة
- يمكن لملحقات IDE توفير إكمال تلقائي مدرك للتعارضات

**المصدر**: مشتقة تلقائياً من ملحقات F5 الأصلية `x-ves-oneof-field-*` أثناء إثراء خط الأنابيب.

### دلالات قواعد التحقق

يفحص المُثري `x-ves-validation-rules` لاستنتاج حالة الطلب:

| القاعدة | الدلالة |
|------|-------------|
| `ves.io.schema.rules.message.required: "true"` | الحقل مطلوب |
| `ves.io.schema.rules.uint32.gte: N` | إذا كان N >= 1 ولا توجد قيمة افتراضية من الخادم، الحقل مطلوب |
| `ves.io.schema.rules.repeated.min_items: N` | إذا كان N >= 1، المصفوفة تتطلب N عنصر على الأقل |
| `ves.io.schema.rules.string.min_bytes: N` | إذا كان N >= 1، السلسلة النصية تتطلب N بايت على الأقل |

### الموارد ذات القيم الافتراضية المُطبّقة من الخادم

بعض الموارد تقبل مواصفات فارغة لأن الخادم يطبق القيم الافتراضية:

| المورد | القيم الافتراضية المُطبّقة من الخادم |
|----------|------------------------|
| `app_firewall` | `monitoring: \{\}`, `default_detection_settings: \{\}` |
| `rate_limiter` | `limits: []`, `user_identification: []` |
| `api_definition` | `swagger_specs: []`, `api_groups` افتراضية |
| `healthcheck` | `jitter: 0`, `jitter_percent: 0`, قيم http_health_check الافتراضية المتداخلة |

لهذه الموارد، قد تكون `x-f5xc-required-for.create` بقيمة `false` حتى عندما تكون `x-ves-required` بقيمة `true`.

### أنماط القيم الافتراضية

| النمط | النوع | مثال |
|---------|------|---------|
| `\{\}` | كائن فارغ (تحديد خيار) | `monitoring: \{\}` |
| `[]` | مصفوفة فارغة | `expected_status_codes: []` |
| `0` | رقمي | `jitter: 0` |
| `""` | سلسلة نصية | `expected_response: ""` |
| `false` | قيمة منطقية | `use_http2: false` |

### إضافة القيم الافتراضية المُكتشفة

1. أنشئ مورداً في F5 XC بمواصفات بسيطة عبر API
2. اقرأ المورد مجدداً لرؤية القيم المُطبّقة من الخادم
3. وثّق القيم الافتراضية في `config/discovered_defaults.yaml`
4. شغّل خط الأنابيب: `make pipeline`
5. تحقق باستخدام: `jq '.components.schemas | to_entries[] | select(.key | contains("resource_name"))'`

## استكشاف الأخطاء وإصلاحها

### فحوصات Pre-commit تستغرق وقتاً طويلاً

يعمل خط الأنابيب على كل التزام لضمان الاتساق. للالتزامات قيد العمل:

```bash
git commit --no-verify -m "WIP: work in progress"
# Run before final commit: make pre-commit-run
```

### فشل الاكتشاف

```bash
# Check VPN
ping your-tenant.console.ves.volterra.io

# Check credentials
echo $F5XC_API_TOKEN | head -c 10

# Check API URL format
echo $F5XC_API_URL
# Format: https://tenant.console.ves.volterra.io/api
```

### أخطاء الفحص على المواصفات المُنشأة

```bash
# Check lint report
cat reports/lint-report.json | jq '.errors'
```

أصلح المشكلات في تكوين الإثراء/التطبيع، وليس في ملفات المخرجات.

### نظام الإصدارات

يُشتق الإصدار من وسوم git (مثل `v2.0.38`)، مما يزيل حالات السباق التي كانت تسبب تعارضات الدمج في طلبات السحب المتزامنة.

```python
# Version is calculated dynamically from git tags
from scripts.utils.version_calculator import get_version_from_tags
version = get_version_from_tags()  # Returns "2.0.38"
```

### مواصفات مفقودة بعد الاستنساخ

المواصفات المُنشأة متجاهلة في git:

```bash
make build  # Downloads and generates everything
```

### ملف كبير محظور

أضف استثناءً إلى `.pre-commit-config.yaml`:

```yaml
- id: check-added-large-files
  exclude: ^path/to/large/file\.json$
```
