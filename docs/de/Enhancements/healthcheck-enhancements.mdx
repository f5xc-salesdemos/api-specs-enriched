Anreicherungsmetadaten für Healthcheck-bezogene Schemas. Siehe [OpenAPI-Erweiterungen](../development/#openapi-extensions) für Erweiterungsdefinitionen.

## Angereicherte Schemas

### Muster

Alle Schemas, die dem Muster `healthcheck.*SpecType` entsprechen, erhalten Anreicherungen:

- `healthcheckCreateSpecType`
- `healthcheckReplaceSpecType`
- `healthcheckGetSpecType`

Zusätzlich enthält das verschachtelte Schema `healthcheckHttpHealthCheck` Anreicherungen für die HTTP-Healthcheck-Konfiguration.

## Serverseitig angewandte Standardwerte

Felder, die mit `x-f5xc-server-default: true` gekennzeichnet sind, erhalten ihren `default`-Wert vom F5 XC API-Server, wenn sie in Anfragen ausgelassen werden.

### Felder auf oberster Ebene

| Feld | Standardwert | Typ | Beschreibung |
|------|-------------|-----|--------------|
| `jitter` | `0` | integer | Absoluter Jitter-Wert für Timing-Randomisierung |
| `jitter_percent` | `0` | integer | Prozentbasierter Jitter für Timing-Randomisierung |

### healthcheckHttpHealthCheck-Schema

| Feld | Standardwert | Typ | Beschreibung |
|------|-------------|-----|--------------|
| `use_origin_server_name` | `{}` | object | Origin-Server-Name für Host-Header |
| `headers` | `{}` | object | Benutzerdefinierte Header |
| `request_headers_to_remove` | `[]` | array | Header, die aus Anfragen entfernt werden |
| `use_http2` | `false` | boolean | HTTP/2-Unterstützung |
| `expected_status_codes` | `[]` | array | Akzeptierte Statuscodes (leer = 200-299) |

## Empfohlene Werte

Felder, die mit `x-f5xc-recommended-value` gekennzeichnet sind, geben Werte an, die die F5 XC Webkonsole beim Erstellen neuer Ressourcen vorab ausfüllt.

### Felder auf oberster Ebene

| Feld | Empfohlener Wert | Typ | Beschreibung |
|------|-----------------|-----|--------------|
| `timeout` | `3` | integer | Healthcheck-Timeout in Sekunden |
| `interval` | `15` | integer | Intervall zwischen Healthchecks in Sekunden |
| `unhealthy_threshold` | `1` | integer | Aufeinanderfolgende Fehler vor der Markierung als ungesund |
| `healthy_threshold` | `3` | integer | Aufeinanderfolgende Erfolge vor der Markierung als gesund |
| `jitter_percent` | `30` | integer | Jitter-Prozentsatz für den Produktionseinsatz |

### healthcheckHttpHealthCheck-Schema

| Feld | Empfohlener Wert | Typ | Beschreibung |
|------|-----------------|-----|--------------|
| `path` | `"/"` | string | Healthcheck-Endpunktpfad |
| `use_http2` | `false` | boolean | HTTP/2-Unterstützungseinstellung |
| `expected_status_codes` | `["200"]` | array | Statuscodes, die einen gesunden Origin anzeigen |
| `use_origin_server_name` | `{}` | object | Origin-Server-Name für Host-Header |

## OneOf-Varianten-Empfehlungen

Schemas, die sich gegenseitig ausschließende Feldgruppen (OneOf) enthalten, beinhalten `x-f5xc-recommended-oneof-variant`, um anzuzeigen, welche Variante am häufigsten verwendet wird. Diese Metadaten identifizieren die typische Wahl, wenn mehrere Optionen existieren, basierend auf F5 XC-Konsolenstandards und gängigen Nutzungsmustern.

### OneOf-Gruppen auf oberster Ebene

| Schema | OneOf-Gruppe | Empfohlene Variante | Beschreibung |
|--------|-------------|--------------------:|--------------|
| `healthcheckCreateSpecType` | `health_check` | `http_health_check` | HTTP-Healthcheck-Typ |
| `healthcheckReplaceSpecType` | `health_check` | `http_health_check` | HTTP-Healthcheck-Typ |

### healthcheckHttpHealthCheck OneOf-Gruppen

| Schema | OneOf-Gruppe | Empfohlene Variante | Beschreibung |
|--------|-------------|--------------------:|--------------|
| `healthcheckHttpHealthCheck` | `host_header_choice` | `use_origin_server_name` | Host-Header für Healthcheck-Anfragen |

#### Host-Header-Auswahl

Die OneOf-Gruppe `host_header_choice` steuert, wie der Host-Header in HTTP-Healthcheck-Anfragen angegeben wird.

| Variante | Typ | Beschreibung |
|----------|-----|--------------|
| `use_origin_server_name` | `object` (leer) | Verwendet den Origin-Server-Namen als Host-Header. Dies ist die Standardauswahl in der F5 XC-Konsole. |
| `host_header` | `string` | Gibt einen benutzerdefinierten Host-Header-Wert für Healthcheck-Anfragen an. |

**API-Schema-Referenz**: `x-ves-oneof-field-host_header_choice: ["host_header", "use_origin_server_name"]`

## OpenAPI-Erweiterungsreferenz

Diese herstellerspezifischen Erweiterungen werden dem standardmäßigen OpenAPI-Schema hinzugefügt, um F5 XC-spezifisches Standardverhalten zu vermitteln.

### x-f5xc-server-default

**Typ**: `boolean`

Wenn `true`, zeigt dies an, dass der zugehörige `default`-Wert vom F5 XC API-Server durchgesetzt wird. Felder mit dieser Erweiterung können sicher aus API-Anfragen ausgelassen werden – der Server wendet den Standardwert automatisch an.

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

### x-f5xc-recommended-value

**Typ**: `any` (entspricht dem Feldtyp)

Gibt einen Wert an, den die F5 XC Webkonsole als vorab ausgefüllten Standardwert verwendet. Dieser Wert wird nicht serverseitig durchgesetzt, stellt aber die typische Ausgangskonfiguration für neue Ressourcen dar, die über die Konsole erstellt werden.

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

### x-f5xc-recommended-oneof-variant

**Typ**: `object` (Zuordnung von Gruppenname zu Variantenname)

Für Schemas mit sich gegenseitig ausschließenden Feldgruppen wird angegeben, welche Variante die Standard- oder häufigste Wahl ist. Der Schlüssel ist der OneOf-Gruppenname und der Wert ist der empfohlene Varianten-Feldname.

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

```text

├── server_applied      # Felder mit x-f5xc-server-default: true

└── nested_recommended  # Empfohlene Werte für verschachtelte Schemas
```
