Pod-Orchestrierung ohne die volle Cluster-Komplexität. Edge-Site-Ausführung, Quota-Durchsetzung und standardisierte Compute-Profile für verteilte Anwendungen.

- **Kategorie**: Infrastruktur
- **Komplexität**: Mittel
- **Pfade**: 13 | **Schemas**: 159
- **Stufe**: Erweitert
- **Verwandte Domänen**: `managed_kubernetes`, `sites`, `service_mesh`

## Anwendungsfälle

- Bereitstellung von XCCS (Container Services) Namespaces für mandantenfähige Workloads
- Verwaltung von Container-Workloads mit vereinfachter Orchestrierung
- Konfiguration verteilter Edge-Container-Bereitstellungen
- Ausführung containerisierter Anwendungen ohne die volle K8s-Komplexität

## Primäre Ressourcen

- **virtual_k8s**: Virtueller Kubernetes-Namespace für Container-Workloads
- **workload**: Konfiguration der Container-Workload-Bereitstellung
- **pod_security_policy**: Pod-Sicherheitsrichtlinie für Container-Laufzeitbeschränkungen
