Pod-Orchestrierung ohne vollständige Cluster-Komplexität. Edge-Site-Ausführung, Kontingentdurchsetzung und standardisierte Compute-Profile für verteilte Apps.

- **Kategorie**: Infrastruktur
- **Komplexität**: moderat
- **Pfade**: 13 | **Schemata**: 159
- **Stufe**: Erweitert
- **Verwandte Domänen**: `managed_kubernetes`, `sites`, `service_mesh`

## Anwendungsfälle

- XCCS (Container Services)-Namespaces für mandantenfähige Workloads bereitstellen
- Container-Workloads mit vereinfachter Orchestrierung verwalten
- Verteilte Edge-Container-Deployments konfigurieren
- Containerisierte Anwendungen ohne vollständige K8s-Komplexität ausführen

## Primäre Ressourcen

- **virtual_k8s**: Virtueller Kubernetes-Namespace für Container-Workloads
- **workload**: Konfiguration für Container-Workload-Deployment
- **pod_security_policy**: Pod-Sicherheitsrichtlinie für Container-Laufzeitbeschränkungen
