Richtlinien für sensible Daten mit benutzerdefinierten Klassifizierungsregeln. LMA-Regionskonfiguration und geobasierte Compliance-Steuerungen.

- **Kategorie**: Sicherheit
- **Komplexität**: einfach
- **Pfade**: 11 | **Schemas**: 65
- **Stufe**: Erweitert
- **Verwandte Domänen**: `blindfold`, `client_side_defense`

## Anwendungsfälle

- Richtlinien zur Erkennung sensibler Daten konfigurieren
- Benutzerdefinierte Datentypen für die PII-Klassifizierung definieren
- LMA-Regionskonfigurationen verwalten
- Geo-Konfigurationen für Compliance integrieren

## Primäre Ressourcen

- **sensitive_data_policy**: Richtlinie für sensible Daten zum PII-Schutz
- **data_classification**: Datenklassifizierung zur Inhaltskategorisierung
