Kubernetes-Rollenbindungen und Zugangsrichtlinien. Registry-Integration für EKS-, AKS- und GKE-Workloads.

- **Kategorie**: Infrastruktur
- **Komplexität**: moderat
- **Pfade**: 20 | **Schemas**: 95
- **Stufe**: Erweitert
- **Verwandte Domänen**: `container_services`, `sites`, `service_mesh`

## Anwendungsfälle

- Cluster-RBAC und Sicherheit für XCKS (Verwaltetes Kubernetes) verwalten
- Pod-Sicherheitsrichtlinien und Zugangscontroller konfigurieren
- Container-Registrys für Unternehmensbereitstellungen verwalten
- Mit externen Kubernetes-Clustern (EKS, AKS, GKE) integrieren

## Primäre Ressourcen

- **mk8s_cluster**: Konfiguration des verwalteten Kubernetes-Clusters
- **k8s_cluster_role**: Kubernetes-Cluster-RBAC-Rollenkonfiguration
- **container_registry**: Container-Image-Registry für Workload-Images
