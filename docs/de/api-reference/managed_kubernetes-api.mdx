Kubernetes-Rollenbindungen und Admission-Richtlinien. Registry-Integration für EKS-, AKS- und GKE-Workloads.

- **Kategorie**: Infrastruktur
- **Komplexität**: mittel
- **Pfade**: 20 | **Schemas**: 95
- **Stufe**: Erweitert
- **Verwandte Domänen**: `container_services`, `sites`, `service_mesh`

## Anwendungsfälle

- XCKS (Managed Kubernetes) Cluster-RBAC und -Sicherheit verwalten
- Pod-Sicherheitsrichtlinien und Admission-Controller konfigurieren
- Container-Registries für Unternehmensbereitstellungen verwalten
- Integration mit externen Kubernetes-Clustern (EKS, AKS, GKE)

## Primäre Ressourcen

- **mk8s_cluster**: Konfiguration des Managed-Kubernetes-Clusters
- **k8s_cluster_role**: Konfiguration der Kubernetes-Cluster-RBAC-Rolle
- **container_registry**: Container-Image-Registry für Workload-Images
