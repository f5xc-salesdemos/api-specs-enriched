Firewall-Regeln mit Routing-Entscheidungen basierend auf Quelle, Ziel oder Protokoll. Segmentierung isoliert Workloads, während ausgehende Proxys den Zugriff steuern.

- **Kategorie**: Sicherheit
- **Komplexität**: Fortgeschritten
- **Pfade**: 62 | **Schemas**: 386
- **Stufe**: Erweitert
- **Verwandte Domänen**: `virtual`, `api`, `network`

## Anwendungsfälle

- Netzwerk-Firewall- und ACL-Richtlinien konfigurieren
- NAT-Richtlinien und Portweiterleitung verwalten
- Richtlinienbasiertes Routing konfigurieren
- Netzwerksegmente und Richtlinien definieren
- Forward-Proxy-Richtlinien konfigurieren

## Primäre Ressourcen

- **network_policy**: Netzwerksicherheitsrichtlinie zur Datenverkehrsfilterung
- **forward_proxy_policy**: Forward-Proxy-Richtlinie zur Steuerung des ausgehenden Datenverkehrs
- **network_firewall**: Netzwerk-Firewall zum Schutz des Datenverkehrs auf Schicht 3/4
