Datenverkehrsverteilung über Regionen hinweg mit Routing-Regeln. Health Checks und Failover-Richtlinien.

- **Kategorie**: Netzwerk
- **Komplexität**: Fortgeschritten
- **Pfade**: 133 | **Schemas**: 903
- **Stufe**: Fortgeschritten
- **Verwandte Domänen**: `dns`, `service_policy`, `network`

## Anwendungsfälle

- HTTP/TCP/UDP-Load-Balancer konfigurieren
- Origin-Pools und Dienste verwalten
- Virtuelle Hosts und Routing konfigurieren
- Rate-Limiter und Service-Richtlinien definieren
- Geo-Standort-basiertes Routing verwalten
- Proxy- und Weiterleitungsrichtlinien konfigurieren
- Malware-Schutz und Bedrohungskampagnen verwalten
- Health Checks und Endpoint-Überwachung konfigurieren

## Primäre Ressourcen

- **http_loadbalancer**: Layer-7-HTTP/HTTPS-Load-Balancer für die Verteilung von Anwendungsdatenverkehr
- **tcp_loadbalancer**: Layer-4-TCP/UDP-Load-Balancer für Nicht-HTTP-Protokoll-Datenverkehr
- **origin_pool**: Backend-Servergruppe für die Load-Balancer-Datenverkehrsverteilung
- **healthcheck**: Health-Monitoring-Konfiguration für die Verfügbarkeit von Origin-Servern
- **app_firewall**: Web Application Firewall-Richtlinie zum Schutz von HTTP-Datenverkehr
- **service_policy**: Zugriffskontroll- und Datenverkehrsverwaltungsregeln auf Dienstebene
- **malicious_user_detection**: Automatische Erkennung und Abwehr von bösartigem Benutzerverhalten
