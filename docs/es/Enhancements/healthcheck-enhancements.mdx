Metadatos de enriquecimiento para esquemas relacionados con healthcheck. Consulte [Extensiones OpenAPI](../development/#openapi-extensions) para las definiciones de extensiones.

## Esquemas Enriquecidos

### Patrón

Todos los esquemas que coinciden con `healthcheck.*SpecType` reciben enriquecimientos:

- `healthcheckCreateSpecType`
- `healthcheckReplaceSpecType`
- `healthcheckGetSpecType`

Adicionalmente, el esquema anidado `healthcheckHttpHealthCheck` contiene enriquecimientos para la configuración de verificación de salud HTTP.

## Valores Predeterminados Aplicados por el Servidor

Los campos marcados con `x-f5xc-server-default: true` tienen su valor `default` aplicado por el servidor de la API de F5 XC cuando se omiten en las solicitudes.

### Campos de Nivel Superior

| Campo | Valor Predeterminado | Tipo | Descripción |
|-------|----------------------|------|-------------|
| `jitter` | `0` | integer | Valor absoluto de jitter para aleatorización de temporización |
| `jitter_percent` | `0` | integer | Jitter basado en porcentaje para aleatorización de temporización |

### Esquema healthcheckHttpHealthCheck

| Campo | Valor Predeterminado | Tipo | Descripción |
|-------|----------------------|------|-------------|
| `use_origin_server_name` | `{}` | object | Nombre del servidor de origen para el encabezado Host |
| `headers` | `{}` | object | Encabezados personalizados |
| `request_headers_to_remove` | `[]` | array | Encabezados a eliminar de las solicitudes |
| `use_http2` | `false` | boolean | Soporte HTTP/2 |
| `expected_status_codes` | `[]` | array | Códigos de estado aceptados (vacío = 200-299) |

## Valores Recomendados

Los campos marcados con `x-f5xc-recommended-value` indican valores que la consola web de F5 XC pre-rellena al crear nuevos recursos.

### Campos de Nivel Superior

| Campo | Valor Recomendado | Tipo | Descripción |
|-------|-------------------|------|-------------|
| `timeout` | `3` | integer | Tiempo de espera de la verificación de salud en segundos |
| `interval` | `15` | integer | Intervalo entre verificaciones de salud en segundos |
| `unhealthy_threshold` | `1` | integer | Fallos consecutivos antes de marcar como no saludable |
| `healthy_threshold` | `3` | integer | Éxitos consecutivos antes de marcar como saludable |
| `jitter_percent` | `30` | integer | Porcentaje de jitter para uso en producción |

### Esquema healthcheckHttpHealthCheck

| Campo | Valor Recomendado | Tipo | Descripción |
|-------|-------------------|------|-------------|
| `path` | `"/"` | string | Ruta del endpoint de verificación de salud |
| `use_http2` | `false` | boolean | Configuración de soporte HTTP/2 |
| `expected_status_codes` | `["200"]` | array | Códigos de estado que indican un origen saludable |
| `use_origin_server_name` | `{}` | object | Nombre del servidor de origen para el encabezado Host |

## Recomendaciones de Variantes OneOf

Los esquemas que contienen grupos de campos mutuamente excluyentes (OneOf) incluyen `x-f5xc-recommended-oneof-variant` para indicar qué variante es la más comúnmente utilizada. Estos metadatos identifican la elección típica cuando existen múltiples opciones, basándose en los valores predeterminados de la consola de F5 XC y los patrones de uso comunes.

### Grupos OneOf de Nivel Superior

| Esquema | Grupo OneOf | Variante Recomendada | Descripción |
|---------|-------------|----------------------|-------------|
| `healthcheckCreateSpecType` | `health_check` | `http_health_check` | Tipo de verificación de salud HTTP |
| `healthcheckReplaceSpecType` | `health_check` | `http_health_check` | Tipo de verificación de salud HTTP |

### Grupos OneOf de healthcheckHttpHealthCheck

| Esquema | Grupo OneOf | Variante Recomendada | Descripción |
|---------|-------------|----------------------|-------------|
| `healthcheckHttpHealthCheck` | `host_header_choice` | `use_origin_server_name` | Encabezado Host para solicitudes de verificación de salud |

#### Elección del Encabezado Host

El grupo OneOf `host_header_choice` controla cómo se especifica el encabezado Host en las solicitudes HTTP de verificación de salud.

| Variante | Tipo | Descripción |
|----------|------|-------------|
| `use_origin_server_name` | `object` (vacío) | Usa el nombre del servidor de origen como encabezado Host. Esta es la selección predeterminada en la consola de F5 XC. |
| `host_header` | `string` | Especifica un valor personalizado del encabezado Host para las solicitudes de verificación de salud. |

**Referencia del Esquema de API**: `x-ves-oneof-field-host_header_choice: ["host_header", "use_origin_server_name"]`

## Referencia de Extensiones OpenAPI

Estas extensiones de proveedor se agregan al esquema OpenAPI estándar para transmitir el comportamiento predeterminado específico de F5 XC.

### x-f5xc-server-default

**Tipo**: `boolean`

Cuando es `true`, indica que el valor `default` acompañante es aplicado por el servidor de la API de F5 XC. Los campos con esta extensión pueden omitirse de forma segura en las solicitudes de API: el servidor aplica el valor predeterminado automáticamente.

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

### x-f5xc-recommended-value

**Tipo**: `any` (coincide con el tipo del campo)

Especifica un valor que la consola web de F5 XC utiliza como valor pre-rellenado predeterminado. Este valor no es aplicado por el servidor, pero representa la configuración inicial típica para nuevos recursos creados a través de la consola.

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

### x-f5xc-recommended-oneof-variant

**Tipo**: `object` (mapa de nombre de grupo a nombre de variante)

Para esquemas con grupos de campos mutuamente excluyentes, identifica qué variante es la opción predeterminada o más común. La clave es el nombre del grupo OneOf y el valor es el nombre del campo de la variante recomendada.

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```
