Orquestación de pods sin la complejidad completa de un clúster. Ejecución en sitios de borde, cumplimiento de cuotas y perfiles de cómputo estandarizados para aplicaciones distribuidas.

- **Categoría**: Infraestructura
- **Complejidad**: moderada
- **Rutas**: 13 | **Esquemas**: 159
- **Nivel**: Avanzado
- **Dominios relacionados**: `managed_kubernetes`, `sites`, `service_mesh`

## Casos de Uso

- Desplegar namespaces de XCCS (Servicios de Contenedores) para cargas de trabajo multi-inquilino
- Gestionar cargas de trabajo de contenedores con orquestación simplificada
- Configurar despliegues de contenedores distribuidos en el borde
- Ejecutar aplicaciones en contenedores sin la complejidad completa de K8s

## Recursos Principales

- **virtual_k8s**: Namespace virtual de Kubernetes para cargas de trabajo en contenedores
- **workload**: Configuración de despliegue de carga de trabajo en contenedores
- **pod_security_policy**: Política de seguridad de pods para restricciones del entorno de ejecución de contenedores
