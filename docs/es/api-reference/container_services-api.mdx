Orquestación de pods sin la complejidad total de un clúster. Ejecución en sitios de borde, aplicación de cuotas y perfiles de cómputo estandarizados para aplicaciones distribuidas.

- **Categoría**: Infraestructura
- **Complejidad**: moderada
- **Rutas**: 13 | **Esquemas**: 159
- **Nivel**: Avanzado
- **Dominios relacionados**: `managed_kubernetes`, `sites`, `service_mesh`

## Casos de uso

- Implementar espacios de nombres XCCS (Container Services) para cargas de trabajo multiusuario
- Gestionar cargas de trabajo en contenedores con orquestación simplificada
- Configurar implementaciones de contenedores en borde distribuido
- Ejecutar aplicaciones en contenedores sin la complejidad total de K8s

## Recursos principales

- **virtual_k8s**: Espacio de nombres de Kubernetes virtual para cargas de trabajo en contenedores
- **workload**: Configuración de implementación de carga de trabajo en contenedor
- **pod_security_policy**: Política de seguridad de pod para restricciones del entorno de ejecución de contenedores
