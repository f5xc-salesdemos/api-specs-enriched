Políticas de datos sensibles con reglas de clasificación personalizadas. Configuración de regiones LMA y controles de cumplimiento basados en geolocalización.

- **Categoría**: Seguridad
- **Complejidad**: simple
- **Rutas**: 11 | **Esquemas**: 65
- **Nivel**: Avanzado
- **Dominios relacionados**: `blindfold`, `client_side_defense`

## Casos de Uso

- Configurar políticas de detección de datos sensibles
- Definir tipos de datos personalizados para la clasificación de PII
- Gestionar configuraciones de regiones LMA
- Integrar geo-configuraciones para cumplimiento normativo

## Recursos Principales

- **sensitive_data_policy**: Política de datos sensibles para la protección de PII
- **data_classification**: Clasificación de datos para la categorización de contenido
