Vínculos de roles y políticas de admisión de Kubernetes. Integración de registros para cargas de trabajo de EKS, AKS y GKE.

- **Categoría**: Infraestructura
- **Complejidad**: moderada
- **Rutas**: 20 | **Esquemas**: 95
- **Nivel**: Avanzado
- **Dominios relacionados**: `container_services`, `sites`, `service_mesh`

## Casos de uso

- Gestionar RBAC y seguridad de clústeres XCKS (Kubernetes gestionado)
- Configurar políticas de seguridad de pods y controladores de admisión
- Gestionar registros de contenedores para implementaciones empresariales
- Integrar con clústeres externos de Kubernetes (EKS, AKS, GKE)

## Recursos principales

- **mk8s_cluster**: Configuración de clúster de Kubernetes gestionado
- **k8s_cluster_role**: Configuración de rol RBAC de clúster de Kubernetes
- **container_registry**: Registro de imágenes de contenedores para imágenes de cargas de trabajo
