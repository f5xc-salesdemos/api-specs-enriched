Reglas de firewall con decisiones de enrutamiento basadas en origen, destino o protocolo. La segmentación aísla las cargas de trabajo mientras que los proxies de salida gobiernan el acceso.

- **Categoría**: Seguridad
- **Complejidad**: avanzada
- **Rutas**: 62 | **Esquemas**: 386
- **Nivel**: Avanzado
- **Dominios relacionados**: `virtual`, `api`, `network`

## Casos de uso

- Configurar políticas de firewall de red y ACL
- Gestionar políticas NAT y reenvío de puertos
- Configurar enrutamiento basado en políticas
- Definir segmentos de red y políticas
- Configurar políticas de proxy de reenvío

## Recursos principales

- **network_policy**: Política de seguridad de red para filtrado de tráfico
- **forward_proxy_policy**: Política de proxy de reenvío para el control del tráfico saliente
- **network_firewall**: Firewall de red para la protección del tráfico de capa 3/4
