Distribución de tráfico entre regiones con reglas de enrutamiento. Verificaciones de salud y políticas de conmutación por error.

- **Categoría**: Redes
- **Complejidad**: avanzada
- **Rutas**: 133 | **Esquemas**: 903
- **Nivel**: Avanzado
- **Dominios relacionados**: `dns`, `service_policy`, `network`

## Casos de Uso

- Configurar balanceadores de carga HTTP/TCP/UDP
- Gestionar pools de origen y servicios
- Configurar hosts virtuales y enrutamiento
- Definir políticas de limitación de velocidad y políticas de servicio
- Gestionar enrutamiento basado en geolocalización
- Configurar políticas de proxy y reenvío
- Gestionar protección contra malware y campañas de amenazas
- Configurar verificaciones de salud y monitoreo de endpoints

## Recursos Principales

- **http_loadbalancer**: Balanceador de carga HTTP/HTTPS de capa 7 para distribución de tráfico de aplicaciones
- **tcp_loadbalancer**: Balanceador de carga TCP/UDP de capa 4 para tráfico de protocolos no HTTP
- **origin_pool**: Grupo de servidores backend para distribución de tráfico del balanceador de carga
- **healthcheck**: Configuración de monitoreo de salud para disponibilidad de servidores de origen
- **app_firewall**: Política de Firewall de Aplicaciones Web para protección de tráfico HTTP
- **service_policy**: Reglas de control de acceso a nivel de servicio y gestión de tráfico
- **malicious_user_detection**: Detección y mitigación automatizada de comportamiento de usuarios maliciosos
