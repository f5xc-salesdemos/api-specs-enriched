Métadonnées d'enrichissement pour les schémas HTTP load balancer. Voir [Extensions OpenAPI](../development/#openapi-extensions) pour les définitions des extensions.

## Schémas enrichis

### Modèle

Tous les schémas correspondant à `viewshttp_loadbalancer.*SpecType` reçoivent des enrichissements :

- `viewshttp_loadbalancerCreateSpecType`
- `viewshttp_loadbalancerReplaceSpecType`
- `viewshttp_loadbalancerGetSpecType`

## Métadonnées de configuration minimale

L'extension `x-f5xc-minimum-configuration` fournit des métadonnées CLI complètes pour créer des HTTP load balancers minimaux viables.

### Champs requis

| Champ | Contrainte | Description |
|-------|-----------|-------------|
| `metadata.name` | Format label DNS : `[a-z0-9]([-a-z0-9]*[a-z0-9])?` | Nom de la ressource |
| `metadata.namespace` | Format label DNS | Espace de noms |
| `spec.domains` | Tableau, min_items: 1 | Domaines à servir |

### Type de Load Balancer (OneOf requis)

Un des types de load balancer suivants doit être spécifié :

| Variante | Description |
|---------|-------------|
| `spec.http` | HTTP uniquement |
| `spec.https` | HTTPS avec certificat manuel |
| `spec.https_auto_cert` | HTTPS avec certificat automatique |
| `spec.http_https` | HTTP et HTTPS |

**Note** : Le groupe OneOf du type de load balancer est désigné comme :
- `lb_type` dans les fichiers de configuration (forme courte dans `config/minimum_configs.yaml`)
- `loadbalancer_type` dans les extensions natives de la spécification OpenAPI (nom de champ natif)

**Référence du schéma API** : `x-ves-oneof-field-loadbalancer_type: ["http", "https", "https_auto_cert", "http_https"]`

### Configuration minimale viable

```json
{
  "metadata": {
    "name": "example-app",
    "namespace": "default"
  },
  "spec": {
    "domains": ["example.com"],
    "https_auto_cert": {
      "port": 443,
      "tls_config": {"default_security": {}}
    },
    "advertise_on_public_default_vip": {},
    "routes": [{"prefix": "/", "origin_pool": {"pool_name": "backend-pool"}}]
  }
}
```

## Valeurs par défaut appliquées par le serveur

Les champs marqués avec `x-f5xc-server-default: true` ont leur valeur `default` appliquée par le serveur API F5 XC lorsqu'ils sont omis des requêtes. Le HTTP load balancer possède un grand nombre de valeurs par défaut appliquées par le serveur car la plupart des fonctionnalités de sécurité et des options de protocole sont par défaut dans un état désactivé ou sûr.

### Valeurs par défaut HTTPS Auto-Cert

Lorsque `https_auto_cert` est spécifié, le serveur applique ces valeurs par défaut pour les champs omis :

| Champ | Valeur par défaut | Type | Description |
|-------|---------------|------|-------------|
| `port` | `443` | integer | Port d'écoute HTTPS |
| `http_redirect` | `false` | boolean | Redirection HTTP vers HTTPS |
| `add_hsts` | `false` | boolean | En-tête HTTP Strict Transport Security |
| `tls_config.default_security` | `{}` | object | TLS 1.2+ avec chiffrements forts |
| `no_mtls` | `{}` | object | TLS mutuel désactivé |
| `default_header` | `{}` | object | Gestion par défaut de l'en-tête du nom de serveur |
| `enable_path_normalize` | `{}` | object | Normalisation des chemins activée |
| `default_loadbalancer` | `{}` | object | Paramètres par défaut du load balancer |
| `header_transformation_type.legacy_header_transformation` | `{}` | object | Transformation d'en-tête héritée |
| `connection_idle_timeout` | `120000` | integer | Délai d'inactivité de connexion en millisecondes (2 minutes) |
| `http_protocol_options.http_protocol_enable_v1_v2` | `{}` | object | HTTP/1.1 et HTTP/2 activés |
| `coalescing_options.default_coalescing` | `{}` | object | Coalescence de connexion HTTP/2 par défaut |

| `disable_malware_protection` | `{}` | Protection contre les logiciels malveillants désactivée |

## Recommandations de variantes OneOf (Extension future)

**Statut** : Pas encore appliqué aux schémas http_loadbalancer.

Contrairement au healthcheck, qui inclut `x-f5xc-recommended-oneof-variant` pour indiquer le choix le plus courant pour les groupes OneOf, http_loadbalancer ne dispose pas encore de cette extension.

### Comparaison avec Healthcheck

**Healthcheck** (implémenté) :
```yaml
viewshealthcheckCreateSpecType:
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

**HTTP LoadBalancer** (pas encore implémenté) :
```yaml
viewshttp_loadbalancerCreateSpecType:
  x-f5xc-recommended-oneof-variant: null  # Would indicate recommended lb_type variant
```

### Implémentation future

Pour ajouter cette extension pour http_loadbalancer :
1. Observer les sélections par défaut de la console F5 XC (par exemple, quel lb_type est présélectionné dans l'interface)
2. Ajouter la configuration à `config/discovered_defaults.yaml`
3. Relancer le pipeline d'enrichissement pour appliquer l'extension

**Structure attendue une fois ajoutée** :
```yaml
viewshttp_loadbalancerCreateSpecType:
  x-f5xc-recommended-oneof-variant:
    loadbalancer_type: "https_auto_cert"  # Example - requires verification
```

Cette extension permettrait aux outils en aval de présélectionner la variante la plus couramment utilisée lors de la présentation des options de configuration aux utilisateurs.

## Intégration des workflows guidés

Le pipeline d'enrichissement ajoute des métadonnées `x-f5xc-guided-workflows` au niveau de la spécification. Ces workflows fournissent des instructions de déploiement étape par étape consommées par les assistants IA et les outils CLI.

### Déployer un HTTP Load Balancer

Un workflow en 5 étapes pour créer un HTTP load balancer entièrement configuré avec un pool d'origine backend (défini dans `config/guided_workflows.yaml`) :

| Étape | Action | Ressource | Champs requis | Optionnel |
|------|--------|----------|-----------------|----------|
| 1 | Créer le pool d'origine | `origin_pool` | `name`, `origin_servers`, `port` | Non |
| 2 | Configurer le contrôle de santé | `healthcheck` | `name`, `http_health_check` | Oui |
| 3 | Attacher le contrôle de santé | - | - | Oui |
| 4 | Créer le HTTP Load Balancer | `http_loadbalancer` | `name`, `domains`, `http.port` | Non |
| 5 | Vérifier le déploiement | - | - | Non |

**Prérequis** : Espace de noms valide dans le tenant cible, application backend accessible via IP ou DNS, certificat SSL (optionnel pour HTTPS).

### Déployer un HTTPS Load Balancer avec TLS

Un workflow en 7 étapes pour créer un HTTPS load balancer avec terminaison SSL/TLS :

| Étape | Action | Ressource | Champs requis | Optionnel |
|------|--------|----------|-----------------|----------|
| 1 | Télécharger le certificat SSL | `certificate` | `name`, `certificate_chain`, `private_key` | Non |
| 2 | Créer le pool d'origine | `origin_pool` | `name`, `origin_servers` | Non |
| 3 | Configurer le contrôle de santé | `healthcheck` | - | Oui |
| 4 | Configurer la politique WAF | `app_firewall` | - | Oui |
| 5 | Créer le HTTPS Load Balancer | `http_loadbalancer` | `name`, `domains`, `https.tls_parameters` | Non |
| 6 | Attacher la politique WAF | - | - | Oui |
| 7 | Vérifier le déploiement HTTPS | - | - | Non |

**Prérequis** : Espace de noms valide dans le tenant cible, certificat SSL et clé privée, application backend accessible.

### Consommation des workflows

Les outils en aval utilisent ces workflows pour guider les utilisateurs à travers des déploiements en plusieurs étapes :

- **xcsh CLI** : Présente les workflows sous forme de commandes guidées interactives (par exemple, `xcsh deploy http-lb`)
- **Extension VS Code** : Affiche les étapes du workflow dans un panneau latéral avec suivi de la progression
- **Assistants IA** : Suivent les étapes du workflow séquentiellement, créant chaque ressource et validant les dépendances

## Référence des extensions OpenAPI

Ces extensions vendor sont ajoutées au schéma OpenAPI standard pour transmettre des métadonnées spécifiques à F5 XC.

### x-f5xc-cli-domain

**Type** : `string`

Indique la classification de domaine pour l'organisation CLI et outillage.

```yaml
viewshttp_loadbalancerCreateSpecType:
  type: object
  x-f5xc-cli-domain: "virtual"
```

### x-f5xc-minimum-configuration

**Type** : `object`

Fournit des métadonnées complètes pour créer des configurations minimales viables. Inclut :
- Description
- Champs requis avec contraintes
- Groupes mutuellement exclusifs
- Exemples de configuration (YAML, JSON)
- Exemples de commandes curl

```yaml
viewshttp_loadbalancerCreateSpecType:
  type: object
  x-f5xc-minimum-configuration:
    description: "HTTP/HTTPS load balancer for distributing traffic across origin pools"
    required_fields:
      - "metadata.name"
      - "metadata.namespace"
      - "spec.domains"
    mutually_exclusive_groups:
      - name: "lb_type"
        fields: ["spec.http", "spec.https", "spec.https_auto_cert", "spec.http_https"]
        reason: "Choose exactly one load balancer type"
    example_yaml: |
      ...
    example_json: |
      ...
    example_curl: |
      ...
```

### x-f5xc-server-default

**Type** : `boolean`

Lorsque `true`, indique que la valeur `default` accompagnante est appliquée par le serveur API F5 XC. Les champs avec cette extension peuvent être omis en toute sécurité des requêtes API -- le serveur applique la valeur par défaut automatiquement.

```yaml
disable_waf:
  type: object
  default: {}
  x-f5xc-server-default: true
```

### x-f5xc-conflicts-with

**Type** : `array of strings`

Liste les noms de champs qui sont mutuellement exclusifs avec le champ actuel, indiquant des modèles OneOf.

```yaml
advertise_custom:
  type: object
  x-f5xc-conflicts-with:
    - advertise_on_public
    - advertise_on_public_default_vip
    - do_not_advertise
```

### x-f5xc-constraints

**Type** : `object`

Fournit des contraintes de validation incluant les limites de taille de tableau, les plages d'entiers, les exigences d'unicité et les métadonnées de découverte.

```yaml
routes:
  type: array
  x-f5xc-constraints:
    constraintType: "array"
    minItems: 1
    maxItems: 256
    uniqueItems: false
    metadata:
      source: "discovery"
      confidence: 0.99
      validatedAt: "2026-01-19T12:00:00Z"
```

```yaml
port:
  type: integer
  x-f5xc-constraints:
    constraintType: "number"
    minimum: 1
    maximum: 65535
    metadata:
      source: "discovery"
      confidence: 0.99
```

### x-f5xc-description-short et x-f5xc-description-medium

**Type** : `string`

Fournit des longueurs de description alternatives pour différents cas d'utilisation :
- `x-f5xc-description-short` : 60 caractères maximum (colonnes CLI, badges)
- `x-f5xc-description-medium` : 150 caractères maximum (infobulles, résumés)

```yaml
domains:
  type: array
  description: "List of domains the load balancer will serve"
  x-f5xc-description-short: "Domains to serve"
  x-f5xc-description-medium: "List of domains for which the load balancer accepts traffic"
```

### x-f5xc-example

**Type** : `any` (correspond au type du champ)

Fournit des exemples concrets de champs pour la documentation et l'outillage.

```yaml
name:
  type: string
  x-f5xc-example: "example-app"

domains:
  type: array
  x-f5xc-example: ["example.com", "www.example.com"]
```

### x-f5xc-required-for

**Type** : `array of strings`

Indique les exigences spécifiques au contexte à l'aide d'indicateurs : `minimum_config`, `create`, `update`, `read`.

```yaml
metadata.name:
  type: string
  x-f5xc-required-for: ["minimum_config", "create"]

metadata.uid:
  type: string
  x-f5xc-required-for: ["read"]
```
