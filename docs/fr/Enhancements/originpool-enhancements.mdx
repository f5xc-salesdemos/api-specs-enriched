Métadonnées d'enrichissement pour les schémas liés à origin_pool. Voir [Extensions OpenAPI](../development/#openapi-extensions) pour les définitions des extensions.

## Schémas enrichis

### Modèle

Tous les schémas correspondant à `origin_pool.*SpecType` reçoivent des enrichissements :

- `origin_poolCreateSpecType`
- `origin_poolReplaceSpecType`
- `origin_poolGetSpecType`

## Champs obligatoires

| Champ | Obligatoire pour | Notes |
|-------|------------------|-------|
| `metadata.name` | Toutes les opérations | 1-63 caractères, alphanumérique minuscule, modèle : `^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |
| `metadata.namespace` | Toutes les opérations | Identifiant de l'espace de noms |
| `spec.origin_servers` | Création | Minimum 1 élément |
| `spec.port` | Création | 1-65535, pas de valeur par défaut serveur |

## Configuration minimale viable

```json
{
  "metadata": {
    "name": "my-origin-pool",
    "namespace": "default"
  },
  "spec": {
    "origin_servers": [
      {
        "public_name": {
          "dns_name": "backend.example.com"
        }
      }
    ],
    "port": 443
  }
}
```

## Valeurs par défaut appliquées par le serveur

Les champs marqués avec `x-f5xc-server-default: true` ont leur valeur `default` appliquée par le serveur API F5 XC lorsqu'ils sont omis des requêtes.

### Champs de niveau supérieur

| Champ | Valeur par défaut | Type | Description |
|-------|-------------------|------|-------------|
| `no_tls` | `{}` | objet | TLS vers l'origine désactivé |
| `healthcheck` | `[]` | tableau | Aucune vérification d'état configurée |
| `loadbalancer_algorithm` | `ROUND_ROBIN` | enum | Répartition de charge en tourniquet |
| `endpoint_selection` | `DISTRIBUTED` | enum | Utiliser tous les points de terminaison (locaux + distants) |

### Valeurs par défaut de advanced_options

Lorsque `advanced_options` n'est pas spécifié, le serveur se comporte comme si ces valeurs étaient définies :

| Champ | Valeur par défaut | Type | Description |
|-------|-------------------|------|-------------|
| `connection_timeout` | `2000` | entier | Délai d'expiration de connexion en millisecondes |
| `http_idle_timeout` | `300000` | entier | Délai d'inactivité HTTP en millisecondes (5 min) |
| `same_as_endpoint_port` | `{}` | objet | La vérification d'état utilise le port du point de terminaison |
| `default_circuit_breaker` | `{}` | objet | Paramètres de disjoncteur par défaut |
| `disable_outlier_detection` | `{}` | objet | Détection des anomalies désactivée |
| `no_panic_threshold` | `{}` | objet | Pas de seuil de panique |
| `disable_subsets` | `{}` | objet | Répartition de charge par sous-ensembles désactivée |
| `auto_http_config` | `{}` | objet | Négociation automatique du protocole HTTP |
| `disable_proxy_protocol` | `{}` | objet | Protocole proxy désactivé |
| `disable_lb_source_ip_persistance` | `{}` | objet | Persistance IP source du répartiteur de charge désactivée |

### Valeurs par défaut des objets imbriqués

| Chemin | Valeur par défaut | Description |
|--------|-------------------|-------------|
| `origin_servers[].labels` | `{}` | Objet de labels vide |
| `origin_servers[].public_name.refresh_interval` | `0` | Utiliser l'intervalle de rafraîchissement DNS par défaut du système |

## Différences entre l'interface utilisateur et les valeurs par défaut du serveur

L'interface web F5 XC présélectionne des valeurs différentes de celles que l'API applique lorsque les champs sont omis.

| Champ | Présélection IU | Appliqué par le serveur | Note |
|-------|-----------------|-------------------------|------|
| `loadbalancer_algorithm` | `LB_OVERRIDE` | `ROUND_ROBIN` | Les ressources créées via l'IU et celles créées via l'API diffèrent lorsque le champ est omis |

## Valeurs d'énumération

### loadbalancer_algorithm

| Valeur | Description | Notes |
|--------|-------------|-------|
| `ROUND_ROBIN` | Chaque point de terminaison sain est sélectionné en ordre de tourniquet | Valeur par défaut du serveur |
| `LEAST_REQUEST` | Le point de terminaison avec le moins de requêtes actives est sélectionné | |
| `RING_HASH` | Hachage cohérent utilisant un anneau de hachage des noms de points de terminaison | |
| `RANDOM` | Sélection aléatoire d'un point de terminaison sain | |
| `LB_OVERRIDE` | Politique de hachage héritée du répartiteur de charge parent | Valeur par défaut de l'IU |

### endpoint_selection

| Valeur | Description | Notes |
|--------|-------------|-------|
| `DISTRIBUTED` | Considérer les points de terminaison distants et locaux | Valeur par défaut du serveur |
| `LOCAL_ONLY` | Seuls les points de terminaison locaux sont utilisés | |
| `LOCAL_PREFERRED` | Préférer les locaux, basculer vers les distants si indisponible | |

## Groupes de champs OneOf

Groupes de champs mutuellement exclusifs. Un seul champ de chaque groupe peut être spécifié :

| Groupe | Champs | Valeur par défaut du serveur |
|--------|--------|------------------------------|
| Configuration du port | `port`, `automatic_port`, `lb_port` | `port` (explicite) |
| TLS vers l'origine | `no_tls`, `use_tls` | `no_tls` |
| Port de vérification d'état | `same_as_endpoint_port`, `health_check_port` | `same_as_endpoint_port` |
| Disjoncteur | `default_circuit_breaker`, `disable_circuit_breaker`, `circuit_breaker` | `default_circuit_breaker` |
| Détection des anomalies | `disable_outlier_detection`, `outlier_detection` | `disable_outlier_detection` |
| Seuil de panique | `no_panic_threshold`, `panic_threshold` | `no_panic_threshold` |
| Sous-ensembles de répartition | `disable_subsets`, `enable_subsets` | `disable_subsets` |
| Protocole HTTP | `auto_http_config`, `http1_config`, `http2_options` | `auto_http_config` |
| Protocole proxy | `disable_proxy_protocol`, `proxy_protocol_v1`, `proxy_protocol_v2` | `disable_proxy_protocol` |
| IP source du répartiteur | `disable_lb_source_ip_persistance`, `enable_lb_source_ip_persistance` | `disable_lb_source_ip_persistance` |
| Pool de connexions | `enable_conn_pool_reuse`, `disable_conn_pool_reuse` | `enable_conn_pool_reuse` |

## Contraintes des champs

| Champ | Type | Contrainte |
|-------|------|------------|
| `spec.port` | entier | 1-65535 |
| `spec.advanced_options.connection_timeout` | entier | 0-1 800 000 ms |
| `spec.advanced_options.http_idle_timeout` | entier | 0-600 000 ms |
| `spec.advanced_options.panic_threshold` | entier | 0-100 (pourcentage) |
| `metadata.name` | chaîne | 1-63 caractères, modèle : `^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |

## Types de serveurs d'origine

Les serveurs d'origine peuvent être spécifiés à l'aide de différentes variantes de type. Le type détermine quels champs supplémentaires sont requis.

### Types implémentés

| Type | Description | Champs obligatoires | Statut |
|------|-------------|---------------------|--------|
| `public_name` | Serveur d'origine avec nom DNS public | `dns_name` | ✅ Complet |
| `public_ip` | Serveur d'origine avec adresse IP publique | `ip` (IPv4) | ✅ Complet |

### Exemple : public_name

```json
{
  "origin_servers": [
    {
      "public_name": {
        "dns_name": "backend.example.com"
      }
    }
  ]
}
```

### Exemple : public_ip

```json
{
  "origin_servers": [
    {
      "public_ip": {
        "ip": "8.8.8.8"
      }
    }
  ]
}
```

## À FAIRE : Types de serveurs d'origine dépendants du site

Les types de serveurs d'origine suivants nécessitent le développement de la logique de ressource « site » avant leur implémentation :

| Type | Description | Champs obligatoires | Statut |
|------|-------------|---------------------|--------|
| `private_ip` | Serveur d'origine avec IP privée/publique et informations de site | `ip`, `site_locator`, `network_choice`, `snat_pool` | 🔲 En attente de la logique site |
| `private_name` | Serveur d'origine avec nom DNS et informations de site | `dns_name`, `site_locator`, `network_choice`, `snat_pool` | 🔲 En attente de la logique site |
| `k8s_service` | Service Kubernetes avec informations de site | `service_name`, `site_locator`, `network_choice` | 🔲 En attente de la logique site |
| `consul_service` | Service HashiCorp Consul avec informations de site | `service_name`, `site_locator`, `network_choice` | 🔲 En attente de la logique site |

### Prérequis d'implémentation

Avant que ces types puissent être entièrement documentés :

1. **Découverte de la ressource site** - Documenter le schéma de la ressource `site` et les modèles site_locator
2. **Énumération Network Choice** - Documenter les valeurs contraintes de `network_choice` (site_local_inside, site_local_outside, etc.)
3. **Options du pool SNAT** - Documenter les modèles de configuration du pool SNAT
4. **Références inter-ressources** - Définir comment origin_pool référence les ressources site

### Suivi

- Espace réservé de configuration : `config/discovered_defaults.yaml` → `origin_pool.origin_server_types.enums`
- Sprint associé : Enrichissements du schéma de ressource site
