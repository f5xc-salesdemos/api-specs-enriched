Connexions multi-cloud avec appairage de passerelles et configuration de chemins réseau. Intégration du coffre-fort d'identifiants et énumération des sous-réseaux.

- **Catégorie** : Infrastructure
- **Complexité** : modérée
- **Chemins** : 31 | **Schémas** : 208
- **Niveau** : Standard
- **Domaines associés** : `sites`, `customer_edge`

## Cas d'utilisation

- Se connecter aux fournisseurs cloud (AWS, Azure, GCP)
- Gérer les identifiants cloud et l'authentification
- Configurer la connectivité cloud et le provisionnement élastique
- Lier et gérer les régions cloud

## Ressources principales

- **aws_vpc_site** : Déploiement de site AWS VPC avec provisionnement de nœud edge
- **azure_vnet_site** : Déploiement de site Azure VNet avec provisionnement de nœud edge
- **gcp_vpc_site** : Déploiement de site Google Cloud VPC avec provisionnement de nœud edge
- **cloud_credentials** : Identifiants d'authentification du fournisseur cloud pour le déploiement de sites
