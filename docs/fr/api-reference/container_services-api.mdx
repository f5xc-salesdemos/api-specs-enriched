Orchestration de pods sans la complexité d'un cluster complet. Exécution sur sites edge, application de quotas et profils de calcul standardisés pour les applications distribuées.

- **Catégorie** : Infrastructure
- **Complexité** : modérée
- **Chemins** : 13 | **Schémas** : 159
- **Niveau** : Avancé
- **Domaines associés** : `managed_kubernetes`, `sites`, `service_mesh`

## Cas d'utilisation

- Déployer des espaces de noms XCCS (Services de Conteneurs) pour les charges de travail multi-locataires
- Gérer les charges de travail conteneurisées avec une orchestration simplifiée
- Configurer des déploiements de conteneurs edge distribués
- Exécuter des applications conteneurisées sans la complexité complète de K8s

## Ressources principales

- **virtual_k8s** : Espace de noms Kubernetes virtuel pour les charges de travail conteneurisées
- **workload** : Configuration de déploiement des charges de travail conteneurisées
- **pod_security_policy** : Politique de sécurité des pods pour les contraintes d'exécution des conteneurs
