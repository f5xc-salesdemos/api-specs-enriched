Politiques de données sensibles avec règles de classification personnalisées. Configuration des régions LMA et contrôles de conformité basés sur la géolocalisation.

- **Catégorie** : Sécurité
- **Complexité** : simple
- **Chemins** : 11 | **Schémas** : 65
- **Niveau** : Avancé
- **Domaines associés** : `blindfold`, `client_side_defense`

## Cas d'utilisation

- Configurer les politiques de détection des données sensibles
- Définir des types de données personnalisés pour la classification des données personnelles (PII)
- Gérer les configurations des régions LMA
- Intégrer les géo-configurations pour la conformité

## Ressources principales

- **sensitive_data_policy** : Politique de données sensibles pour la protection des données personnelles (PII)
- **data_classification** : Classification des données pour la catégorisation du contenu
