Liaisons de rôles Kubernetes et politiques d'admission. Intégration de registres pour les charges de travail EKS, AKS et GKE.

- **Catégorie** : Infrastructure
- **Complexité** : modérée
- **Chemins** : 20 | **Schémas** : 95
- **Niveau** : Avancé
- **Domaines associés** : `container_services`, `sites`, `service_mesh`

## Cas d'utilisation

- Gérer le RBAC et la sécurité des clusters XCKS (Kubernetes Managé)
- Configurer les politiques de sécurité des pods et les contrôleurs d'admission
- Gérer les registres de conteneurs pour les déploiements d'entreprise
- S'intégrer avec des clusters Kubernetes externes (EKS, AKS, GKE)

## Ressources principales

- **mk8s_cluster** : Configuration de cluster Kubernetes managé
- **k8s_cluster_role** : Configuration de rôle RBAC de cluster Kubernetes
- **container_registry** : Registre d'images de conteneurs pour les images de charges de travail
