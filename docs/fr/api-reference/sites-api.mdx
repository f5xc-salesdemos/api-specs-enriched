Intégration AWS, Azure, GCP VPC avec des passerelles de transit. Sélection basée sur les labels pour l'application des politiques à travers les régions.

- **Catégorie** : Infrastructure
- **Complexité** : avancé
- **Chemins** : 133 | **Schémas** : 1015
- **Niveau** : Standard
- **Domaines associés** : `cloud_infrastructure`, `customer_edge`, `managed_kubernetes`

## Cas d'utilisation

- Déployer F5 XC à travers les fournisseurs cloud (AWS, Azure, GCP)
- Gérer les déploiements de sites XCKS (Managed Kubernetes) (anciennement AppStack)
- Déployer des sites Secure Mesh pour les déploiements edge orientés réseau
- Intégrer des clusters Kubernetes externes en tant que Customer Edge
- Configurer les sites AWS VPC, Azure VNet et GCP VPC
- Gérer les sites virtuels et les politiques de sites

## Ressources principales

- **site** : Emplacement de déploiement physique ou cloud pour les services edge
- **virtual_site** : Regroupement logique de sites utilisant des sélecteurs de labels
- **site_mesh_group** : Configuration de connectivité maillée entre plusieurs sites
