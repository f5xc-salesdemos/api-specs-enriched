हेल्थचेक-संबंधित स्कीमाओं के लिए संवर्धन मेटाडेटा। एक्सटेंशन परिभाषाओं के लिए [OpenAPI एक्सटेंशन](../development/#openapi-extensions) देखें।

## संवर्धित स्कीमाएं

### पैटर्न

`healthcheck.*SpecType` से मेल खाने वाली सभी स्कीमाओं को संवर्धन प्राप्त होता है:

- `healthcheckCreateSpecType`
- `healthcheckReplaceSpecType`
- `healthcheckGetSpecType`

इसके अतिरिक्त, नेस्टेड स्कीमा `healthcheckHttpHealthCheck` में HTTP हेल्थ चेक कॉन्फ़िगरेशन के लिए संवर्धन शामिल हैं।

## सर्वर-द्वारा लागू डिफ़ॉल्ट

`x-f5xc-server-default: true` से चिह्नित फ़ील्ड का `default` मान F5 XC API सर्वर द्वारा तब लागू किया जाता है जब अनुरोधों से छोड़ दिया जाता है।

### शीर्ष-स्तरीय फ़ील्ड

| फ़ील्ड | डिफ़ॉल्ट मान | प्रकार | विवरण |
|-------|---------------|------|-------------|
| `jitter` | `0` | integer | समय यादृच्छिकीकरण के लिए पूर्ण जिटर मान |
| `jitter_percent` | `0` | integer | समय यादृच्छिकीकरण के लिए प्रतिशत-आधारित जिटर |

### healthcheckHttpHealthCheck स्कीमा

| फ़ील्ड | डिफ़ॉल्ट मान | प्रकार | विवरण |
|-------|---------------|------|-------------|
| `use_origin_server_name` | `{}` | object | Host हेडर के लिए ओरिजिन सर्वर नाम |
| `headers` | `{}` | object | कस्टम हेडर |
| `request_headers_to_remove` | `[]` | array | अनुरोधों से हटाए जाने वाले हेडर |
| `use_http2` | `false` | boolean | HTTP/2 सपोर्ट |
| `expected_status_codes` | `[]` | array | स्वीकृत स्टेटस कोड (रिक्त = 200-299) |

## अनुशंसित मान

`x-f5xc-recommended-value` से चिह्नित फ़ील्ड उन मानों को इंगित करते हैं जो नए संसाधन बनाते समय F5 XC वेब कंसोल पूर्व-भरता है।

### शीर्ष-स्तरीय फ़ील्ड

| फ़ील्ड | अनुशंसित मान | प्रकार | विवरण |
|-------|-------------------|------|-------------|
| `timeout` | `3` | integer | हेल्थ चेक टाइमआउट सेकंड में |
| `interval` | `15` | integer | हेल्थ चेक के बीच अंतराल सेकंड में |
| `unhealthy_threshold` | `1` | integer | अस्वस्थ चिह्नित करने से पहले लगातार विफलताएं |
| `healthy_threshold` | `3` | integer | स्वस्थ चिह्नित करने से पहले लगातार सफलताएं |
| `jitter_percent` | `30` | integer | प्रोडक्शन उपयोग के लिए जिटर प्रतिशत |

### healthcheckHttpHealthCheck स्कीमा

| फ़ील्ड | अनुशंसित मान | प्रकार | विवरण |
|-------|-------------------|------|-------------|
| `path` | `"/"` | string | हेल्थ चेक एंडपॉइंट पथ |
| `use_http2` | `false` | boolean | HTTP/2 सपोर्ट सेटिंग |
| `expected_status_codes` | `["200"]` | array | स्वस्थ ओरिजिन इंगित करने वाले स्टेटस कोड |
| `use_origin_server_name` | `{}` | object | Host हेडर के लिए ओरिजिन सर्वर नाम |

## OneOf वैरिएंट अनुशंसाएं

परस्पर अनन्य फ़ील्ड समूहों (OneOf) वाली स्कीमाओं में `x-f5xc-recommended-oneof-variant` शामिल होता है जो यह इंगित करता है कि कौन सा वैरिएंट सबसे अधिक उपयोग किया जाता है। यह मेटाडेटा F5 XC कंसोल डिफ़ॉल्ट और सामान्य उपयोग पैटर्न के आधार पर, एकाधिक विकल्प उपलब्ध होने पर सामान्य चयन की पहचान करता है।

### शीर्ष-स्तरीय OneOf समूह

| स्कीमा | OneOf समूह | अनुशंसित वैरिएंट | विवरण |
|--------|-------------|---------------------|-------------|
| `healthcheckCreateSpecType` | `health_check` | `http_health_check` | HTTP हेल्थ चेक प्रकार |
| `healthcheckReplaceSpecType` | `health_check` | `http_health_check` | HTTP हेल्थ चेक प्रकार |

### healthcheckHttpHealthCheck OneOf समूह

| स्कीमा | OneOf समूह | अनुशंसित वैरिएंट | विवरण |
|--------|-------------|---------------------|-------------|
| `healthcheckHttpHealthCheck` | `host_header_choice` | `use_origin_server_name` | हेल्थ चेक अनुरोधों के लिए Host हेडर |

#### Host हेडर चयन

`host_header_choice` OneOf समूह यह नियंत्रित करता है कि हेल्थ चेक HTTP अनुरोधों में Host हेडर कैसे निर्दिष्ट किया जाता है।

| वैरिएंट | प्रकार | विवरण |
|---------|------|-------------|
| `use_origin_server_name` | `object` (रिक्त) | ओरिजिन सर्वर नाम को Host हेडर के रूप में उपयोग करें। यह F5 XC कंसोल में डिफ़ॉल्ट चयन है। |
| `host_header` | `string` | हेल्थ चेक अनुरोधों के लिए एक कस्टम Host हेडर मान निर्दिष्ट करें। |

**API स्कीमा संदर्भ**: `x-ves-oneof-field-host_header_choice: ["host_header", "use_origin_server_name"]`

## OpenAPI एक्सटेंशन संदर्भ

ये वेंडर एक्सटेंशन F5 XC-विशिष्ट डिफ़ॉल्ट व्यवहार को व्यक्त करने के लिए मानक OpenAPI स्कीमा में जोड़े जाते हैं।

### x-f5xc-server-default

**प्रकार**: `boolean`

जब `true` होता है, तो यह इंगित करता है कि संलग्न `default` मान F5 XC API सर्वर द्वारा लागू किया जाता है। इस एक्सटेंशन वाले फ़ील्ड को API अनुरोधों से सुरक्षित रूप से छोड़ा जा सकता है—सर्वर स्वचालित रूप से डिफ़ॉल्ट लागू करता है।

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

### x-f5xc-recommended-value

**प्रकार**: `any` (फ़ील्ड प्रकार से मेल खाता है)

एक ऐसा मान निर्दिष्ट करता है जिसे F5 XC वेब कंसोल पूर्व-भरे डिफ़ॉल्ट के रूप में उपयोग करता है। यह मान सर्वर-लागू नहीं है लेकिन कंसोल के माध्यम से बनाए गए नए संसाधनों के लिए सामान्य प्रारंभिक कॉन्फ़िगरेशन का प्रतिनिधित्व करता है।

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

### x-f5xc-recommended-oneof-variant

**प्रकार**: `object` (समूह नाम से वैरिएंट नाम का मैप)

परस्पर अनन्य फ़ील्ड समूहों वाली स्कीमाओं के लिए, यह पहचानता है कि कौन सा वैरिएंट डिफ़ॉल्ट या सबसे सामान्य विकल्प है। कुंजी OneOf समूह नाम है और मान अनुशंसित वैरिएंट फ़ील्ड नाम है।

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```
## डेटा एक्सेस

### OpenAPI विनिर्देश

| फ़ाइल | सामग्री |
|------|---------|
| `docs/specifications/api/virtual.json` | `healthcheckHttpHealthCheck`, `healthcheckCreateSpecType`, `healthcheckReplaceSpecType`, `healthcheckGetSpecType` |
| `docs/specifications/api/openapi.json` | सभी स्कीमाओं के साथ मर्ज किया गया विनिर्देश |

### validation.json संरचना

```text
defaults.resources.healthcheck
├── server_applied      # x-f5xc-server-default: true वाले फ़ील्ड
├── recommended         # x-f5xc-recommended-value वाले फ़ील्ड
├── oneof_recommended   # OneOf वैरिएंट अनुशंसाएं
└── nested_recommended  # नेस्टेड स्कीमा अनुशंसित मान
```
