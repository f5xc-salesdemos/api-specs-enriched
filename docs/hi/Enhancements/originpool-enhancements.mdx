origin_pool-संबंधित स्कीमा के लिए एनरिचमेंट मेटाडेटा। एक्सटेंशन परिभाषाओं के लिए [OpenAPI एक्सटेंशन](../development/#openapi-extensions) देखें।

## एनरिच्ड स्कीमा

### पैटर्न

`origin_pool.*SpecType` से मिलान करने वाली सभी स्कीमा को एनरिचमेंट प्राप्त होता है:

- `origin_poolCreateSpecType`
- `origin_poolReplaceSpecType`
- `origin_poolGetSpecType`

## आवश्यक फ़ील्ड

| फ़ील्ड | इसके लिए आवश्यक | नोट्स |
|-------|--------------|-------|
| `metadata.name` | सभी ऑपरेशन | 1-63 कैरेक्टर, लोअरकेस अल्फ़ान्यूमेरिक, पैटर्न: `^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |
| `metadata.namespace` | सभी ऑपरेशन | नेमस्पेस आइडेंटिफ़ायर |
| `spec.origin_servers` | Create | न्यूनतम 1 आइटम |
| `spec.port` | Create | 1-65535, कोई सर्वर डिफ़ॉल्ट नहीं |

## न्यूनतम व्यवहार्य कॉन्फ़िगरेशन

```json
{
  "metadata": {
    "name": "my-origin-pool",
    "namespace": "default"
  },
  "spec": {
    "origin_servers": [
      {
        "public_name": {
          "dns_name": "backend.example.com"
        }
      }
    ],
    "port": 443
  }
}
```

## सर्वर-लागू डिफ़ॉल्ट

`x-f5xc-server-default: true` से चिह्नित फ़ील्ड का `default` मान F5 XC API सर्वर द्वारा तब लागू किया जाता है जब उन्हें अनुरोधों से छोड़ दिया जाता है।

### शीर्ष-स्तरीय फ़ील्ड

| फ़ील्ड | डिफ़ॉल्ट मान | प्रकार | विवरण |
|-------|---------------|------|-------------|
| `no_tls` | `{}` | object | ऑरिजिन के लिए TLS अक्षम |
| `healthcheck` | `[]` | array | कोई हेल्थ चेक कॉन्फ़िगर नहीं |
| `loadbalancer_algorithm` | `ROUND_ROBIN` | enum | राउंड-रॉबिन लोड बैलेंसिंग |
| `endpoint_selection` | `DISTRIBUTED` | enum | सभी एंडपॉइंट का उपयोग (लोकल + रिमोट) |

### advanced_options डिफ़ॉल्ट

जब `advanced_options` निर्दिष्ट नहीं होता, तो सर्वर ऐसे व्यवहार करता है जैसे ये मान सेट किए गए हों:

| फ़ील्ड | डिफ़ॉल्ट मान | प्रकार | विवरण |
|-------|---------------|------|-------------|
| `connection_timeout` | `2000` | integer | कनेक्शन टाइमआउट मिलीसेकंड में |
| `http_idle_timeout` | `300000` | integer | HTTP आइडल टाइमआउट मिलीसेकंड में (5 मिनट) |
| `same_as_endpoint_port` | `{}` | object | हेल्थ चेक एंडपॉइंट पोर्ट का उपयोग करता है |
| `default_circuit_breaker` | `{}` | object | डिफ़ॉल्ट सर्किट ब्रेकर सेटिंग्स |
| `disable_outlier_detection` | `{}` | object | आउटलायर डिटेक्शन अक्षम |
| `no_panic_threshold` | `{}` | object | कोई पैनिक थ्रेशोल्ड नहीं |
| `disable_subsets` | `{}` | object | सबसेट लोड बैलेंसिंग अक्षम |
| `auto_http_config` | `{}` | object | स्वचालित HTTP प्रोटोकॉल नेगोशिएशन |
| `disable_proxy_protocol` | `{}` | object | प्रॉक्सी प्रोटोकॉल अक्षम |
| `disable_lb_source_ip_persistance` | `{}` | object | LB सोर्स IP पर्सिस्टेंस अक्षम |

### नेस्टेड ऑब्जेक्ट डिफ़ॉल्ट

| पथ | डिफ़ॉल्ट मान | विवरण |
|------|---------------|-------------|
| `origin_servers[].labels` | `{}` | खाली लेबल ऑब्जेक्ट |
| `origin_servers[].public_name.refresh_interval` | `0` | सिस्टम डिफ़ॉल्ट DNS रिफ़्रेश का उपयोग |

## UI बनाम सर्वर डिफ़ॉल्ट विसंगतियाँ

F5 XC वेब UI उन मानों से भिन्न मान पूर्व-चयनित करता है जो API तब लागू करता है जब फ़ील्ड छोड़ दी जाती हैं।

| फ़ील्ड | UI पूर्व-चयनित | सर्वर लागू | नोट |
|-------|-----------------|----------------|------|
| `loadbalancer_algorithm` | `LB_OVERRIDE` | `ROUND_ROBIN` | जब फ़ील्ड छोड़ दी जाती है तो UI-निर्मित और API-निर्मित संसाधन भिन्न होते हैं |

## Enum मान

### loadbalancer_algorithm

| मान | विवरण | नोट्स |
|-------|-------------|-------|
| `ROUND_ROBIN` | प्रत्येक स्वस्थ एंडपॉइंट राउंड-रॉबिन क्रम में चयनित | सर्वर डिफ़ॉल्ट |
| `LEAST_REQUEST` | सबसे कम सक्रिय अनुरोधों वाला एंडपॉइंट चयनित | |
| `RING_HASH` | एंडपॉइंट नामों के रिंग हैश का उपयोग करके कंसिस्टेंट हैशिंग | |
| `RANDOM` | रैंडम स्वस्थ एंडपॉइंट चयन | |
| `LB_OVERRIDE` | हैश पॉलिसी पैरेंट लोड बैलेंसर से इनहेरिट | UI डिफ़ॉल्ट |

### endpoint_selection

| मान | विवरण | नोट्स |
|-------|-------------|-------|
| `DISTRIBUTED` | रिमोट और लोकल दोनों एंडपॉइंट पर विचार | सर्वर डिफ़ॉल्ट |
| `LOCAL_ONLY` | केवल लोकल एंडपॉइंट उपयोग | |
| `LOCAL_PREFERRED` | लोकल को प्राथमिकता, अनुपलब्ध होने पर रिमोट पर फ़ॉलबैक | |

## OneOf फ़ील्ड समूह

परस्पर अनन्य फ़ील्ड समूह। प्रत्येक समूह से केवल एक फ़ील्ड निर्दिष्ट की जा सकती है:

| समूह | फ़ील्ड | सर्वर डिफ़ॉल्ट |
|-------|--------|----------------|
| पोर्ट कॉन्फ़िगरेशन | `port`, `automatic_port`, `lb_port` | `port` (स्पष्ट) |
| ऑरिजिन के लिए TLS | `no_tls`, `use_tls` | `no_tls` |
| हेल्थ चेक पोर्ट | `same_as_endpoint_port`, `health_check_port` | `same_as_endpoint_port` |
| सर्किट ब्रेकर | `default_circuit_breaker`, `disable_circuit_breaker`, `circuit_breaker` | `default_circuit_breaker` |
| आउटलायर डिटेक्शन | `disable_outlier_detection`, `outlier_detection` | `disable_outlier_detection` |
| पैनिक थ्रेशोल्ड | `no_panic_threshold`, `panic_threshold` | `no_panic_threshold` |
| सबसेट LB | `disable_subsets`, `enable_subsets` | `disable_subsets` |
| HTTP प्रोटोकॉल | `auto_http_config`, `http1_config`, `http2_options` | `auto_http_config` |
| प्रॉक्सी प्रोटोकॉल | `disable_proxy_protocol`, `proxy_protocol_v1`, `proxy_protocol_v2` | `disable_proxy_protocol` |
| LB सोर्स IP | `disable_lb_source_ip_persistance`, `enable_lb_source_ip_persistance` | `disable_lb_source_ip_persistance` |
| कनेक्शन पूल | `enable_conn_pool_reuse`, `disable_conn_pool_reuse` | `enable_conn_pool_reuse` |

## फ़ील्ड बाधाएँ

| फ़ील्ड | प्रकार | बाधा |
|-------|------|------------|
| `spec.port` | integer | 1-65535 |

| `spec.advanced_options.http_idle_timeout` | integer | 0-600,000 ms |

## ऑरिजिन सर्वर प्रकार

ऑरिजिन सर्वर विभिन्न प्रकार के वेरिएंट का उपयोग करके निर्दिष्ट किए जा सकते हैं। प्रकार निर्धारित करता है कि कौन से अतिरिक्त फ़ील्ड आवश्यक हैं।

### कार्यान्वित प्रकार

| प्रकार | विवरण | आवश्यक फ़ील्ड | स्थिति |
|------|-------------|-----------------|--------|
| `public_name` | सार्वजनिक DNS नाम वाला ऑरिजिन सर्वर | `dns_name` | ✅ पूर्ण |
| `public_ip` | सार्वजनिक IP पते वाला ऑरिजिन सर्वर | `ip` (IPv4) | ✅ पूर्ण |

### उदाहरण: public_name

```json
{
  "origin_servers": [
    {
      "public_name": {
        "dns_name": "backend.example.com"
      }
    }
  ]
}
```

### उदाहरण: public_ip

```json
{
  "origin_servers": [
    {
      "public_ip": {
        "ip": "8.8.8.8"
      }
    }
  ]
}
```

## TODO: साइट-निर्भर ऑरिजिन सर्वर प्रकार

निम्नलिखित ऑरिजिन सर्वर प्रकारों को कार्यान्वयन से पहले "site" संसाधन लॉजिक विकास की आवश्यकता है:

| प्रकार | विवरण | आवश्यक फ़ील्ड | स्थिति |
|------|-------------|-----------------|--------|
| `private_ip` | प्राइवेट/पब्लिक IP और साइट जानकारी वाला ऑरिजिन सर्वर | `ip`, `site_locator`, `network_choice`, `snat_pool` | 🔲 साइट लॉजिक लंबित |
| `private_name` | DNS नाम और साइट जानकारी वाला ऑरिजिन सर्वर | `dns_name`, `site_locator`, `network_choice`, `snat_pool` | 🔲 साइट लॉजिक लंबित |
| `k8s_service` | साइट जानकारी के साथ Kubernetes सेवा | `service_name`, `site_locator`, `network_choice` | 🔲 साइट लॉजिक लंबित |
| `consul_service` | साइट जानकारी के साथ HashiCorp Consul सेवा | `service_name`, `site_locator`, `network_choice` | 🔲 साइट लॉजिक लंबित |

### कार्यान्वयन आवश्यकताएँ

इन प्रकारों को पूरी तरह से दस्तावेजित करने से पहले:

1. **साइट संसाधन डिस्कवरी** - `site` संसाधन स्कीमा और site_locator पैटर्न का दस्तावेजीकरण
2. **नेटवर्क चॉइस Enum** - `network_choice` बाधित मानों का दस्तावेजीकरण (site_local_inside, site_local_outside, आदि)
3. **SNAT पूल विकल्प** - SNAT पूल कॉन्फ़िगरेशन पैटर्न का दस्तावेजीकरण
4. **क्रॉस-संसाधन संदर्भ** - परिभाषित करें कि origin_pool साइट संसाधनों को कैसे संदर्भित करता है
