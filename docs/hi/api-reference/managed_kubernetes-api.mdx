Kubernetes रोल बाइंडिंग और एडमिशन नीतियाँ। EKS, AKS, और GKE वर्कलोड के लिए रजिस्ट्री इंटीग्रेशन।

- **श्रेणी**: इंफ्रास्ट्रक्चर
- **जटिलता**: मध्यम
- **पाथ**: 20 | **स्कीमा**: 95
- **स्तर**: उन्नत
- **संबंधित डोमेन**: `container_services`, `sites`, `service_mesh`

## उपयोग के मामले

- XCKS (प्रबंधित Kubernetes) क्लस्टर RBAC और सुरक्षा प्रबंधित करें
- पॉड सुरक्षा नीतियाँ और एडमिशन कंट्रोलर कॉन्फ़िगर करें
- एंटरप्राइज़ डिप्लॉयमेंट के लिए कंटेनर रजिस्ट्रियाँ प्रबंधित करें
- बाहरी Kubernetes क्लस्टर (EKS, AKS, GKE) के साथ इंटीग्रेट करें

## प्राथमिक संसाधन

- **mk8s_cluster**: प्रबंधित Kubernetes क्लस्टर कॉन्फ़िगरेशन
- **k8s_cluster_role**: Kubernetes क्लस्टर RBAC रोल कॉन्फ़िगरेशन
- **container_registry**: वर्कलोड इमेज के लिए कंटेनर इमेज रजिस्ट्री
