स्रोत, गंतव्य, या प्रोटोकॉल के आधार पर रूटिंग निर्णयों के साथ फ़ायरवॉल नियम। सेगमेंटेशन वर्कलोड को अलग करता है जबकि आउटबाउंड प्रॉक्सी एक्सेस को नियंत्रित करते हैं।

- **श्रेणी**: सुरक्षा
- **जटिलता**: उन्नत
- **पथ**: 62 | **स्कीमा**: 386
- **स्तर**: उन्नत
- **संबंधित डोमेन**: `virtual`, `api`, `network`

## उपयोग के मामले

- नेटवर्क फ़ायरवॉल और ACL नीतियाँ कॉन्फ़िगर करें
- NAT नीतियाँ और पोर्ट फ़ॉरवर्डिंग प्रबंधित करें
- नीति-आधारित रूटिंग कॉन्फ़िगर करें
- नेटवर्क सेगमेंट और नीतियाँ परिभाषित करें
- फ़ॉरवर्ड प्रॉक्सी नीतियाँ कॉन्फ़िगर करें

## प्राथमिक संसाधन

- **network_policy**: ट्रैफ़िक फ़िल्टरिंग के लिए नेटवर्क सुरक्षा नीति
- **forward_proxy_policy**: आउटबाउंड ट्रैफ़िक नियंत्रण के लिए फ़ॉरवर्ड प्रॉक्सी नीति
- **network_firewall**: लेयर 3/4 ट्रैफ़िक सुरक्षा के लिए नेटवर्क फ़ायरवॉल
