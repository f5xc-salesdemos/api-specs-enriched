ट्रांज़िट गेटवे के साथ AWS, Azure, GCP VPC एकीकरण। क्षेत्रों में पॉलिसी अनुप्रयोग के लिए लेबल-आधारित चयन।

- **श्रेणी**: इन्फ्रास्ट्रक्चर
- **जटिलता**: उन्नत
- **पाथ**: 133 | **स्कीमा**: 1015
- **टियर**: स्टैंडर्ड
- **संबंधित डोमेन**: `cloud_infrastructure`, `customer_edge`, `managed_kubernetes`

## उपयोग के मामले

- क्लाउड प्रदाताओं (AWS, Azure, GCP) में F5 XC डिप्लॉय करें
- XCKS (प्रबंधित Kubernetes) साइट डिप्लॉयमेंट प्रबंधित करें (पूर्व में AppStack)
- नेटवर्किंग-केंद्रित एज डिप्लॉयमेंट के लिए Secure Mesh साइट डिप्लॉय करें
- बाहरी Kubernetes क्लस्टर को Customer Edge के रूप में एकीकृत करें
- AWS VPC, Azure VNet, और GCP VPC साइट कॉन्फ़िगर करें
- वर्चुअल साइट और साइट पॉलिसी प्रबंधित करें

## प्राथमिक संसाधन

- **site**: एज सेवाओं के लिए भौतिक या क्लाउड डिप्लॉयमेंट स्थान
- **virtual_site**: लेबल सिलेक्टर का उपयोग करके साइटों का तार्किक समूहन
- **site_mesh_group**: कई साइटों के बीच मेश कनेक्टिविटी कॉन्फ़िगरेशन
