यह गाइड F5 XC API संवर्धन पाइपलाइन के लिए विकास कार्यप्रवाह का वर्णन करती है।

## त्वरित प्रारंभ

### पहली बार सेटअप

```bash
# Clone the repository
git clone https://github.com/f5xc-salesdemos/api-specs-enriched.git
cd api-specs-enriched

# Create virtual environment and install dependencies
make install

# Install pre-commit hooks
make pre-commit-install

# Download specs and run pipeline
make build

# Preview documentation locally
make serve
```

### दैनिक विकास

```bash
# Quick rebuild (uses cached specs)
make rebuild

# Run all quality checks
make pre-commit-run

# Preview documentation
make serve
```

## कार्यप्रवाह पैटर्न

### 1. डिस्कवरी कार्यप्रवाह

**पूर्वापेक्षाएँ**:
- F5 XC एनवायरनमेंट से VPN कनेक्शन
- वैध API क्रेडेंशियल

```bash
# Set credentials
export F5XC_API_URL="https://your-tenant.console.ves.volterra.io/api"
export F5XC_API_TOKEN="your-api-token"

# Run discovery
make discover

# View results
jq '.statistics' specs/discovered/session.json

# Commit for CI/CD
make push-discovery
```

**डिस्कवरी द्वारा कैप्चर किया जाता है**:
- वास्तविक आवश्यक/वैकल्पिक फ़ील्ड
- लाइव रिस्पॉन्स से Enum मान
- डिफ़ॉल्ट मान
- पैटर्न वैलिडेशन
- रिस्पॉन्स उदाहरण

### 2. रिलीज़ कार्यप्रवाह

रिलीज़ स्वचालित हैं:

1. दैनिक शेड्यूल (6 AM UTC) या मुख्य शाखा में पुश करने पर कार्यप्रवाह ट्रिगर होता है
2. ETag जाँच निर्धारित करती है कि F5 स्पेक्स बदले हैं या नहीं
3. पाइपलाइन स्पेक्स को प्रोसेस और समृद्ध करती है
4. संस्करण git टैग्स से परिकलित: `git describe --tags --abbrev=0`
5. डायरेक्ट कमिट + टैग बनाया जाता है (कोई संस्करण बम्प PR नहीं)
6. चेंजलॉग के साथ GitHub रिलीज़ बनाई जाती है
7. दस्तावेज़ीकरण GitHub Pages पर तैनात होता है

**संस्करण बम्प नियम**:

| स्थिति | बम्प प्रकार | उदाहरण |
|-----------|-----------|---------|
| कमिट में `[major]` | मेजर | 1.0.0 → 2.0.0 |
| कमिट में `BREAKING CHANGE` | मेजर | 1.0.0 → 2.0.0 |
| नया डोमेन स्पेक जोड़ा गया | माइनर | 1.0.0 → 1.1.0 |
| कोई अन्य परिवर्तन | पैच | 1.0.0 → 1.0.1 |

## डिस्कवरी गहन अध्ययन

### डिस्कवरी क्या है?

डिस्कवरी लाइव F5 XC API को खोजती है ताकि निम्नलिखित का पता लगाया जा सके:

- **अदस्तावेज़ीकृत बाधाएँ**: OpenAPI में अंकित नहीं किए गए आवश्यक फ़ील्ड
- **वास्तविक enum मान**: उत्पादन में देखे गए वास्तविक मान
- **डिफ़ॉल्ट व्यवहार**: फ़ील्ड छोड़े जाने पर सर्वर द्वारा लागू मान
- **रिस्पॉन्स पैटर्न**: वास्तविक डेटा आकार

### डिस्कवरी कॉन्फ़िगरेशन

```yaml
# config/discovery.yaml
discovery:
  exploration:
    namespaces:
      - "system"
      - "shared"
    methods:
      - "GET"
      - "OPTIONS"
    max_endpoints_per_run: 500

  schema_inference:
    sample_size: 3
    detect_patterns: true
    detect_constraints: true
```

### डिस्कवरी एक्सटेंशन

डिस्कवरी स्पेक्स में `x-discovered-*` एक्सटेंशन जोड़ती है:

```json
{
  "properties": {
    "name": {
      "type": "string",
      "x-discovered-required": true,
      "x-discovered-pattern": "^[a-z][a-z0-9-]*$",
      "x-discovered-examples": ["my-app", "prod-lb"]
    }
  }
}
```

## OpenAPI एक्सटेंशन

संवर्धन पाइपलाइन वैलिडेशन और डिफ़ॉल्ट मान मेटाडेटा एम्बेड करने के लिए वेंडर एक्सटेंशन का उपयोग करती है।

### एक्सटेंशन श्रेणियाँ

| एक्सटेंशन | अर्थ | निहितार्थ |
|-----------|---------|-------------|
| `x-f5xc-server-default` | F5 XC API सर्वर इस मान को तब लागू करता है जब फ़ील्ड छोड़ी जाती है | फ़ील्ड वैकल्पिक है; इसे छोड़ने पर दस्तावेज़ीकृत डिफ़ॉल्ट व्यवहार प्राप्त होता है |
| `x-f5xc-recommended-value` | F5 XC वेब कंसोल नए संसाधनों के लिए इस मान को पूर्व-भरित करता है | फ़ील्ड में कोई सर्वर डिफ़ॉल्ट नहीं है लेकिन यह मान सामान्य कॉन्फ़िगरेशन दर्शाता है |
| `x-f5xc-recommended-oneof-variant` | F5 XC कंसोल इस OneOf वेरिएंट को पूर्व-चयनित करता है | कई परस्पर अनन्य विकल्पों में से सामान्य चुनाव की पहचान करता है |
| `x-f5xc-conflicts-with` | उन अन्य प्रॉपर्टीज़ की सूची जिन्हें इस फ़ील्ड के साथ उपयोग नहीं किया जा सकता | प्रॉपर्टी एक OneOf समूह का हिस्सा है; परस्पर विरोधी प्रॉपर्टीज़ में से केवल एक निर्दिष्ट की जा सकती है |

### आवश्यक फ़ील्ड एक्सटेंशन

| एक्सटेंशन | स्रोत | उद्देश्य |
|-----------|--------|---------|
| `x-ves-required: "true"` | F5 XC मूल स्पेक | फ़ील्ड को गैर-शून्य/गैर-रिक्त मान की आवश्यकता है |
| `x-f5xc-required-for` | संवर्धन पाइपलाइन | संदर्भ-विशिष्ट आवश्यक स्थिति |

**x-f5xc-required-for संदर्भ**:
- `create`: संसाधन बनाते समय आवश्यक
- `update`: संसाधन अपडेट करते समय आवश्यक
- `minimum_config`: न्यूनतम व्यवहार्य कॉन्फ़िगरेशन के लिए आवश्यक

### डिफ़ॉल्ट मान और OneOf एक्सटेंशन

| एक्सटेंशन | उद्देश्य |
|-----------|---------|
| `x-f5xc-server-default: true` | सर्वर द्वारा लागू डिफ़ॉल्ट को चिह्नित करता है |
| `x-f5xc-recommended-value` | F5 XC कंसोल द्वारा पूर्व-भरित मान |
| `x-f5xc-recommended-oneof-variant` | अनुशंसित OneOf वेरिएंट |
| `x-f5xc-conflicts-with` | अन्य OneOf प्रॉपर्टीज़ के साथ परस्पर अनन्यता |

#### x-f5xc-server-default

**प्रकार**: `boolean`

जब `true` हो, तो यह इंगित करता है कि साथ दिया गया `default` मान F5 XC API सर्वर द्वारा लागू किया जाता है। इस एक्सटेंशन वाले फ़ील्ड को API अनुरोधों से सुरक्षित रूप से छोड़ा जा सकता है — सर्वर डिफ़ॉल्ट स्वचालित रूप से लागू करता है।

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

#### x-f5xc-recommended-value

**प्रकार**: `any` (फ़ील्ड प्रकार से मेल खाता है)

वह मान निर्दिष्ट करता है जिसे F5 XC वेब कंसोल पूर्व-भरित डिफ़ॉल्ट के रूप में उपयोग करता है। यह मान सर्वर द्वारा लागू नहीं किया जाता लेकिन कंसोल के माध्यम से बनाए गए नए संसाधनों के लिए सामान्य प्रारंभिक कॉन्फ़िगरेशन दर्शाता है।

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

#### x-f5xc-recommended-oneof-variant

**प्रकार**: `object` (समूह नाम से वेरिएंट नाम का मैप)

परस्पर अनन्य फ़ील्ड समूहों वाले स्कीमा के लिए, पहचानता है कि कौन सा वेरिएंट डिफ़ॉल्ट या सबसे सामान्य विकल्प है। की OneOf समूह का नाम है और मान अनुशंसित वेरिएंट फ़ील्ड नाम है।

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

#### x-f5xc-conflicts-with

**प्रकार**: स्ट्रिंग्स की `array`

**जोड़ा गया**: v3.2.0 (Issue #494)

OneOf समूह के सदस्यों के बीच परस्पर अनन्यता संबंधों की घोषणा करता है। `x-ves-oneof-field-*` एक्सटेंशन से स्वतः व्युत्पन्न। यह डाउनस्ट्रीम उपकरणों (Terraform, CLI, MCP) को रनटाइम की बजाय स्कीमा स्तर पर संघर्षों को वैलिडेट करने में सक्षम बनाता है।

```yaml
host_header:
  type: string
  x-f5xc-conflicts-with: ["use_origin_server_name"]

use_origin_server_name:
  type: object
  x-f5xc-conflicts-with: ["host_header"]
```

**उपयोग के मामले**:
- Terraform प्रोवाइडर वैलिडेशन नियम जेनरेट कर सकते हैं
- CLI उपकरण परस्पर विरोधी फ़ील्ड संयोजनों के बारे में चेतावनी दे सकते हैं
- AI असिस्टेंट सही OneOf कॉन्फ़िगरेशन जेनरेट कर सकते हैं
- IDE एक्सटेंशन संघर्ष-जागरूक ऑटोकम्पलीशन प्रदान कर सकते हैं

**स्रोत**: पाइपलाइन संवर्धन के दौरान F5 नेटिव `x-ves-oneof-field-*` एक्सटेंशन से स्वतः व्युत्पन्न।

### वैलिडेशन नियम निहितार्थ

संवर्धक आवश्यक स्थिति का अनुमान लगाने के लिए `x-ves-validation-rules` की जाँच करता है:

| नियम | निहितार्थ |
|------|-------------|
| `ves.io.schema.rules.message.required: "true"` | फ़ील्ड आवश्यक है |
| `ves.io.schema.rules.uint32.gte: N` | यदि N >= 1 और कोई सर्वर डिफ़ॉल्ट नहीं, तो फ़ील्ड आवश्यक है |
| `ves.io.schema.rules.repeated.min_items: N` | यदि N >= 1, तो array में कम से कम N आइटम आवश्यक हैं |
| `ves.io.schema.rules.string.min_bytes: N` | यदि N >= 1, तो string में कम से कम N bytes आवश्यक हैं |

### सर्वर-लागू डिफ़ॉल्ट वाले संसाधन

कुछ संसाधन रिक्त स्पेक्स स्वीकार करते हैं क्योंकि सर्वर डिफ़ॉल्ट लागू करता है:

| संसाधन | सर्वर-लागू डिफ़ॉल्ट |
|----------|------------------------|
| `app_firewall` | `monitoring: \{\}`, `default_detection_settings: \{\}` |
| `rate_limiter` | `limits: []`, `user_identification: []` |
| `api_definition` | `swagger_specs: []`, डिफ़ॉल्ट `api_groups` |
| `healthcheck` | `jitter: 0`, `jitter_percent: 0`, नेस्टेड http_health_check डिफ़ॉल्ट |

इन संसाधनों के लिए, `x-f5xc-required-for.create` `false` हो सकता है, भले ही `x-ves-required` `true` हो।

### डिफ़ॉल्ट मान पैटर्न

| पैटर्न | प्रकार | उदाहरण |
|---------|------|---------|
| `\{\}` | रिक्त ऑब्जेक्ट (विकल्प चयन) | `monitoring: \{\}` |
| `[]` | रिक्त array | `expected_status_codes: []` |
| `0` | संख्यात्मक | `jitter: 0` |
| `""` | स्ट्रिंग | `expected_response: ""` |
| `false` | बूलियन | `use_http2: false` |

### खोजे गए डिफ़ॉल्ट जोड़ना

1. API के माध्यम से न्यूनतम स्पेक के साथ F5 XC में संसाधन बनाएँ
2. सर्वर द्वारा लागू मान देखने के लिए संसाधन वापस पढ़ें
3. `config/discovered_defaults.yaml` में डिफ़ॉल्ट दस्तावेज़ीकृत करें
4. पाइपलाइन चलाएँ: `make pipeline`
5. इसके साथ सत्यापित करें: `jq '.components.schemas | to_entries[] | select(.key | contains("resource_name"))'`

आउटपुट फ़ाइलों में नहीं, बल्कि संवर्धन/नॉर्मलाइज़ेशन कॉन्फ़िग में समस्याएँ ठीक करें।

### संस्करण प्रणाली

संस्करण git टैग्स से व्युत्पन्न होता है (उदाहरण, `v2.0.38`), जिससे समवर्ती PRs पर मर्ज संघर्ष उत्पन्न करने वाली रेस कंडीशन समाप्त होती है।

```python
# Version is calculated dynamically from git tags
from scripts.utils.version_calculator import get_version_from_tags
version = get_version_from_tags()  # Returns "2.0.38"
```

## त्वरित संदर्भ

### Make टार्गेट

| टार्गेट | विवरण |
|--------|-------------|
| `make build` | पूर्ण बिल्ड (डाउनलोड + पाइपलाइन) |
| `make rebuild` | त्वरित रिबिल्ड (डाउनलोड छोड़ें) |
| `make download` | स्पेक्स डाउनलोड करें (ETag कैश्ड) |
| `make download-force` | जबरन डाउनलोड करें |
| `make pipeline` | संवर्धन पाइपलाइन चलाएँ |
| `make lint` | Spectral लिंटिंग |
| `make serve` | स्थानीय दस्तावेज़ीकरण सर्वर |
| `make discover` | API डिस्कवरी (VPN आवश्यक) |
| `make push-discovery` | डिस्कवरी डेटा कमिट करें |
| `make clean` | जेनरेट की गई फ़ाइलें हटाएँ |
| `make pre-commit-run` | सभी गुणवत्ता जाँचें चलाएँ |
