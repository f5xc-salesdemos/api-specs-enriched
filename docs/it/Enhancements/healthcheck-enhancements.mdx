Metadati di arricchimento per gli schemi relativi agli healthcheck. Consultare [Estensioni OpenAPI](../development/#openapi-extensions) per le definizioni delle estensioni.

## Schemi Arricchiti

### Pattern

Tutti gli schemi corrispondenti a `healthcheck.*SpecType` ricevono arricchimenti:

- `healthcheckCreateSpecType`
- `healthcheckReplaceSpecType`
- `healthcheckGetSpecType`

Inoltre, lo schema annidato `healthcheckHttpHealthCheck` contiene arricchimenti per la configurazione degli health check HTTP.

## Valori Predefiniti Applicati dal Server

I campi contrassegnati con `x-f5xc-server-default: true` hanno il loro valore `default` applicato dal server API F5 XC quando vengono omessi dalle richieste.

### Campi di Primo Livello

| Campo | Valore Predefinito | Tipo | Descrizione |
|-------|-------------------|------|-------------|
| `jitter` | `0` | integer | Valore assoluto di jitter per la randomizzazione temporale |
| `jitter_percent` | `0` | integer | Jitter basato su percentuale per la randomizzazione temporale |

### Schema healthcheckHttpHealthCheck

| Campo | Valore Predefinito | Tipo | Descrizione |
|-------|-------------------|------|-------------|
| `use_origin_server_name` | `{}` | object | Nome del server di origine per l'header Host |
| `headers` | `{}` | object | Header personalizzati |
| `request_headers_to_remove` | `[]` | array | Header da rimuovere dalle richieste |
| `use_http2` | `false` | boolean | Supporto HTTP/2 |
| `expected_status_codes` | `[]` | array | Codici di stato accettati (vuoto = 200-299) |

## Valori Raccomandati

I campi contrassegnati con `x-f5xc-recommended-value` indicano valori che la console web F5 XC pre-popola durante la creazione di nuove risorse.

### Campi di Primo Livello

| Campo | Valore Raccomandato | Tipo | Descrizione |
|-------|---------------------|------|-------------|
| `timeout` | `3` | integer | Timeout dell'health check in secondi |
| `interval` | `15` | integer | Intervallo tra gli health check in secondi |
| `unhealthy_threshold` | `1` | integer | Fallimenti consecutivi prima di contrassegnare come non integro |
| `healthy_threshold` | `3` | integer | Successi consecutivi prima di contrassegnare come integro |
| `jitter_percent` | `30` | integer | Percentuale di jitter per l'uso in produzione |

### Schema healthcheckHttpHealthCheck

| Campo | Valore Raccomandato | Tipo | Descrizione |
|-------|---------------------|------|-------------|
| `path` | `"/"` | string | Percorso dell'endpoint di health check |
| `use_http2` | `false` | boolean | Impostazione supporto HTTP/2 |
| `expected_status_codes` | `["200"]` | array | Codici di stato che indicano un'origine integra |
| `use_origin_server_name` | `{}` | object | Nome del server di origine per l'header Host |

## Raccomandazioni per Varianti OneOf

Gli schemi contenenti gruppi di campi mutuamente esclusivi (OneOf) includono `x-f5xc-recommended-oneof-variant` per indicare quale variante è più comunemente utilizzata. Questi metadati identificano la scelta tipica quando esistono più opzioni, basandosi sui valori predefiniti della console F5 XC e sui pattern di utilizzo comuni.

### Gruppi OneOf di Primo Livello

| Schema | Gruppo OneOf | Variante Raccomandata | Descrizione |
|--------|-------------|----------------------|-------------|
| `healthcheckCreateSpecType` | `health_check` | `http_health_check` | Tipo health check HTTP |
| `healthcheckReplaceSpecType` | `health_check` | `http_health_check` | Tipo health check HTTP |

### Gruppi OneOf healthcheckHttpHealthCheck

| Schema | Gruppo OneOf | Variante Raccomandata | Descrizione |
|--------|-------------|----------------------|-------------|
| `healthcheckHttpHealthCheck` | `host_header_choice` | `use_origin_server_name` | Header Host per le richieste di health check |

#### Scelta dell'Header Host

Il gruppo OneOf `host_header_choice` controlla come viene specificato l'header Host nelle richieste HTTP di health check.

| Variante | Tipo | Descrizione |
|----------|------|-------------|
| `use_origin_server_name` | `object` (vuoto) | Utilizza il nome del server di origine come header Host. Questa è la selezione predefinita nella console F5 XC. |
| `host_header` | `string` | Specifica un valore personalizzato per l'header Host nelle richieste di health check. |

**Riferimento Schema API**: `x-ves-oneof-field-host_header_choice: ["host_header", "use_origin_server_name"]`

## Riferimento Estensioni OpenAPI

Queste estensioni del vendor vengono aggiunte allo schema OpenAPI standard per trasmettere il comportamento predefinito specifico di F5 XC.

### x-f5xc-server-default

**Tipo**: `boolean`

Quando impostato su `true`, indica che il valore `default` associato è applicato dal server API F5 XC. I campi con questa estensione possono essere tranquillamente omessi dalle richieste API: il server applica automaticamente il valore predefinito.

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

### x-f5xc-recommended-value

**Tipo**: `any` (corrisponde al tipo del campo)

Specifica un valore che la console web F5 XC utilizza come valore predefinito pre-popolato. Questo valore non è applicato dal server ma rappresenta la configurazione iniziale tipica per le nuove risorse create tramite la console.

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

### x-f5xc-recommended-oneof-variant

**Tipo**: `object` (mappa dal nome del gruppo al nome della variante)

Per gli schemi con gruppi di campi mutuamente esclusivi, identifica quale variante è la scelta predefinita o più comune. La chiave è il nome del gruppo OneOf e il valore è il nome del campo della variante raccomandata.

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

