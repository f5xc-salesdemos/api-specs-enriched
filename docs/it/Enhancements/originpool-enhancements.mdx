Metadati di arricchimento per gli schemi relativi a origin_pool. Consultare [Estensioni OpenAPI](../development/#openapi-extensions) per le definizioni delle estensioni.

## Schemi Arricchiti

### Pattern

Tutti gli schemi che corrispondono a `origin_pool.*SpecType` ricevono arricchimenti:

- `origin_poolCreateSpecType`
- `origin_poolReplaceSpecType`
- `origin_poolGetSpecType`

## Campi Obbligatori

| Campo | Obbligatorio Per | Note |
|-------|------------------|------|
| `metadata.name` | Tutte le operazioni | 1-63 caratteri, alfanumerico minuscolo, pattern: `^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |
| `metadata.namespace` | Tutte le operazioni | Identificatore del namespace |
| `spec.origin_servers` | Creazione | Minimo 1 elemento |
| `spec.port` | Creazione | 1-65535, nessun valore predefinito del server |

## Configurazione Minima Funzionante

```json
{
  "metadata": {
    "name": "my-origin-pool",
    "namespace": "default"
  },
  "spec": {
    "origin_servers": [
      {
        "public_name": {
          "dns_name": "backend.example.com"
        }
      }
    ],
    "port": 443
  }
}
```

## Valori Predefiniti Applicati dal Server

I campi contrassegnati con `x-f5xc-server-default: true` hanno il loro valore `default` applicato dal server API di F5 XC quando vengono omessi dalle richieste.

### Campi di Primo Livello

| Campo | Valore Predefinito | Tipo | Descrizione |
|-------|--------------------|------|-------------|
| `no_tls` | `{}` | object | TLS verso l'origin disabilitato |
| `healthcheck` | `[]` | array | Nessun health check configurato |
| `loadbalancer_algorithm` | `ROUND_ROBIN` | enum | Bilanciamento del carico round-robin |
| `endpoint_selection` | `DISTRIBUTED` | enum | Utilizza tutti gli endpoint (locali + remoti) |

### Valori Predefiniti di advanced_options

Quando `advanced_options` non è specificato, il server si comporta come se questi valori fossero impostati:

| Campo | Valore Predefinito | Tipo | Descrizione |
|-------|--------------------|------|-------------|
| `connection_timeout` | `2000` | integer | Timeout di connessione in millisecondi |
| `http_idle_timeout` | `300000` | integer | Timeout di inattività HTTP in millisecondi (5 min) |
| `same_as_endpoint_port` | `{}` | object | L'health check utilizza la porta dell'endpoint |
| `default_circuit_breaker` | `{}` | object | Impostazioni predefinite del circuit breaker |
| `disable_outlier_detection` | `{}` | object | Rilevamento anomalie disabilitato |
| `no_panic_threshold` | `{}` | object | Nessuna soglia di panico |
| `disable_subsets` | `{}` | object | Bilanciamento del carico per sottoinsiemi disabilitato |
| `auto_http_config` | `{}` | object | Negoziazione automatica del protocollo HTTP |
| `disable_proxy_protocol` | `{}` | object | Protocollo proxy disabilitato |
| `disable_lb_source_ip_persistance` | `{}` | object | Persistenza IP sorgente del LB disabilitata |

### Valori Predefiniti degli Oggetti Nidificati

| Percorso | Valore Predefinito | Descrizione |
|----------|--------------------|-------------|
| `origin_servers[].labels` | `{}` | Oggetto labels vuoto |
| `origin_servers[].public_name.refresh_interval` | `0` | Utilizza l'intervallo di aggiornamento DNS predefinito del sistema |

## Discrepanze tra UI e Valori Predefiniti del Server

L'interfaccia web di F5 XC preseleziona valori diversi da quelli applicati dall'API quando i campi vengono omessi.

| Campo | Preselezionato nella UI | Applicato dal Server | Nota |
|-------|-------------------------|----------------------|------|
| `loadbalancer_algorithm` | `LB_OVERRIDE` | `ROUND_ROBIN` | Le risorse create tramite UI e quelle create tramite API differiscono quando il campo è omesso |

## Valori Enum

### loadbalancer_algorithm

| Valore | Descrizione | Note |
|--------|-------------|------|
| `ROUND_ROBIN` | Ogni endpoint sano viene selezionato in ordine round-robin | Valore predefinito del server |
| `LEAST_REQUEST` | Viene selezionato l'endpoint con il minor numero di richieste attive | |
| `RING_HASH` | Hashing consistente tramite ring hash dei nomi degli endpoint | |
| `RANDOM` | Selezione casuale di un endpoint sano | |
| `LB_OVERRIDE` | Policy di hash ereditata dal load balancer padre | Valore predefinito della UI |

### endpoint_selection

| Valore | Descrizione | Note |
|--------|-------------|------|
| `DISTRIBUTED` | Considera sia gli endpoint remoti che quelli locali | Valore predefinito del server |
| `LOCAL_ONLY` | Vengono utilizzati solo gli endpoint locali | |
| `LOCAL_PREFERRED` | Preferisce quelli locali, ricade sui remoti se non disponibili | |

## Gruppi di Campi OneOf

Gruppi di campi mutuamente esclusivi. È possibile specificare UN SOLO campo per ciascun gruppo:

| Gruppo | Campi | Valore Predefinito del Server |
|--------|-------|-------------------------------|
| Configurazione Porta | `port`, `automatic_port`, `lb_port` | `port` (esplicito) |
| TLS verso l'Origin | `no_tls`, `use_tls` | `no_tls` |
| Porta Health Check | `same_as_endpoint_port`, `health_check_port` | `same_as_endpoint_port` |
| Circuit Breaker | `default_circuit_breaker`, `disable_circuit_breaker`, `circuit_breaker` | `default_circuit_breaker` |
| Rilevamento Anomalie | `disable_outlier_detection`, `outlier_detection` | `disable_outlier_detection` |
| Soglia di Panico | `no_panic_threshold`, `panic_threshold` | `no_panic_threshold` |
| LB per Sottoinsiemi | `disable_subsets`, `enable_subsets` | `disable_subsets` |
| Protocollo HTTP | `auto_http_config`, `http1_config`, `http2_options` | `auto_http_config` |
| Protocollo Proxy | `disable_proxy_protocol`, `proxy_protocol_v1`, `proxy_protocol_v2` | `disable_proxy_protocol` |
| IP Sorgente LB | `disable_lb_source_ip_persistance`, `enable_lb_source_ip_persistance` | `disable_lb_source_ip_persistance` |
| Pool di Connessioni | `enable_conn_pool_reuse`, `disable_conn_pool_reuse` | `enable_conn_pool_reuse` |

## Tipi di Origin Server

Gli origin server possono essere specificati utilizzando diverse varianti di tipo. Il tipo determina quali campi aggiuntivi sono necessari.

### Tipi Implementati

| Tipo | Descrizione | Campi Obbligatori | Stato |
|------|-------------|-------------------|-------|
| `public_name` | Origin server con nome DNS pubblico | `dns_name` | ✅ Completo |
| `public_ip` | Origin server con indirizzo IP pubblico | `ip` (IPv4) | ✅ Completo |

### Esempio: public_name

```json
{
  "origin_servers": [
    {
      "public_name": {
        "dns_name": "backend.example.com"
      }
    }
  ]
}
```

### Esempio: public_ip

```json
{
  "origin_servers": [
    {
      "public_ip": {
        "ip": "8.8.8.8"
      }
    }
  ]
}
```
