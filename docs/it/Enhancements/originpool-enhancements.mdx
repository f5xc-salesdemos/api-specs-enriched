Metadati di arricchimento per gli schemi relativi a origin_pool. Consultare [Estensioni OpenAPI](../development/#openapi-extensions) per le definizioni delle estensioni.

## Schemi arricchiti

### Pattern

Tutti gli schemi corrispondenti a `origin_pool.*SpecType` ricevono arricchimenti:

- `origin_poolCreateSpecType`
- `origin_poolReplaceSpecType`
- `origin_poolGetSpecType`

## Campi obbligatori

| Campo | Obbligatorio per | Note |
|-------|-------------------|------|
| `metadata.name` | Tutte le operazioni | 1-63 caratteri, alfanumerico minuscolo, pattern: `^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |
| `metadata.namespace` | Tutte le operazioni | Identificatore del namespace |
| `spec.origin_servers` | Creazione | Minimo 1 elemento |
| `spec.port` | Creazione | 1-65535, nessun valore predefinito del server |

## Configurazione minima funzionante

```json
{
  "metadata": {
    "name": "my-origin-pool",
    "namespace": "default"
  },
  "spec": {
    "origin_servers": [
      {
        "public_name": {
          "dns_name": "backend.example.com"
        }
      }
    ],
    "port": 443
  }
}
```

## Valori predefiniti applicati dal server

I campi contrassegnati con `x-f5xc-server-default: true` hanno il loro valore `default` applicato dal server API F5 XC quando omessi dalle richieste.

### Campi di primo livello

| Campo | Valore predefinito | Tipo | Descrizione |
|-------|---------------------|------|-------------|
| `no_tls` | `{}` | object | TLS verso l'origin disabilitato |
| `healthcheck` | `[]` | array | Nessun health check configurato |
| `loadbalancer_algorithm` | `ROUND_ROBIN` | enum | Bilanciamento del carico round-robin |
| `endpoint_selection` | `DISTRIBUTED` | enum | Usa tutti gli endpoint (locali + remoti) |

## Discrepanze tra UI e valori predefiniti del server

L'interfaccia web F5 XC preseleziona valori diversi rispetto a quelli applicati dall'API quando i campi vengono omessi.

| Campo | Preselezionato dalla UI | Applicato dal server | Nota |
|-------|--------------------------|----------------------|------|
| `loadbalancer_algorithm` | `LB_OVERRIDE` | `ROUND_ROBIN` | Le risorse create tramite UI e quelle create tramite API differiscono quando il campo Ã¨ omesso |

## Valori enum

### loadbalancer_algorithm

| Valore | Descrizione | Note |
|--------|-------------|------|
| `ROUND_ROBIN` | Ogni endpoint sano viene selezionato in ordine round-robin | Valore predefinito del server |
| `LEAST_REQUEST` | Viene selezionato l'endpoint con il minor numero di richieste attive | |
| `RING_HASH` | Hashing consistente utilizzando il ring hash dei nomi degli endpoint | |
| `RANDOM` | Selezione casuale di un endpoint sano | |
| `LB_OVERRIDE` | La politica di hash Ã¨ ereditata dal load balancer padre | Valore predefinito della UI |

### endpoint_selection

| Valore | Descrizione | Note |
|--------|-------------|------|
| `DISTRIBUTED` | Considera sia gli endpoint remoti che quelli locali | Valore predefinito del server |
| `LOCAL_ONLY` | Vengono utilizzati solo gli endpoint locali | |
| `LOCAL_PREFERRED` | Preferenza per quelli locali, fallback sui remoti se non disponibili | |

## Tipi di origin server

Gli origin server possono essere specificati utilizzando diverse varianti di tipo. Il tipo determina quali campi aggiuntivi sono obbligatori.

### Tipi implementati

| Tipo | Descrizione | Campi obbligatori | Stato |
|------|-------------|-------------------|-------|
| `public_name` | Origin server con nome DNS pubblico | `dns_name` | âœ… Completo |
| `public_ip` | Origin server con indirizzo IP pubblico | `ip` (IPv4) | âœ… Completo |

### Esempio: public_name

```json
{
  "origin_servers": [

      "public_name": {
        "dns_name": "backend.example.com"
      }
    }
  ]
```

| Tipo | Descrizione | Campi obbligatori | Stato |
|------|-------------|-------------------|-------|

| `k8s_service` | Servizio Kubernetes con informazioni sul sito | `service_name`, `site_locator`, `network_choice` | ðŸ”² In attesa della logica site |
| `consul_service` | Servizio HashiCorp Consul con informazioni sul sito | `service_name`, `site_locator`, `network_choice` | ðŸ”² In attesa della logica site |

### Requisiti di implementazione

Prima che questi tipi possano essere completamente documentati:

1. **Scoperta della risorsa site** - Documentare lo schema della risorsa `site` e i pattern di site_locator
2. **Enum network choice** - Documentare i valori vincolati di `network_choice` (site_local_inside, site_local_outside, ecc.)
3. **Opzioni SNAT Pool** - Documentare i pattern di configurazione del pool SNAT
4. **Riferimenti tra risorse** - Definire come origin_pool fa riferimento alle risorse site
