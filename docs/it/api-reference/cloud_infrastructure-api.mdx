Connessioni multi-cloud provider con peering gateway e configurazione dei percorsi di rete. Integrazione con vault delle credenziali ed enumerazione delle sottoreti.

- **Categoria**: Infrastruttura
- **Complessità**: moderata
- **Percorsi**: 31 | **Schemi**: 208
- **Livello**: Standard
- **Domini correlati**: `sites`, `customer_edge`

## Casi d'uso

- Connessione ai cloud provider (AWS, Azure, GCP)
- Gestione delle credenziali cloud e dell'autenticazione
- Configurazione della connettività cloud e del provisioning elastico
- Collegamento e gestione delle regioni cloud

## Risorse principali

- **aws_vpc_site**: Distribuzione del sito AWS VPC con provisioning del nodo edge
- **azure_vnet_site**: Distribuzione del sito Azure VNet con provisioning del nodo edge
- **gcp_vpc_site**: Distribuzione del sito Google Cloud VPC con provisioning del nodo edge
- **cloud_credentials**: Credenziali di autenticazione del cloud provider per la distribuzione dei siti
