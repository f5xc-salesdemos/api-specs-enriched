Orchestrazione dei pod senza la complessità di un cluster completo. Esecuzione su siti edge, applicazione delle quote e profili di elaborazione standardizzati per applicazioni distribuite.

- **Categoria**: Infrastruttura
- **Complessità**: moderata
- **Percorsi**: 13 | **Schemi**: 159
- **Livello**: Avanzato
- **Domini correlati**: `managed_kubernetes`, `sites`, `service_mesh`

## Casi d'uso

- Distribuzione di namespace XCCS (Servizi container) per carichi di lavoro multi-tenant
- Gestione dei carichi di lavoro container con orchestrazione semplificata
- Configurazione di distribuzioni container edge distribuite
- Esecuzione di applicazioni containerizzate senza la complessità completa di K8s

## Risorse principali

- **virtual_k8s**: Namespace Kubernetes virtuale per carichi di lavoro container
- **workload**: Configurazione di distribuzione del carico di lavoro container
- **pod_security_policy**: Policy di sicurezza dei pod per vincoli di runtime del container
