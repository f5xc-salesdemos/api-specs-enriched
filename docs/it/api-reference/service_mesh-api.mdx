Definizioni dei tipi di applicazione con discovery e analisi dello schema appreso. Inferenza dei pattern di traffico per la gestione intelligente delle richieste.

- **Categoria**: Infrastruttura
- **Complessità**: avanzata
- **Percorsi**: 38 | **Schemi**: 247
- **Livello**: Avanzato
- **Domini correlati**: `managed_kubernetes`, `container_services`, `virtual`

## Casi d'uso

- Configurare la connettività del service mesh
- Gestire il discovery e il routing degli endpoint
- Configurare i servizi NFV
- Definire impostazioni e tipi di applicazione

## Risorse principali

- **endpoint**: Endpoint del service mesh per il routing del traffico
- **origin_pool**: Gruppo di server backend per la distribuzione del traffico del load balancer
- **service_discovery**: Configurazione del service discovery per endpoint dinamici
