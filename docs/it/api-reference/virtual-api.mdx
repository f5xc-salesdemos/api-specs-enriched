Distribuzione del traffico tra le regioni con regole di routing. Controlli di integrità e politiche di failover.

- **Categoria**: Networking
- **Complessità**: avanzata
- **Percorsi**: 133 | **Schemi**: 903
- **Livello**: Avanzato
- **Domini correlati**: `dns`, `service_policy`, `network`

## Casi d'uso

- Configurare bilanciatori di carico HTTP/TCP/UDP
- Gestire pool di origini e servizi
- Configurare virtual host e routing
- Definire limitatori di frequenza e politiche di servizio
- Gestire il routing basato sulla geolocalizzazione
- Configurare politiche di proxy e inoltro
- Gestire la protezione da malware e le campagne di minacce
- Configurare controlli di integrità e monitoraggio degli endpoint

## Risorse principali

- **http_loadbalancer**: Bilanciatore di carico HTTP/HTTPS di livello 7 per la distribuzione del traffico applicativo
- **tcp_loadbalancer**: Bilanciatore di carico TCP/UDP di livello 4 per il traffico di protocolli non HTTP
- **origin_pool**: Gruppo di server backend per la distribuzione del traffico del bilanciatore di carico
- **healthcheck**: Configurazione del monitoraggio dell'integrità per la disponibilità dei server di origine
- **app_firewall**: Politica Web Application Firewall per la protezione del traffico HTTP
- **service_policy**: Regole di controllo degli accessi a livello di servizio e gestione del traffico
- **malicious_user_detection**: Rilevamento e mitigazione automatica del comportamento di utenti malevoli
