ヘルスチェック関連スキーマのエンリッチメントメタデータ。拡張の定義については [OpenAPI 拡張](../development/#openapi-extensions) を参照してください。

## エンリッチされたスキーマ

### パターン

`healthcheck.*SpecType` に一致するすべてのスキーマがエンリッチメントを受けます：

- `healthcheckCreateSpecType`
- `healthcheckReplaceSpecType`
- `healthcheckGetSpecType`

さらに、ネストされたスキーマ `healthcheckHttpHealthCheck` には、HTTP ヘルスチェック設定のエンリッチメントが含まれています。

## サーバー適用のデフォルト値

`x-f5xc-server-default: true` でマークされたフィールドは、リクエストから省略された場合に F5 XC API サーバーによって `default` 値が適用されます。

### トップレベルフィールド

| フィールド | デフォルト値 | 型 | 説明 |
|-------|---------------|------|-------------|
| `jitter` | `0` | integer | タイミングランダム化の絶対ジッター値 |
| `jitter_percent` | `0` | integer | タイミングランダム化のパーセンテージベースのジッター |

### healthcheckHttpHealthCheck スキーマ

| フィールド | デフォルト値 | 型 | 説明 |
|-------|---------------|------|-------------|
| `use_origin_server_name` | `{}` | object | Host ヘッダー用のオリジンサーバー名 |
| `headers` | `{}` | object | カスタムヘッダー |
| `request_headers_to_remove` | `[]` | array | リクエストから除去するヘッダー |
| `use_http2` | `false` | boolean | HTTP/2 サポート |
| `expected_status_codes` | `[]` | array | 許容されるステータスコード（空の場合 = 200-299） |

## 推奨値

`x-f5xc-recommended-value` でマークされたフィールドは、新しいリソースを作成する際に F5 XC Web コンソールが事前入力する値を示します。

### トップレベルフィールド

| フィールド | 推奨値 | 型 | 説明 |
|-------|-------------------|------|-------------|
| `timeout` | `3` | integer | ヘルスチェックタイムアウト（秒） |
| `interval` | `15` | integer | ヘルスチェック間の間隔（秒） |
| `unhealthy_threshold` | `1` | integer | 異常と判定するまでの連続失敗回数 |
| `healthy_threshold` | `3` | integer | 正常と判定するまでの連続成功回数 |
| `jitter_percent` | `30` | integer | 本番環境向けのジッターパーセンテージ |

### healthcheckHttpHealthCheck スキーマ

| フィールド | 推奨値 | 型 | 説明 |
|-------|-------------------|------|-------------|
| `path` | `"/"` | string | ヘルスチェックエンドポイントパス |
| `use_http2` | `false` | boolean | HTTP/2 サポート設定 |
| `expected_status_codes` | `["200"]` | array | 正常なオリジンを示すステータスコード |
| `use_origin_server_name` | `{}` | object | Host ヘッダー用のオリジンサーバー名 |

## OneOf バリアントの推奨

相互に排他的なフィールドグループ（OneOf）を含むスキーマには、最も一般的に使用されるバリアントを示す `x-f5xc-recommended-oneof-variant` が含まれています。このメタデータは、F5 XC コンソールのデフォルトと一般的な使用パターンに基づいて、複数のオプションが存在する場合の典型的な選択肢を識別します。

### トップレベル OneOf グループ

| スキーマ | OneOf グループ | 推奨バリアント | 説明 |
|--------|-------------|---------------------|-------------|
| `healthcheckCreateSpecType` | `health_check` | `http_health_check` | HTTP ヘルスチェックタイプ |
| `healthcheckReplaceSpecType` | `health_check` | `http_health_check` | HTTP ヘルスチェックタイプ |

### healthcheckHttpHealthCheck OneOf グループ

| スキーマ | OneOf グループ | 推奨バリアント | 説明 |
|--------|-------------|---------------------|-------------|
| `healthcheckHttpHealthCheck` | `host_header_choice` | `use_origin_server_name` | ヘルスチェックリクエスト用の Host ヘッダー |

#### Host ヘッダーの選択

`host_header_choice` OneOf グループは、ヘルスチェック HTTP リクエストで Host ヘッダーがどのように指定されるかを制御します。

| バリアント | 型 | 説明 |
|---------|------|-------------|
| `use_origin_server_name` | `object`（空） | オリジンサーバー名を Host ヘッダーとして使用します。これは F5 XC コンソールでのデフォルト選択です。 |
| `host_header` | `string` | ヘルスチェックリクエスト用のカスタム Host ヘッダー値を指定します。 |

**API スキーマリファレンス**: `x-ves-oneof-field-host_header_choice: ["host_header", "use_origin_server_name"]`

## OpenAPI 拡張リファレンス

これらのベンダー拡張は、F5 XC 固有のデフォルト動作を伝えるために標準 OpenAPI スキーマに追加されます。

### x-f5xc-server-default

**型**: `boolean`

`true` の場合、付随する `default` 値が F5 XC API サーバーによって適用されることを示します。この拡張を持つフィールドは API リクエストから安全に省略でき、サーバーが自動的にデフォルトを適用します。

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

### x-f5xc-recommended-value

**型**: `any`（フィールドの型に一致）

F5 XC Web コンソールが事前入力のデフォルトとして使用する値を指定します。この値はサーバーによって強制されるものではなく、コンソール経由で作成される新しいリソースの典型的な初期設定を表します。

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

### x-f5xc-recommended-oneof-variant

**型**: `object`（グループ名からバリアント名へのマップ）

相互に排他的なフィールドグループを持つスキーマに対して、デフォルトまたは最も一般的な選択肢であるバリアントを識別します。キーは OneOf グループ名で、値は推奨バリアントのフィールド名です。

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```
## データアクセス

### OpenAPI 仕様

| ファイル | 内容 |
|------|---------|
| `docs/specifications/api/virtual.json` | `healthcheckHttpHealthCheck`、`healthcheckCreateSpecType`、`healthcheckReplaceSpecType`、`healthcheckGetSpecType` |
| `docs/specifications/api/openapi.json` | すべてのスキーマを含むマージ済み仕様 |

### validation.json の構造

```text
defaults.resources.healthcheck
├── server_applied      # x-f5xc-server-default: true のフィールド
├── recommended         # x-f5xc-recommended-value のフィールド
├── oneof_recommended   # OneOf バリアントの推奨
└── nested_recommended  # ネストされたスキーマの推奨値
```
