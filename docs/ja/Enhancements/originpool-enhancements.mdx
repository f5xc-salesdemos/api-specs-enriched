origin_pool 関連スキーマのエンリッチメントメタデータ。拡張定義については [OpenAPI Extensions](../development/#openapi-extensions) を参照してください。

## エンリッチ対象のスキーマ

### パターン

`origin_pool.*SpecType` に一致するすべてのスキーマがエンリッチメントを受けます：

- `origin_poolCreateSpecType`
- `origin_poolReplaceSpecType`
- `origin_poolGetSpecType`

## 必須フィールド

| フィールド | 必須となる操作 | 備考 |
|-------|--------------|-------|
| `metadata.name` | すべての操作 | 1-63文字、小文字英数字、パターン: `^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |
| `metadata.namespace` | すべての操作 | 名前空間識別子 |
| `spec.origin_servers` | Create | 最低1項目 |
| `spec.port` | Create | 1-65535、サーバーデフォルトなし |

## 最小限の構成

```json
{
  "metadata": {
    "name": "my-origin-pool",
    "namespace": "default"
  },
  "spec": {
    "origin_servers": [
      {
        "public_name": {
          "dns_name": "backend.example.com"
        }
      }
    ],
    "port": 443
  }
}
```

## サーバー適用のデフォルト値

`x-f5xc-server-default: true` が設定されたフィールドは、リクエストから省略された場合に F5 XC API サーバーによって `default` 値が適用されます。

### トップレベルフィールド

| フィールド | デフォルト値 | 型 | 説明 |
|-------|---------------|------|-------------|
| `no_tls` | `{}` | object | オリジンへの TLS が無効 |
| `healthcheck` | `[]` | array | ヘルスチェック未設定 |
| `loadbalancer_algorithm` | `ROUND_ROBIN` | enum | ラウンドロビン負荷分散 |
| `endpoint_selection` | `DISTRIBUTED` | enum | すべてのエンドポイントを使用（ローカル＋リモート） |

### advanced_options のデフォルト値

`advanced_options` が指定されていない場合、サーバーはこれらの値が設定されているかのように動作します：

| フィールド | デフォルト値 | 型 | 説明 |
|-------|---------------|------|-------------|
| `connection_timeout` | `2000` | integer | 接続タイムアウト（ミリ秒） |
| `http_idle_timeout` | `300000` | integer | HTTP アイドルタイムアウト（ミリ秒、5分） |
| `same_as_endpoint_port` | `{}` | object | ヘルスチェックにエンドポイントポートを使用 |
| `default_circuit_breaker` | `{}` | object | デフォルトのサーキットブレーカー設定 |
| `disable_outlier_detection` | `{}` | object | 外れ値検出が無効 |
| `no_panic_threshold` | `{}` | object | パニック閾値なし |
| `disable_subsets` | `{}` | object | サブセット負荷分散が無効 |
| `auto_http_config` | `{}` | object | 自動 HTTP プロトコルネゴシエーション |
| `disable_proxy_protocol` | `{}` | object | プロキシプロトコルが無効 |
| `disable_lb_source_ip_persistance` | `{}` | object | LB ソース IP パーシステンスが無効 |

### ネストされたオブジェクトのデフォルト値

| パス | デフォルト値 | 説明 |
|------|---------------|-------------|
| `origin_servers[].labels` | `{}` | 空のラベルオブジェクト |
| `origin_servers[].public_name.refresh_interval` | `0` | システムデフォルトの DNS リフレッシュを使用 |

## UI とサーバーデフォルトの不一致

F5 XC Web UI は、フィールドが省略された場合に API が適用する値とは異なる値を事前選択します。

| フィールド | UI の事前選択値 | サーバー適用値 | 備考 |
|-------|-----------------|----------------|------|
| `loadbalancer_algorithm` | `LB_OVERRIDE` | `ROUND_ROBIN` | フィールドが省略された場合、UI で作成されたリソースと API で作成されたリソースが異なる |

## Enum 値

### loadbalancer_algorithm

| 値 | 説明 | 備考 |
|-------|-------------|-------|
| `ROUND_ROBIN` | 各正常なエンドポイントをラウンドロビン順で選択 | サーバーデフォルト |
| `LEAST_REQUEST` | アクティブリクエストが最も少ないエンドポイントを選択 | |
| `RING_HASH` | エンドポイント名のリングハッシュを使用した一貫性ハッシュ | |
| `RANDOM` | 正常なエンドポイントをランダムに選択 | |
| `LB_OVERRIDE` | 親ロードバランサーからハッシュポリシーを継承 | UI デフォルト |

### endpoint_selection

| 値 | 説明 | 備考 |
|-------|-------------|-------|
| `DISTRIBUTED` | リモートとローカルの両方のエンドポイントを考慮 | サーバーデフォルト |
| `LOCAL_ONLY` | ローカルエンドポイントのみを使用 | |
| `LOCAL_PREFERRED` | ローカルを優先し、利用不可の場合はリモートにフォールバック | |

## OneOf フィールドグループ

相互排他的なフィールドグループ。各グループからフィールドを1つだけ指定できます：

| グループ | フィールド | サーバーデフォルト |
|-------|--------|----------------|
| ポート設定 | `port`, `automatic_port`, `lb_port` | `port`（明示的） |
| オリジンへの TLS | `no_tls`, `use_tls` | `no_tls` |
| ヘルスチェックポート | `same_as_endpoint_port`, `health_check_port` | `same_as_endpoint_port` |
| サーキットブレーカー | `default_circuit_breaker`, `disable_circuit_breaker`, `circuit_breaker` | `default_circuit_breaker` |
| 外れ値検出 | `disable_outlier_detection`, `outlier_detection` | `disable_outlier_detection` |
| パニック閾値 | `no_panic_threshold`, `panic_threshold` | `no_panic_threshold` |
| サブセット LB | `disable_subsets`, `enable_subsets` | `disable_subsets` |
| HTTP プロトコル | `auto_http_config`, `http1_config`, `http2_options` | `auto_http_config` |
| プロキシプロトコル | `disable_proxy_protocol`, `proxy_protocol_v1`, `proxy_protocol_v2` | `disable_proxy_protocol` |
| LB ソース IP | `disable_lb_source_ip_persistance`, `enable_lb_source_ip_persistance` | `disable_lb_source_ip_persistance` |
| コネクションプール | `enable_conn_pool_reuse`, `disable_conn_pool_reuse` | `enable_conn_pool_reuse` |

## フィールド制約

| フィールド | 型 | 制約 |
|-------|------|------------|
| `spec.port` | integer | 1-65535 |
| `spec.advanced_options.connection_timeout` | integer | 0-1,800,000 ms |
| `spec.advanced_options.http_idle_timeout` | integer | 0-600,000 ms |
| `spec.advanced_options.panic_threshold` | integer | 0-100（パーセンテージ） |
| `metadata.name` | string | 1-63文字、パターン: `^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |

## UI 設定オプション

F5 XC Web コンソールでは、Origin Pool に対して15の設定オプションが表示されます：

| # | UI ラベル | API フィールドパス | 型 | サーバーデフォルト |
|---|----------|----------------|------|----------------|
| 1 | Origin Server Port | `spec.[port\|automatic_port\|lb_port]` | OneOf | `port`（明示的） |
| 2 | Connection Pool Reuse | `spec.[enable_conn_pool_reuse\|disable_conn_pool_reuse]` | OneOf | `enable_conn_pool_reuse` |
| 3 | Health Check Port | `spec.advanced_options.[same_as_endpoint_port\|health_check_port]` | OneOf | `same_as_endpoint_port` |
| 4 | LoadBalancer Algorithm | `spec.loadbalancer_algorithm` | enum | `ROUND_ROBIN` |
| 5 | Endpoint Selection | `spec.endpoint_selection` | enum | `DISTRIBUTED` |
| 6 | TLS to Origin | `spec.[no_tls\|use_tls]` | OneOf | `no_tls` |
| 7 | Connection Timeout | `spec.advanced_options.connection_timeout` | integer | 2000 ms |
| 8 | HTTP Idle Timeout | `spec.advanced_options.http_idle_timeout` | integer | 300000 ms |
| 9 | Circuit Breaker | `spec.advanced_options.[default_circuit_breaker\|disable_circuit_breaker\|circuit_breaker]` | OneOf | `default_circuit_breaker` |
| 10 | Outlier Detection | `spec.advanced_options.[disable_outlier_detection\|outlier_detection]` | OneOf | `disable_outlier_detection` |
| 11 | Panic Threshold | `spec.advanced_options.[no_panic_threshold\|panic_threshold]` | OneOf | `no_panic_threshold` |
| 12 | Subset Load Balancing | `spec.advanced_options.[disable_subsets\|enable_subsets]` | OneOf | `disable_subsets` |
| 13 | HTTP Protocol | `spec.advanced_options.[auto_http_config\|http1_config\|http2_options]` | OneOf | `auto_http_config` |
| 14 | Proxy Protocol | `spec.advanced_options.[disable_proxy_protocol\|proxy_protocol_v1\|proxy_protocol_v2]` | OneOf | `disable_proxy_protocol` |
| 15 | LB Source IP Persistence | `spec.advanced_options.[disable_lb_source_ip_persistance\|enable_lb_source_ip_persistance]` | OneOf | `disable_lb_source_ip_persistance` |

## データアクセス

### OpenAPI 仕様

| ファイル | 内容 |
|------|---------|
| `docs/specifications/api/virtual.json` | `origin_poolCreateSpecType`, `origin_poolReplaceSpecType`, `origin_poolGetSpecType` |
| `docs/specifications/api/openapi.json` | すべてのスキーマを含むマージ済み仕様 |

### validation.json の構造

```text
defaults.resources.origin_pool
├── server_applied      # x-f5xc-server-default: true が設定されたフィールド
├── recommended         # x-f5xc-recommended-value が設定されたフィールド
├── advanced_options    # ネストされた advanced_options のデフォルト値
├── oneof_choices       # OneOf のデフォルト選択
└── ui_vs_server        # UI とサーバーの不一致
```

## オリジンサーバータイプ

オリジンサーバーは異なるタイプバリアントを使用して指定できます。タイプによって必要な追加フィールドが決まります。

### 実装済みのタイプ

| タイプ | 説明 | 必須フィールド | ステータス |
|------|-------------|-----------------|--------|
| `public_name` | パブリック DNS 名を持つオリジンサーバー | `dns_name` | ✅ 完了 |
| `public_ip` | パブリック IP アドレスを持つオリジンサーバー | `ip`（IPv4） | ✅ 完了 |

### 例: public_name

```json
{
  "origin_servers": [
    {
      "public_name": {
        "dns_name": "backend.example.com"
      }
    }
  ]
}
```

### 例: public_ip

```json
{
  "origin_servers": [
    {
      "public_ip": {
        "ip": "8.8.8.8"
      }
    }
  ]
}
```

以下のオリジンサーバータイプは、実装前に「site」リソースロジックの開発が必要です：

| タイプ | 説明 | 必須フィールド | ステータス |
|------|-------------|-----------------|--------|
| `private_ip` | プライベート/パブリック IP とサイト情報を持つオリジンサーバー | `ip`, `site_locator`, `network_choice`, `snat_pool` | 🔲 サイトロジック待ち |
| `private_name` | DNS 名とサイト情報を持つオリジンサーバー | `dns_name`, `site_locator`, `network_choice`, `snat_pool` | 🔲 サイトロジック待ち |
| `k8s_service` | サイト情報を持つ Kubernetes サービス | `service_name`, `site_locator`, `network_choice` | 🔲 サイトロジック待ち |
| `consul_service` | サイト情報を持つ HashiCorp Consul サービス | `service_name`, `site_locator`, `network_choice` | 🔲 サイトロジック待ち |

### 実装要件

これらのタイプを完全にドキュメント化する前に、以下が必要です：

1. **サイトリソースのディスカバリー** - `site` リソーススキーマと site_locator パターンのドキュメント化
2. **Network Choice Enum** - `network_choice` の制約付き値のドキュメント化（site_local_inside、site_local_outside など）
3. **SNAT プールオプション** - SNAT プール設定パターンのドキュメント化
4. **クロスリソース参照** - origin_pool がサイトリソースをどのように参照するかの定義
