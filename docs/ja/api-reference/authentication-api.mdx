プロバイダー統合、アクセスポリシー、資格情報のライフサイクル制御によるID管理。

- **カテゴリ**: プラットフォーム
- **複雑度**: シンプル
- **パス数**: 14 | **スキーマ数**: 38
- **ティア**: スタンダード
- **関連ドメイン**: `system`, `users`

## ユースケース

- 認証メカニズムの設定
- OIDCおよびOAuthプロバイダーの管理
- SCIMユーザープロビジョニングの設定
- API資格情報とアクセスの管理
- アカウントサインアップポリシーの設定

## 主要リソース

- **authentication_policy**: ユーザーおよびAPIアクセスの認証ポリシー
- **token**: プログラムによるアクセス用のAPIトークン
- **api_credential**: サービス認証用のAPI資格情報
