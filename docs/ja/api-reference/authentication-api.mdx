プロバイダー統合、アクセスポリシー、および認証情報のライフサイクル管理によるアイデンティティ管理。

- **カテゴリ**: プラットフォーム
- **複雑さ**: シンプル
- **パス数**: 14 | **スキーマ数**: 38
- **ティア**: スタンダード
- **関連ドメイン**: `system`, `users`

## ユースケース

- 認証メカニズムの設定
- OIDC および OAuth プロバイダーの管理
- SCIM ユーザープロビジョニングの設定
- API 認証情報とアクセスの管理
- アカウントサインアップポリシーの設定

## 主要リソース

- **authentication_policy**: ユーザーおよび API アクセスの認証ポリシー
- **token**: プログラムによるアクセス用の API トークン
- **api_credential**: サービス認証用の API 認証情報
