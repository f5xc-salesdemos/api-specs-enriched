トークンベースのプロビジョニングにより、イメージダウンロードとアップグレード前の検証が可能です。フリートグループ化により、分散拠点全体での一括操作が可能になります。

- **カテゴリ**: インフラストラクチャ
- **複雑度**: 中程度
- **パス**: 28 | **スキーマ**: 231
- **ティア**: スタンダード
- **関連ドメイン**: `customer_edge`, `sites`

## ユースケース

- カスタマーエッジサイトのライフサイクル管理
- ネットワークインターフェースとフリート設定の構成
- サイト登録とトークンワークフローの処理
- アップグレード前チェックを含むサイトアップグレードの実行

## 主要リソース

- **site_config**: エッジノード設定のためのサイト構成
- **fleet_config**: マルチサイト管理のためのフリート構成
- **registration_token**: サイトオンボーディングのための登録トークン
