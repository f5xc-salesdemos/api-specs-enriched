完全なクラスターの複雑さを伴わない Pod オーケストレーション。エッジサイトでの実行、クォータの適用、分散アプリケーション向けの標準化されたコンピュートプロファイル。

- **カテゴリ**: インフラストラクチャ
- **複雑度**: 中程度
- **パス数**: 13 | **スキーマ数**: 159
- **ティア**: アドバンスド
- **関連ドメイン**: `managed_kubernetes`, `sites`, `service_mesh`

## ユースケース

- マルチテナントワークロード向けの XCCS（コンテナサービス）名前空間のデプロイ
- 簡素化されたオーケストレーションによるコンテナワークロードの管理
- 分散エッジコンテナデプロイメントの構成
- 完全な K8s の複雑さなしでのコンテナ化アプリケーションの実行

## 主要リソース

- **virtual_k8s**: コンテナワークロード向けの仮想 Kubernetes 名前空間
- **workload**: コンテナワークロードのデプロイメント構成
- **pod_security_policy**: コンテナランタイム制約のための Pod セキュリティポリシー
