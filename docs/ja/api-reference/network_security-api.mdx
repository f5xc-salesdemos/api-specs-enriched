ソース、宛先、またはプロトコルに基づくルーティング決定を伴うファイアウォールルール。セグメンテーションはワークロードを分離し、アウトバウンドプロキシはアクセスを制御します。

- **カテゴリ**: セキュリティ
- **複雑度**: 上級
- **パス数**: 62 | **スキーマ数**: 386
- **ティア**: Advanced
- **関連ドメイン**: `virtual`, `api`, `network`

## ユースケース

- ネットワークファイアウォールおよび ACL ポリシーの設定
- NAT ポリシーおよびポートフォワーディングの管理
- ポリシーベースルーティングの設定
- ネットワークセグメントおよびポリシーの定義
- フォワードプロキシポリシーの設定

## 主要リソース

- **network_policy**: トラフィックフィルタリングのためのネットワークセキュリティポリシー
- **forward_proxy_policy**: アウトバウンドトラフィック制御のためのフォワードプロキシポリシー
- **network_firewall**: レイヤー 3/4 トラフィック保護のためのネットワークファイアウォール
