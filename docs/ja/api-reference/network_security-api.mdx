送信元、送信先、またはプロトコルに基づくルーティング判断を伴うファイアウォールルール。セグメンテーションはワークロードを分離し、アウトバウンドプロキシはアクセスを制御します。

- **カテゴリ**: セキュリティ
- **複雑度**: 上級
- **パス数**: 62 | **スキーマ数**: 386
- **ティア**: Advanced
- **関連ドメイン**: `virtual`, `api`, `network`

## ユースケース

- ネットワークファイアウォールおよびACLポリシーの設定
- NATポリシーとポートフォワーディングの管理
- ポリシーベースルーティングの設定
- ネットワークセグメントとポリシーの定義
- フォワードプロキシポリシーの設定

## 主要リソース

- **network_policy**: トラフィックフィルタリング用のネットワークセキュリティポリシー
- **forward_proxy_policy**: アウトバウンドトラフィック制御用のフォワードプロキシポリシー
- **network_firewall**: レイヤー3/4トラフィック保護用のネットワークファイアウォール
