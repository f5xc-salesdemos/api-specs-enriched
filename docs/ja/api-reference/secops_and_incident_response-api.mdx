脅威レベル分類による悪意のあるユーザーの緩和。不審な行動パターンに対する自動レスポンスアクション。

- **カテゴリ**: セキュリティ
- **複雑さ**: シンプル
- **パス数**: 4 | **スキーマ数**: 32
- **ティア**: アドバンスド
- **関連ドメイン**: `bot_defense`, `virtual`, `network_security`

## ユースケース

- 自動脅威緩和ポリシーの設定
- 悪意のあるユーザー検出ルールの定義
- インシデントレスポンスワークフローの管理
- 脅威へのブロッキングまたはレート制限の適用

## 主要リソース

- **mitigation_policy**: インシデントレスポンス用の緩和ポリシー
- **malicious_user_rule**: 脅威緩和用の悪意のあるユーザールール

## エンドポイント

| メソッド | パス | 説明 |
|--------|------|-------------|
| POST | `/api/config/namespaces/&#123;metadata.namespace&#125;/malicious_user_mitigations` | 悪意のあるユーザー緩和の作成。 |
| PUT | `/api/config/namespaces/&#123;metadata.namespace&#125;/malicious_user_mitigations/&#123;metadata.name&#125;` | 悪意のあるユーザー緩和の置換。 |
| GET | `/api/config/namespaces/&#123;namespace&#125;/malicious_user_mitigations` | 悪意のあるユーザー緩和の一覧取得。 |
| GET | `/api/config/namespaces/&#123;namespace&#125;/malicious_user_mitigations/&#123;name&#125;` | 悪意のあるユーザー緩和の取得。 |
| DELETE | `/api/config/namespaces/&#123;namespace&#125;/malicious_user_mitigations/&#123;name&#125;` | 悪意のあるユーザー緩和の削除。 |
