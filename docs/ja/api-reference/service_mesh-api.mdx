ディスカバリーと学習済みスキーマ分析によるアプリケーションタイプ定義。インテリジェントなリクエストハンドリングのためのトラフィックパターン推論。

- **カテゴリ**: インフラストラクチャ
- **複雑度**: 上級
- **パス数**: 38 | **スキーマ数**: 247
- **ティア**: アドバンスド
- **関連ドメイン**: `managed_kubernetes`, `container_services`, `virtual`

## ユースケース

- サービスメッシュ接続の構成
- エンドポイントディスカバリーとルーティングの管理
- NFV サービスの構成
- アプリケーション設定とタイプの定義

## 主要リソース

- **endpoint**: トラフィックルーティング用のサービスメッシュエンドポイント
- **origin_pool**: ロードバランサーのトラフィック分散用バックエンドサーバーグループ
- **service_discovery**: 動的エンドポイントのためのサービスディスカバリー構成
