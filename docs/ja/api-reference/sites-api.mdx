AWS、Azure、GCP VPC とトランジットゲートウェイの統合。リージョン間のポリシー適用のためのラベルベースの選択。

- **カテゴリ**: インフラストラクチャ
- **複雑度**: 上級
- **パス数**: 133 | **スキーマ数**: 1015
- **ティア**: Standard
- **関連ドメイン**: `cloud_infrastructure`, `customer_edge`, `managed_kubernetes`

## ユースケース

- クラウドプロバイダー（AWS、Azure、GCP）間で F5 XC をデプロイ
- XCKS（マネージド Kubernetes）サイトデプロイメントの管理（旧 AppStack）
- ネットワーキング重視のエッジデプロイメント向けに Secure Mesh サイトをデプロイ
- 外部 Kubernetes クラスターをカスタマーエッジとして統合
- AWS VPC、Azure VNet、GCP VPC サイトを構成
- 仮想サイトとサイトポリシーを管理

## 主要リソース

- **site**: エッジサービスのための物理またはクラウドデプロイメントロケーション
- **virtual_site**: ラベルセレクターを使用したサイトの論理グループ
- **site_mesh_group**: 複数サイト間のメッシュ接続構成
