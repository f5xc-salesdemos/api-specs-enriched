ルーティングルールによるリージョン間のトラフィック分散。ヘルスチェックとフェイルオーバーポリシー。

- **カテゴリ**: ネットワーキング
- **複雑度**: 上級
- **パス数**: 133 | **スキーマ数**: 903
- **ティア**: Advanced
- **関連ドメイン**: `dns`, `service_policy`, `network`

## ユースケース

- HTTP/TCP/UDPロードバランサーの設定
- オリジンプールとサービスの管理
- 仮想ホストとルーティングの設定
- レートリミッターとサービスポリシーの定義
- 地理位置情報ベースのルーティングの管理
- プロキシと転送ポリシーの設定
- マルウェア保護と脅威キャンペーンの管理
- ヘルスチェックとエンドポイント監視の設定

## 主要リソース

- **http_loadbalancer**: アプリケーショントラフィック分散用のレイヤー7 HTTP/HTTPSロードバランサー
- **tcp_loadbalancer**: 非HTTPプロトコルトラフィック用のレイヤー4 TCP/UDPロードバランサー
- **origin_pool**: ロードバランサーのトラフィック分散用バックエンドサーバーグループ
- **healthcheck**: オリジンサーバーの可用性を監視するヘルスモニタリング設定
- **app_firewall**: HTTPトラフィック保護用のWebアプリケーションファイアウォールポリシー
- **service_policy**: サービスレベルのアクセス制御とトラフィック管理ルール
- **malicious_user_detection**: 悪意のあるユーザー行動の自動検出と緩和
