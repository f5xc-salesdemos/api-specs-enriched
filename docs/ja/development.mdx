このガイドでは、F5 XC APIエンリッチメントパイプラインの開発ワークフローについて説明します。

## クイックスタート

### 初回セットアップ

```bash
# リポジトリをクローン
git clone https://github.com/f5xc-salesdemos/api-specs-enriched.git
cd api-specs-enriched

# 仮想環境を作成し依存関係をインストール
make install

# pre-commitフックをインストール
make pre-commit-install

# スペックをダウンロードしパイプラインを実行
make build

# ドキュメントをローカルでプレビュー
make serve
```

### 日常の開発

```bash
# クイックリビルド（キャッシュされたスペックを使用）
make rebuild

# すべての品質チェックを実行
make pre-commit-run

# ドキュメントをプレビュー
make serve
```

## ワークフローパターン

### 1. ディスカバリワークフロー

**前提条件**:
- F5 XC環境へのVPN接続
- 有効なAPI認証情報

```bash
# 認証情報を設定
export F5XC_API_URL="https://your-tenant.console.ves.volterra.io/api"
export F5XC_API_TOKEN="your-api-token"

# ディスカバリを実行
make discover

# 結果を確認
jq '.statistics' specs/discovered/session.json

# CI/CD用にコミット
make push-discovery
```

**ディスカバリで取得される情報**:
- 実際の必須/任意フィールド
- ライブレスポンスからのEnum値
- デフォルト値
- パターンバリデーション
- レスポンスの例

### 2. リリースワークフロー

リリースは自動化されています:

1. 日次スケジュール（UTC 6:00 AM）またはmainへのプッシュでワークフローがトリガーされます
2. ETagチェックでF5スペックの変更有無を判定します
3. パイプラインがスペックを処理しエンリッチメントします
4. gitタグからバージョンを算出: `git describe --tags --abbrev=0`
5. 直接コミット + タグを作成（バージョンバンプPRなし）
6. チェンジログ付きのGitHub Releaseを作成します
7. ドキュメントをGitHub Pagesにデプロイします

**バージョンバンプルール**:

| 条件 | バンプタイプ | 例 |
|-----------|-----------|---------|
| コミットに`[major]`を含む | メジャー | 1.0.0 → 2.0.0 |
| コミットに`BREAKING CHANGE`を含む | メジャー | 1.0.0 → 2.0.0 |
| 新しいドメインスペックが追加された | マイナー | 1.0.0 → 1.1.0 |
| その他の変更 | パッチ | 1.0.0 → 1.0.1 |

### 3. 開発ワークフロー

```bash
# フィーチャーブランチを作成
git checkout -b feature/my-change

# 設定またはスクリプトを変更

# ローカルでテスト
make pipeline
make lint

# コミット（pre-commitフックが自動実行されます）
git add .
git commit -m "feat: add new enrichment rule"

# プッシュしてPRを作成
git push -u origin feature/my-change
gh pr create
```

## ディスカバリの詳細

### ディスカバリとは？

ディスカバリは、ライブF5 XC APIを探索して以下を発見します:

- **ドキュメント化されていない制約**: OpenAPIで必須とマークされていない必須フィールド
- **実際のEnum値**: 本番環境で確認された実際の値
- **デフォルト動作**: フィールドが省略された場合にサーバーが適用する値
- **レスポンスパターン**: 実際のデータ構造

### ディスカバリ設定

```yaml
# config/discovery.yaml
discovery:
  exploration:
    namespaces:
      - "system"
      - "shared"
    methods:
      - "GET"
      - "OPTIONS"
    max_endpoints_per_run: 500

  schema_inference:
    sample_size: 3
    detect_patterns: true
    detect_constraints: true
```

### ディスカバリ拡張

ディスカバリはスペックに`x-discovered-*`拡張を追加します:

```json
{
  "properties": {
    "name": {
      "type": "string",
      "x-discovered-required": true,
      "x-discovered-pattern": "^[a-z][a-z0-9-]*$",
      "x-discovered-examples": ["my-app", "prod-lb"]
    }
  }
}
```

### 制約分析レポート

```bash
make constraint-report
# 出力: reports/constraint-analysis.md
```

## OpenAPI拡張

エンリッチメントパイプラインは、バリデーションとデフォルト値のメタデータを埋め込むためにベンダー拡張を使用します。

### 拡張カテゴリ

| 拡張 | 意味 | 含意 |
|-----------|---------|-------------|
| `x-f5xc-server-default` | フィールドが省略された場合にF5 XC APIサーバーがこの値を適用する | フィールドは任意。省略するとドキュメント化されたデフォルト動作が適用される |
| `x-f5xc-recommended-value` | F5 XCウェブコンソールが新規リソースに対してこの値を事前入力する | サーバーデフォルトはないが、この値が一般的な設定を表す |
| `x-f5xc-recommended-oneof-variant` | F5 XCコンソールがこのOneOfバリアントを事前選択する | 相互排他的な複数のオプションがある場合の一般的な選択を示す |
| `x-f5xc-conflicts-with` | このフィールドと同時に使用できない他のプロパティを一覧表示する | プロパティはOneOfグループの一部であり、競合するプロパティのうち1つのみ指定可能 |

### 必須フィールド拡張

| 拡張 | ソース | 目的 |
|-----------|--------|---------|
| `x-ves-required: "true"` | F5 XCオリジナルスペック | フィールドに非ゼロ/非空の値が必要 |
| `x-f5xc-required-for` | エンリッチメントパイプライン | コンテキスト固有の必須ステータス |

**x-f5xc-required-forのコンテキスト**:
- `create`: リソース作成時に必須
- `update`: リソース更新時に必須
- `minimum_config`: 最小限の実用構成に必須

### デフォルト値とOneOf拡張

| 拡張 | 目的 |
|-----------|---------|
| `x-f5xc-server-default: true` | サーバー適用デフォルトをマーク |
| `x-f5xc-recommended-value` | F5 XCコンソールの事前入力値 |
| `x-f5xc-recommended-oneof-variant` | 推奨OneOfバリアント |
| `x-f5xc-conflicts-with` | 他のOneOfプロパティとの相互排他性 |

#### x-f5xc-server-default

**型**: `boolean`

`true`の場合、付随する`default`値がF5 XC APIサーバーによって適用されることを示します。この拡張を持つフィールドはAPIリクエストから安全に省略できます — サーバーが自動的にデフォルトを適用します。

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

#### x-f5xc-recommended-value

**型**: `any`（フィールドの型に一致）

F5 XCウェブコンソールが事前入力されたデフォルトとして使用する値を指定します。この値はサーバーによって強制されませんが、コンソール経由で作成される新規リソースの一般的な初期設定を表します。

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

#### x-f5xc-recommended-oneof-variant

**型**: `object`（グループ名からバリアント名へのマップ）

相互排他的なフィールドグループを持つスキーマにおいて、デフォルトまたは最も一般的な選択肢となるバリアントを示します。キーはOneOfグループ名、値は推奨バリアントのフィールド名です。

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

#### x-f5xc-conflicts-with

**型**: 文字列の`array`

**追加**: v3.2.0（Issue #494）

OneOfグループメンバー間の相互排他性関係を宣言します。`x-ves-oneof-field-*`拡張から自動導出されます。これにより、ダウンストリームツール（Terraform、CLI、MCP）がランタイムではなくスキーマレベルで競合を検証できるようになります。

```yaml
host_header:
  type: string
  x-f5xc-conflicts-with: ["use_origin_server_name"]

use_origin_server_name:
  type: object
  x-f5xc-conflicts-with: ["host_header"]
```

**ユースケース**:
- Terraformプロバイダーがバリデーションルールを生成可能
- CLIツールが競合するフィールドの組み合わせについて警告可能
- AIアシスタントが正しいOneOf設定を生成可能
- IDE拡張が競合を認識したオートコンプリートを提供可能

**ソース**: パイプラインエンリッチメント中にF5ネイティブの`x-ves-oneof-field-*`拡張から自動導出されます。

### バリデーションルールの含意

エンリッチャーは`x-ves-validation-rules`を検査して必須ステータスを推論します:

| ルール | 含意 |
|------|-------------|
| `ves.io.schema.rules.message.required: "true"` | フィールドは必須 |
| `ves.io.schema.rules.uint32.gte: N` | Nが1以上でサーバーデフォルトがない場合、フィールドは必須 |
| `ves.io.schema.rules.repeated.min_items: N` | Nが1以上の場合、配列に少なくともN個の要素が必要 |
| `ves.io.schema.rules.string.min_bytes: N` | Nが1以上の場合、文字列に少なくともNバイトが必要 |

### サーバー適用デフォルトを持つリソース

一部のリソースは、サーバーがデフォルトを適用するため空のスペックを受け入れます:

| リソース | サーバー適用デフォルト |
|----------|------------------------|
| `app_firewall` | `monitoring: \{\}`、`default_detection_settings: \{\}` |
| `rate_limiter` | `limits: []`、`user_identification: []` |
| `api_definition` | `swagger_specs: []`、デフォルト`api_groups` |
| `healthcheck` | `jitter: 0`、`jitter_percent: 0`、ネストされたhttp_health_checkデフォルト |

これらのリソースでは、`x-ves-required`が`true`であっても`x-f5xc-required-for.create`が`false`になる場合があります。

### デフォルト値パターン

| パターン | 型 | 例 |
|---------|------|---------|
| `\{\}` | 空オブジェクト（選択肢の選択） | `monitoring: \{\}` |
| `[]` | 空配列 | `expected_status_codes: []` |
| `0` | 数値 | `jitter: 0` |
| `""` | 文字列 | `expected_response: ""` |
| `false` | ブール値 | `use_http2: false` |

### ディスカバリで発見したデフォルトの追加

1. API経由で最小限のスペックでF5 XCにリソースを作成
2. リソースを読み戻してサーバー適用値を確認
3. `config/discovered_defaults.yaml`にデフォルトを記録
4. パイプラインを実行: `make pipeline`
5. 以下で確認: `jq '.components.schemas | to_entries[] | select(.key | contains("resource_name"))'`

## トラブルシューティング

### pre-commitに時間がかかりすぎる

パイプラインは一貫性を確保するため、すべてのコミットで実行されます。作業中のコミットの場合:

```bash
git commit --no-verify -m "WIP: work in progress"
# 最終コミット前に実行: make pre-commit-run
```

### ディスカバリが失敗する

```bash
# VPNを確認
ping your-tenant.console.ves.volterra.io

# 認証情報を確認
echo $F5XC_API_TOKEN | head -c 10

# API URLの形式を確認
echo $F5XC_API_URL
# 形式: https://tenant.console.ves.volterra.io/api
```

### 生成されたスペックのリントエラー

```bash
# リントレポートを確認
cat reports/lint-report.json | jq '.errors'
```

出力ファイルではなく、エンリッチメント/正規化設定で問題を修正してください。

### バージョンシステム

バージョンはgitタグ（例: `v2.0.38`）から導出され、同時PRでのマージコンフリクトを引き起こすレースコンディションを排除します。

```python
# バージョンはgitタグから動的に計算されます
from scripts.utils.version_calculator import get_version_from_tags
version = get_version_from_tags()  # "2.0.38"を返す
```

### クローン後にスペックがない

生成されたスペックはgitignoreされています:

```bash
make build  # すべてをダウンロードして生成
```

### 大きなファイルがブロックされる

`.pre-commit-config.yaml`に除外を追加してください:

```yaml
- id: check-added-large-files
  exclude: ^path/to/large/file\.json$
```
