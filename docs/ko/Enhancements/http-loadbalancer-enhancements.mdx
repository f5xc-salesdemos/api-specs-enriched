HTTP 로드 밸런서 스키마에 대한 보강 메타데이터입니다. 확장 정의에 대해서는 [OpenAPI 확장](../development/#openapi-extensions)을 참조하세요.

## 보강된 스키마

### 패턴

`viewshttp_loadbalancer.*SpecType`와 일치하는 모든 스키마가 보강을 받습니다:

- `viewshttp_loadbalancerCreateSpecType`
- `viewshttp_loadbalancerReplaceSpecType`
- `viewshttp_loadbalancerGetSpecType`

## 최소 구성 메타데이터

`x-f5xc-minimum-configuration` 확장은 최소 실행 가능한 HTTP 로드 밸런서를 생성하기 위한 포괄적인 CLI 메타데이터를 제공합니다.

### 필수 필드

| 필드 | 제약 조건 | 설명 |
|-------|-----------|-------------|
| `metadata.name` | DNS 레이블 형식: `[a-z0-9]([-a-z0-9]*[a-z0-9])?` | 리소스 이름 |
| `metadata.namespace` | DNS 레이블 형식 | 네임스페이스 |
| `spec.domains` | 배열, min_items: 1 | 서비스할 도메인 |

### 로드 밸런서 유형 (OneOf 필수)

다음 로드 밸런서 유형 중 하나를 지정해야 합니다:

| 변형 | 설명 |
|---------|-------------|
| `spec.http` | HTTP 전용 |
| `spec.https` | 수동 인증서를 사용하는 HTTPS |
| `spec.https_auto_cert` | 자동 인증서를 사용하는 HTTPS |
| `spec.http_https` | HTTP 및 HTTPS 모두 |

**참고**: 로드 밸런서 유형 OneOf 그룹은 다음과 같이 참조됩니다:
- 구성 파일에서는 `lb_type` (`config/minimum_configs.yaml`의 약식)
- OpenAPI 사양 네이티브 확장에서는 `loadbalancer_type` (네이티브 필드 이름)

**API 스키마 참조**: `x-ves-oneof-field-loadbalancer_type: ["http", "https", "https_auto_cert", "http_https"]`

### 최소 실행 가능 구성

```json
{
  "metadata": {
    "name": "example-app",
    "namespace": "default"
  },
  "spec": {
    "domains": ["example.com"],
    "https_auto_cert": {
      "port": 443,
      "tls_config": {"default_security": {}}
    },
    "advertise_on_public_default_vip": {},
    "routes": [{"prefix": "/", "origin_pool": {"pool_name": "backend-pool"}}]
  }
}
```

## 서버 적용 기본값

`x-f5xc-server-default: true`로 표시된 필드는 요청에서 생략될 때 F5 XC API 서버에 의해 `default` 값이 적용됩니다. HTTP 로드 밸런서는 대부분의 보안 기능과 프로토콜 옵션이 비활성화 또는 안전한 상태로 기본 설정되기 때문에 서버 적용 기본값이 많습니다.

### HTTPS 자동 인증서 기본값

`https_auto_cert`가 지정된 경우, 생략된 필드에 대해 서버는 다음 기본값을 적용합니다:

| 필드 | 기본값 | 유형 | 설명 |
|-------|---------------|------|-------------|
| `port` | `443` | integer | HTTPS 수신 포트 |
| `http_redirect` | `false` | boolean | HTTP에서 HTTPS로 리다이렉트 |
| `add_hsts` | `false` | boolean | HTTP Strict Transport Security 헤더 |
| `tls_config.default_security` | `{}` | object | 강력한 암호화 스위트가 포함된 TLS 1.2+ |
| `no_mtls` | `{}` | object | 상호 TLS 비활성화 |
| `default_header` | `{}` | object | 기본 서버 이름 헤더 처리 |
| `enable_path_normalize` | `{}` | object | 경로 정규화 활성화 |
| `default_loadbalancer` | `{}` | object | 기본 로드 밸런서 설정 |
| `header_transformation_type.legacy_header_transformation` | `{}` | object | 레거시 헤더 변환 |
| `connection_idle_timeout` | `120000` | integer | 연결 유휴 시간 초과(밀리초, 2분) |
| `http_protocol_options.http_protocol_enable_v1_v2` | `{}` | object | HTTP/1.1 및 HTTP/2 활성화 |
| `coalescing_options.default_coalescing` | `{}` | object | 기본 HTTP/2 연결 병합 |

생략 시 모든 보안 기능은 비활성화로 기본 설정됩니다:

| 필드 | 기본값 | 설명 |

| `disable_rate_limit` | `{}` | 요청 속도 제한 비활성화 |
| `disable_api_discovery` | `{}` | API 디스커버리 비활성화 |
| `disable_api_testing` | `{}` | API 테스팅 비활성화 |
| `disable_api_definition` | `{}` | API 정의 비활성화 |
| `disable_malware_protection` | `{}` | 맬웨어 보호 비활성화 |
| `disable_client_side_defense` | `{}` | 클라이언트 측 방어 비활성화 |
| `disable_ip_reputation` | `{}` | IP 평판 비활성화 |
| `disable_threat_mesh` | `{}` | 위협 메시 비활성화 |
| `disable_malicious_user_detection` | `{}` | 악성 사용자 탐지 비활성화 |

| 필드 | 기본값 | 설명 |

| `add_location` | `true` | boolean | 응답에 위치 헤더 추가 |
| `system_default_timeouts` | `{}` | object | 시스템 기본 시간 초과 사용 |
| `service_policies_from_namespace` | `{}` | object | 네임스페이스에서 서비스 정책 상속 |

## OneOf 변형 권장 사항 (향후 확장)

**상태**: 아직 http_loadbalancer 스키마에 적용되지 않았습니다.

OneOf 그룹에 대해 가장 일반적인 선택을 나타내는 `x-f5xc-recommended-oneof-variant`를 포함하는 healthcheck와 달리, http_loadbalancer에는 현재 이 확장이 없습니다.

### Healthcheck와의 비교

**Healthcheck** (구현됨):
```yaml
viewshealthcheckCreateSpecType:
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

**HTTP LoadBalancer** (아직 미구현):
```yaml
viewshttp_loadbalancerCreateSpecType:
  x-f5xc-recommended-oneof-variant: null  # 권장 lb_type 변형을 나타낼 예정
```

### 향후 구현

http_loadbalancer에 이 확장을 추가하려면:
1. F5 XC 콘솔 기본 선택 사항 관찰 (예: UI에서 어떤 lb_type이 미리 선택되는지)
2. `config/discovered_defaults.yaml`에 구성 추가
3. 보강 파이프라인을 다시 실행하여 확장 적용

**추가 시 예상 구조**:
```yaml
viewshttp_loadbalancerCreateSpecType:
  x-f5xc-recommended-oneof-variant:
    loadbalancer_type: "https_auto_cert"  # 예시 - 검증 필요
```

이 확장은 다운스트림 도구가 사용자에게 구성 옵션을 제시할 때 가장 일반적으로 사용되는 변형을 미리 선택할 수 있게 합니다.

## 제약 조건 메타데이터

`x-f5xc-constraints`로 표시된 필드에는 API 분석에서 발견된 유효성 검사 제약 조건이 포함됩니다.

### 배열 제약 조건

| 필드 | minItems | maxItems | uniqueItems |
|-------|----------|----------|-------------|
| `spec.domains` | 1 | - | false |
| `spec.routes` | 1 | 256 | false |
| `spec.blocked_clients` | 1 | 128 | true |
| `spec.trusted_clients` | 1 | 128 | true |
| `spec.data_guard_rules` | 1 | 256 | true |

### 정수 제약 조건

| 필드 | 최솟값 | 최댓값 | 기본값 | 설명 |

| `spec.https_auto_cert.connection_idle_timeout` | 1000 | 3600000 | 120000 | 연결 유휴 시간 초과(밀리초, 1초~1시간) |

### 열거형 제약 조건

| 필드 | 값 | 기본값 | 설명 |

## 가이드 워크플로 통합

보강 파이프라인은 사양 수준에서 `x-f5xc-guided-workflows` 메타데이터를 추가합니다. 이러한 워크플로는 AI 어시스턴트와 CLI 도구에서 사용되는 단계별 배포 가이드를 제공합니다.

### HTTP 로드 밸런서 배포

백엔드 오리진 풀을 포함하여 완전히 구성된 HTTP 로드 밸런서를 생성하기 위한 5단계 워크플로입니다 (`config/guided_workflows.yaml`에 정의됨):

| 단계 | 작업 | 리소스 | 필수 필드 | 선택 사항 |
|------|--------|----------|-----------------|----------|
| 1 | 오리진 풀 생성 | `origin_pool` | `name`, `origin_servers`, `port` | 아니오 |
| 2 | 헬스 체크 구성 | `healthcheck` | `name`, `http_health_check` | 예 |
| 3 | 헬스 체크 연결 | - | - | 예 |
| 4 | HTTP 로드 밸런서 생성 | `http_loadbalancer` | `name`, `domains`, `http.port` | 아니오 |
| 5 | 배포 확인 | - | - | 아니오 |

**사전 요구 사항**: 대상 테넌트에 유효한 네임스페이스, IP 또는 DNS를 통해 접근 가능한 백엔드 애플리케이션, SSL 인증서(HTTPS의 경우 선택 사항).

### TLS를 포함한 HTTPS 로드 밸런서 배포

SSL/TLS 종료를 포함한 HTTPS 로드 밸런서를 생성하기 위한 7단계 워크플로입니다:

| 단계 | 작업 | 리소스 | 필수 필드 | 선택 사항 |
|------|--------|----------|-----------------|----------|
| 1 | SSL 인증서 업로드 | `certificate` | `name`, `certificate_chain`, `private_key` | 아니오 |
| 2 | 오리진 풀 생성 | `origin_pool` | `name`, `origin_servers` | 아니오 |
| 3 | 헬스 체크 구성 | `healthcheck` | - | 예 |
| 4 | WAF 정책 구성 | `app_firewall` | - | 예 |
| 5 | HTTPS 로드 밸런서 생성 | `http_loadbalancer` | `name`, `domains`, `https.tls_parameters` | 아니오 |
| 6 | WAF 정책 연결 | - | - | 예 |
| 7 | HTTPS 배포 확인 | - | - | 아니오 |

**사전 요구 사항**: 대상 테넌트에 유효한 네임스페이스, SSL 인증서 및 개인 키, 접근 가능한 백엔드 애플리케이션.

### 워크플로 활용

다운스트림 도구는 이러한 워크플로를 사용하여 다단계 배포를 통해 사용자를 안내합니다:

- **xcsh CLI**: 대화형 가이드 명령으로 워크플로를 제시합니다 (예: `xcsh deploy http-lb`)
- **VS Code 확장**: 진행 상황 추적이 포함된 사이드바 패널에 워크플로 단계를 표시합니다
- **AI 어시스턴트**: 워크플로 단계를 순차적으로 따르며, 각 리소스를 생성하고 종속성을 검증합니다

## OpenAPI 확장 참조

이러한 벤더 확장은 F5 XC 관련 메타데이터를 전달하기 위해 표준 OpenAPI 스키마에 추가됩니다.

### x-f5xc-cli-domain

**유형**: `string`

CLI 및 도구 구성을 위한 도메인 분류를 나타냅니다.

```yaml
viewshttp_loadbalancerCreateSpecType:
  type: object
  x-f5xc-cli-domain: "virtual"
```

### x-f5xc-minimum-configuration

**유형**: `object`

최소 실행 가능 구성을 생성하기 위한 포괄적인 메타데이터를 제공합니다. 포함 사항:
- 설명
- 제약 조건이 있는 필수 필드
- 상호 배타적 그룹
- 예제 구성 (YAML, JSON)
- curl 명령 예제

```yaml
viewshttp_loadbalancerCreateSpecType:
  type: object
  x-f5xc-minimum-configuration:
    description: "HTTP/HTTPS load balancer for distributing traffic across origin pools"
    required_fields:
      - "metadata.name"
      - "metadata.namespace"
      - "spec.domains"
    mutually_exclusive_groups:
      - name: "lb_type"
        fields: ["spec.http", "spec.https", "spec.https_auto_cert", "spec.http_https"]
        reason: "Choose exactly one load balancer type"
    example_yaml: |
      ...
    example_json: |
      ...
    example_curl: |
      ...
```

### x-f5xc-server-default

**유형**: `boolean`

`true`일 때, 함께 포함된 `default` 값이 F5 XC API 서버에 의해 적용됨을 나타냅니다. 이 확장이 있는 필드는 API 요청에서 안전하게 생략할 수 있으며, 서버가 자동으로 기본값을 적용합니다.

```yaml
disable_waf:
  type: object
  default: {}
  x-f5xc-server-default: true
```

### x-f5xc-conflicts-with

**유형**: `array of strings`

현재 필드와 상호 배타적인 필드 이름을 나열하여 OneOf 패턴을 나타냅니다.

```yaml
advertise_custom:
  type: object
  x-f5xc-conflicts-with:
    - advertise_on_public
    - advertise_on_public_default_vip
    - do_not_advertise
```

### x-f5xc-constraints

**유형**: `object`

배열 크기 제한, 정수 범위, 고유성 요구 사항 및 발견 메타데이터를 포함한 유효성 검사 제약 조건을 제공합니다.

```yaml
routes:
  type: array
  x-f5xc-constraints:
    constraintType: "array"
    minItems: 1
    maxItems: 256
    uniqueItems: false
    metadata:
      source: "discovery"
      confidence: 0.99
      validatedAt: "2026-01-19T12:00:00Z"
```

```yaml
port:
  type: integer
  x-f5xc-constraints:
    constraintType: "number"
    minimum: 1
    maximum: 65535
    metadata:
      source: "discovery"
      confidence: 0.99
```

### x-f5xc-description-short 및 x-f5xc-description-medium

**유형**: `string`

다양한 사용 사례에 대한 대체 설명 길이를 제공합니다:
- `x-f5xc-description-short`: 최대 60자 (CLI 열, 배지)
- `x-f5xc-description-medium`: 최대 150자 (툴팁, 요약)

```yaml
domains:
  type: array
  description: "List of domains the load balancer will serve"
  x-f5xc-description-short: "Domains to serve"
  x-f5xc-description-medium: "List of domains for which the load balancer accepts traffic"
```

### x-f5xc-example

**유형**: `any` (필드 유형에 일치)

문서화 및 도구를 위한 구체적인 필드 예제를 제공합니다.

```yaml
name:
  type: string
  x-f5xc-example: "example-app"

domains:
  type: array
  x-f5xc-example: ["example.com", "www.example.com"]
```

### x-f5xc-required-for

**유형**: `array of strings`

플래그를 사용하여 컨텍스트별 요구 사항을 나타냅니다: `minimum_config`, `create`, `update`, `read`.

```yaml
metadata.name:
  type: string
  x-f5xc-required-for: ["minimum_config", "create"]

metadata.uid:
  type: string
  x-f5xc-required-for: ["read"]
```
