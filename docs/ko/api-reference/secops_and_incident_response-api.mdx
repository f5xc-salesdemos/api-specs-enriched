위협 수준 분류를 통한 악의적 사용자 완화. 의심스러운 행동 패턴에 대한 자동화된 대응 조치.

- **카테고리**: 보안
- **복잡도**: 단순
- **경로**: 4 | **스키마**: 32
- **티어**: 고급
- **관련 도메인**: `bot_defense`, `virtual`, `network_security`

## 사용 사례

- 자동화된 위협 완화 정책 구성
- 악의적 사용자 탐지를 위한 규칙 정의
- 인시던트 대응 워크플로우 관리
- 위협에 대한 차단 또는 속도 제한 적용

## 주요 리소스

- **mitigation_policy**: 인시던트 대응을 위한 완화 정책
- **malicious_user_rule**: 위협 완화를 위한 악의적 사용자 규칙

## 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|-------------|
| POST | `/api/config/namespaces/&#123;metadata.namespace&#125;/malicious_user_mitigations` | 악의적 사용자 완화 생성. |
| PUT | `/api/config/namespaces/&#123;metadata.namespace&#125;/malicious_user_mitigations/&#123;metadata.name&#125;` | 악의적 사용자 완화 교체. |
| GET | `/api/config/namespaces/&#123;namespace&#125;/malicious_user_mitigations` | 악의적 사용자 완화 목록 조회. |
| GET | `/api/config/namespaces/&#123;namespace&#125;/malicious_user_mitigations/&#123;name&#125;` | 악의적 사용자 완화 조회. |
| DELETE | `/api/config/namespaces/&#123;namespace&#125;/malicious_user_mitigations/&#123;name&#125;` | 악의적 사용자 완화 삭제. |
