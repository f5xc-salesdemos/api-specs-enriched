Orquestração de pods sem a complexidade completa de um cluster. Execução em sites de borda, aplicação de cotas e perfis de computação padronizados para aplicações distribuídas.

- **Categoria**: Infraestrutura
- **Complexidade**: moderada
- **Caminhos**: 13 | **Schemas**: 159
- **Nível**: Avançado
- **Domínios relacionados**: `managed_kubernetes`, `sites`, `service_mesh`

## Casos de Uso

- Implantar namespaces XCCS (Container Services) para cargas de trabalho multi-tenant
- Gerenciar cargas de trabalho de containers com orquestração simplificada
- Configurar implantações de containers distribuídos na borda
- Executar aplicações containerizadas sem a complexidade completa do K8s

## Recursos Principais

- **virtual_k8s**: Namespace Kubernetes virtual para cargas de trabalho de containers
- **workload**: Configuração de implantação de carga de trabalho de container
- **pod_security_policy**: Política de segurança de pod para restrições de runtime de containers
