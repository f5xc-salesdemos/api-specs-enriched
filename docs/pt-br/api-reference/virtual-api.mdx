Distribuição de tráfego entre regiões com regras de roteamento. Verificações de saúde e políticas de failover.

- **Categoria**: Rede
- **Complexidade**: avançada
- **Caminhos**: 133 | **Schemas**: 903
- **Nível**: Avançado
- **Domínios relacionados**: `dns`, `service_policy`, `network`

## Casos de Uso

- Configurar balanceadores de carga HTTP/TCP/UDP
- Gerenciar pools de origem e serviços
- Configurar hosts virtuais e roteamento
- Definir limitadores de taxa e políticas de serviço
- Gerenciar roteamento baseado em geolocalização
- Configurar políticas de proxy e encaminhamento
- Gerenciar proteção contra malware e campanhas de ameaças
- Configurar verificações de saúde e monitoramento de endpoints

## Recursos Principais

- **http_loadbalancer**: Balanceador de carga HTTP/HTTPS de Camada 7 para distribuição de tráfego de aplicações
- **tcp_loadbalancer**: Balanceador de carga TCP/UDP de Camada 4 para tráfego de protocolos não-HTTP
- **origin_pool**: Grupo de servidores backend para distribuição de tráfego do balanceador de carga
- **healthcheck**: Configuração de monitoramento de saúde para disponibilidade de servidores de origem
- **app_firewall**: Política de Web Application Firewall para proteção de tráfego HTTP
- **service_policy**: Regras de controle de acesso e gerenciamento de tráfego em nível de serviço
- **malicious_user_detection**: Detecção e mitigação automatizada de comportamento de usuários maliciosos
