เมตาดาต้าเสริมสำหรับสคีมาที่เกี่ยวข้องกับ healthcheck ดู [ส่วนขยาย OpenAPI](../development/#openapi-extensions) สำหรับคำจำกัดความของส่วนขยาย

## สคีมาที่ได้รับการเสริม

### รูปแบบ

สคีมาทั้งหมดที่ตรงกับ `healthcheck.*SpecType` จะได้รับการเสริม:

- `healthcheckCreateSpecType`
- `healthcheckReplaceSpecType`
- `healthcheckGetSpecType`

นอกจากนี้ สคีมาซ้อน `healthcheckHttpHealthCheck` ยังมีการเสริมสำหรับการกำหนดค่า HTTP health check

## ค่าเริ่มต้นที่เซิร์ฟเวอร์กำหนด

ฟิลด์ที่ระบุด้วย `x-f5xc-server-default: true` จะมี `default` ที่ถูกกำหนดโดยเซิร์ฟเวอร์ F5 XC API เมื่อถูกละเว้นจากคำขอ

### ฟิลด์ระดับบนสุด

| ฟิลด์ | ค่าเริ่มต้น | ประเภท | คำอธิบาย |
|-------|-------------|--------|----------|
| `jitter` | `0` | integer | ค่า jitter แบบสัมบูรณ์สำหรับการสุ่มเวลา |
| `jitter_percent` | `0` | integer | jitter แบบเปอร์เซ็นต์สำหรับการสุ่มเวลา |

### สคีมา healthcheckHttpHealthCheck

| ฟิลด์ | ค่าเริ่มต้น | ประเภท | คำอธิบาย |
|-------|-------------|--------|----------|
| `use_origin_server_name` | `{}` | object | ชื่อเซิร์ฟเวอร์ต้นทางสำหรับ Host header |
| `headers` | `{}` | object | เฮดเดอร์ที่กำหนดเอง |
| `request_headers_to_remove` | `[]` | array | เฮดเดอร์ที่จะลบออกจากคำขอ |
| `use_http2` | `false` | boolean | การรองรับ HTTP/2 |
| `expected_status_codes` | `[]` | array | รหัสสถานะที่ยอมรับ (ว่าง = 200-299) |

## ค่าที่แนะนำ

ฟิลด์ที่ระบุด้วย `x-f5xc-recommended-value` แสดงถึงค่าที่คอนโซลเว็บ F5 XC กรอกล่วงหน้าเมื่อสร้างทรัพยากรใหม่

### ฟิลด์ระดับบนสุด

| ฟิลด์ | ค่าที่แนะนำ | ประเภท | คำอธิบาย |
|-------|-------------|--------|----------|
| `timeout` | `3` | integer | ระยะหมดเวลา health check เป็นวินาที |
| `interval` | `15` | integer | ช่วงเวลาระหว่าง health check เป็นวินาที |
| `unhealthy_threshold` | `1` | integer | จำนวนความล้มเหลวต่อเนื่องก่อนทำเครื่องหมายว่าไม่สมบูรณ์ |
| `healthy_threshold` | `3` | integer | จำนวนความสำเร็จต่อเนื่องก่อนทำเครื่องหมายว่าสมบูรณ์ |
| `jitter_percent` | `30` | integer | เปอร์เซ็นต์ jitter สำหรับการใช้งานจริง |

### สคีมา healthcheckHttpHealthCheck

| ฟิลด์ | ค่าที่แนะนำ | ประเภท | คำอธิบาย |
|-------|-------------|--------|----------|
| `path` | `"/"` | string | เส้นทาง endpoint สำหรับ health check |
| `use_http2` | `false` | boolean | การตั้งค่าการรองรับ HTTP/2 |
| `expected_status_codes` | `["200"]` | array | รหัสสถานะที่บ่งบอกว่า origin สมบูรณ์ |
| `use_origin_server_name` | `{}` | object | ชื่อเซิร์ฟเวอร์ต้นทางสำหรับ Host header |

## คำแนะนำตัวเลือก OneOf

สคีมาที่มีกลุ่มฟิลด์ที่ไม่สามารถใช้ร่วมกันได้ (OneOf) จะมี `x-f5xc-recommended-oneof-variant` เพื่อระบุว่าตัวเลือกใดที่ใช้กันมากที่สุด เมตาดาต้านี้ระบุตัวเลือกทั่วไปเมื่อมีหลายตัวเลือกให้เลือก โดยอิงจากค่าเริ่มต้นของคอนโซล F5 XC และรูปแบบการใช้งานทั่วไป

### กลุ่ม OneOf ระดับบนสุด

| สคีมา | กลุ่ม OneOf | ตัวเลือกที่แนะนำ | คำอธิบาย |
|-------|-------------|-------------------|----------|
| `healthcheckCreateSpecType` | `health_check` | `http_health_check` | ประเภท HTTP health check |
| `healthcheckReplaceSpecType` | `health_check` | `http_health_check` | ประเภท HTTP health check |

### กลุ่ม OneOf ของ healthcheckHttpHealthCheck

| สคีมา | กลุ่ม OneOf | ตัวเลือกที่แนะนำ | คำอธิบาย |
|-------|-------------|-------------------|----------|
| `healthcheckHttpHealthCheck` | `host_header_choice` | `use_origin_server_name` | Host header สำหรับคำขอ health check |

#### ตัวเลือก Host Header

กลุ่ม OneOf `host_header_choice` ควบคุมวิธีการระบุ Host header ในคำขอ HTTP สำหรับ health check

| ตัวเลือก | ประเภท | คำอธิบาย |
|----------|--------|----------|
| `use_origin_server_name` | `object` (ว่าง) | ใช้ชื่อเซิร์ฟเวอร์ต้นทางเป็น Host header นี่คือการเลือกเริ่มต้นในคอนโซล F5 XC |
| `host_header` | `string` | ระบุค่า Host header ที่กำหนดเองสำหรับคำขอ health check |

**อ้างอิงสคีมา API**: `x-ves-oneof-field-host_header_choice: ["host_header", "use_origin_server_name"]`

## อ้างอิงส่วนขยาย OpenAPI

ส่วนขยายของผู้จำหน่ายเหล่านี้ถูกเพิ่มเข้าในสคีมา OpenAPI มาตรฐานเพื่อสื่อสารพฤติกรรมค่าเริ่มต้นเฉพาะของ F5 XC

### x-f5xc-server-default

**ประเภท**: `boolean`

เมื่อเป็น `true` แสดงว่าค่า `default` ที่แนบมาถูกบังคับใช้โดยเซิร์ฟเวอร์ F5 XC API ฟิลด์ที่มีส่วนขยายนี้สามารถละเว้นจากคำขอ API ได้อย่างปลอดภัย เซิร์ฟเวอร์จะกำหนดค่าเริ่มต้นโดยอัตโนมัติ

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

### x-f5xc-recommended-value

**ประเภท**: `any` (ตรงกับประเภทของฟิลด์)

ระบุค่าที่คอนโซลเว็บ F5 XC ใช้เป็นค่าเริ่มต้นที่กรอกล่วงหน้า ค่านี้ไม่ได้ถูกบังคับใช้โดยเซิร์ฟเวอร์ แต่แสดงถึงการกำหนดค่าเริ่มต้นทั่วไปสำหรับทรัพยากรใหม่ที่สร้างผ่านคอนโซล

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

### x-f5xc-recommended-oneof-variant

**ประเภท**: `object` (แผนที่จากชื่อกลุ่มไปยังชื่อตัวเลือก)

สำหรับสคีมาที่มีกลุ่มฟิลด์ที่ไม่สามารถใช้ร่วมกันได้ ระบุว่าตัวเลือกใดเป็นค่าเริ่มต้นหรือเป็นตัวเลือกที่พบบ่อยที่สุด คีย์คือชื่อกลุ่ม OneOf และค่าคือชื่อฟิลด์ของตัวเลือกที่แนะนำ

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```
## การเข้าถึงข้อมูล

### ข้อกำหนด OpenAPI

| ไฟล์ | เนื้อหา |
|------|---------|
| `docs/specifications/api/virtual.json` | `healthcheckHttpHealthCheck`, `healthcheckCreateSpecType`, `healthcheckReplaceSpecType`, `healthcheckGetSpecType` |
| `docs/specifications/api/openapi.json` | ข้อกำหนดที่รวมสคีมาทั้งหมด |

### โครงสร้าง validation.json

```text
defaults.resources.healthcheck
├── server_applied      # ฟิลด์ที่มี x-f5xc-server-default: true
├── recommended         # ฟิลด์ที่มี x-f5xc-recommended-value
├── oneof_recommended   # คำแนะนำตัวเลือก OneOf
└── nested_recommended  # ค่าที่แนะนำสำหรับสคีมาซ้อน
```
