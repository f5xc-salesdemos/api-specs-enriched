เมตาดาต้าเสริมสำหรับ schemas ที่เกี่ยวข้องกับ origin_pool ดู [OpenAPI Extensions](../development/#openapi-extensions) สำหรับคำจำกัดความของ extension

## Schemas ที่ได้รับการเสริม

### รูปแบบ

schemas ทั้งหมดที่ตรงกับ `origin_pool.*SpecType` จะได้รับการเสริม:

- `origin_poolCreateSpecType`
- `origin_poolReplaceSpecType`
- `origin_poolGetSpecType`

## ฟิลด์ที่จำเป็น

| ฟิลด์ | จำเป็นสำหรับ | หมายเหตุ |
|-------|--------------|-------|
| `metadata.name` | การดำเนินการทั้งหมด | 1-63 ตัวอักษร, ตัวพิมพ์เล็กและตัวเลข, รูปแบบ: `^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |
| `metadata.namespace` | การดำเนินการทั้งหมด | ตัวระบุ namespace |
| `spec.origin_servers` | สร้าง | ขั้นต่ำ 1 รายการ |
| `spec.port` | สร้าง | 1-65535, ไม่มีค่าเริ่มต้นจากเซิร์ฟเวอร์ |

## การกำหนดค่าขั้นต่ำที่ใช้งานได้

```json
{
  "metadata": {
    "name": "my-origin-pool",
    "namespace": "default"
  },
  "spec": {
    "origin_servers": [
      {
        "public_name": {
          "dns_name": "backend.example.com"
        }
      }
    ],
    "port": 443
  }
}
```

## ค่าเริ่มต้นที่กำหนดโดยเซิร์ฟเวอร์

ฟิลด์ที่ระบุด้วย `x-f5xc-server-default: true` จะมีค่า `default` ถูกกำหนดโดย F5 XC API server เมื่อไม่ได้ระบุในคำขอ

### ฟิลด์ระดับบนสุด

| ฟิลด์ | ค่าเริ่มต้น | ประเภท | คำอธิบาย |
|-------|---------------|------|-------------|
| `no_tls` | `{}` | object | ปิดใช้งาน TLS ไปยัง origin |
| `healthcheck` | `[]` | array | ไม่มีการกำหนดค่า health check |
| `loadbalancer_algorithm` | `ROUND_ROBIN` | enum | การกระจายโหลดแบบ round-robin |
| `endpoint_selection` | `DISTRIBUTED` | enum | ใช้ endpoint ทั้งหมด (local + remote) |

### ค่าเริ่มต้นของ advanced_options

เมื่อไม่ได้ระบุ `advanced_options` เซิร์ฟเวอร์จะทำงานเสมือนว่ามีการตั้งค่าเหล่านี้:

| ฟิลด์ | ค่าเริ่มต้น | ประเภท | คำอธิบาย |
|-------|---------------|------|-------------|
| `connection_timeout` | `2000` | integer | ระยะเวลาหมดเวลาการเชื่อมต่อเป็นมิลลิวินาที |
| `http_idle_timeout` | `300000` | integer | ระยะเวลาหมดเวลา HTTP idle เป็นมิลลิวินาที (5 นาที) |
| `same_as_endpoint_port` | `{}` | object | Health check ใช้พอร์ตเดียวกับ endpoint |
| `default_circuit_breaker` | `{}` | object | การตั้งค่า circuit breaker เริ่มต้น |
| `disable_outlier_detection` | `{}` | object | ปิดใช้งานการตรวจจับ outlier |
| `no_panic_threshold` | `{}` | object | ไม่มี panic threshold |
| `disable_subsets` | `{}` | object | ปิดใช้งานการกระจายโหลดแบบ subset |
| `auto_http_config` | `{}` | object | การเจรจาโปรโตคอล HTTP อัตโนมัติ |
| `disable_proxy_protocol` | `{}` | object | ปิดใช้งาน proxy protocol |
| `disable_lb_source_ip_persistance` | `{}` | object | ปิดใช้งาน LB source IP persistence |

### ค่าเริ่มต้นของ object ที่ซ้อนกัน

| เส้นทาง | ค่าเริ่มต้น | คำอธิบาย |
|------|---------------|-------------|
| `origin_servers[].labels` | `{}` | object labels ว่าง |
| `origin_servers[].public_name.refresh_interval` | `0` | ใช้ค่าเริ่มต้นของระบบสำหรับการรีเฟรช DNS |

## ความแตกต่างระหว่างค่าเริ่มต้นของ UI กับเซิร์ฟเวอร์

F5 XC web UI เลือกค่าล่วงหน้าที่แตกต่างจากค่าที่ API กำหนดเมื่อไม่ได้ระบุฟิลด์

| ฟิลด์ | UI เลือกล่วงหน้า | เซิร์ฟเวอร์กำหนด | หมายเหตุ |
|-------|-----------------|----------------|------|
| `loadbalancer_algorithm` | `LB_OVERRIDE` | `ROUND_ROBIN` | ทรัพยากรที่สร้างจาก UI และที่สร้างจาก API จะแตกต่างกันเมื่อไม่ได้ระบุฟิลด์ |

## ค่า Enum

### loadbalancer_algorithm

| ค่า | คำอธิบาย | หมายเหตุ |
|-------|-------------|-------|
| `ROUND_ROBIN` | เลือก endpoint ที่ healthy แต่ละตัวตามลำดับ round-robin | ค่าเริ่มต้นของเซิร์ฟเวอร์ |
| `LEAST_REQUEST` | เลือก endpoint ที่มีคำขอที่ active น้อยที่สุด | |
| `RING_HASH` | consistent hashing โดยใช้ ring hash ของชื่อ endpoint | |
| `RANDOM` | เลือก endpoint ที่ healthy แบบสุ่ม | |
| `LB_OVERRIDE` | hash policy สืบทอดจาก parent load balancer | ค่าเริ่มต้นของ UI |

### endpoint_selection

| ค่า | คำอธิบาย | หมายเหตุ |
|-------|-------------|-------|
| `DISTRIBUTED` | พิจารณาทั้ง endpoint ระยะไกลและในเครื่อง | ค่าเริ่มต้นของเซิร์ฟเวอร์ |
| `LOCAL_ONLY` | ใช้เฉพาะ endpoint ในเครื่องเท่านั้น | |
| `LOCAL_PREFERRED` | ให้ความสำคัญกับ endpoint ในเครื่อง สำรองไปยังระยะไกลหากไม่พร้อมใช้งาน | |

## กลุ่มฟิลด์ OneOf

กลุ่มฟิลด์ที่ไม่สามารถใช้ร่วมกันได้ สามารถระบุได้เพียงฟิลด์เดียวจากแต่ละกลุ่ม:

| กลุ่ม | ฟิลด์ | ค่าเริ่มต้นของเซิร์ฟเวอร์ |
|-------|--------|----------------|
| การกำหนดค่าพอร์ต | `port`, `automatic_port`, `lb_port` | `port` (ระบุชัดเจน) |
| TLS ไปยัง Origin | `no_tls`, `use_tls` | `no_tls` |
| พอร์ต Health Check | `same_as_endpoint_port`, `health_check_port` | `same_as_endpoint_port` |
| Circuit Breaker | `default_circuit_breaker`, `disable_circuit_breaker`, `circuit_breaker` | `default_circuit_breaker` |
| การตรวจจับ Outlier | `disable_outlier_detection`, `outlier_detection` | `disable_outlier_detection` |
| Panic Threshold | `no_panic_threshold`, `panic_threshold` | `no_panic_threshold` |
| Subset LB | `disable_subsets`, `enable_subsets` | `disable_subsets` |
| โปรโตคอล HTTP | `auto_http_config`, `http1_config`, `http2_options` | `auto_http_config` |
| Proxy Protocol | `disable_proxy_protocol`, `proxy_protocol_v1`, `proxy_protocol_v2` | `disable_proxy_protocol` |
| LB Source IP | `disable_lb_source_ip_persistance`, `enable_lb_source_ip_persistance` | `disable_lb_source_ip_persistance` |
| Connection Pool | `enable_conn_pool_reuse`, `disable_conn_pool_reuse` | `enable_conn_pool_reuse` |

| ฟิลด์ | ประเภท | ข้อจำกัด |

## ประเภท Origin Server

Origin server สามารถระบุได้โดยใช้ตัวแปรประเภทที่แตกต่างกัน ประเภทจะกำหนดว่าต้องระบุฟิลด์เพิ่มเติมใดบ้าง

### ประเภทที่ใช้งานได้แล้ว

| ประเภท | คำอธิบาย | ฟิลด์ที่จำเป็น | สถานะ |
|------|-------------|-----------------|--------|
| `public_name` | Origin server พร้อมชื่อ DNS สาธารณะ | `dns_name` | ✅ เสร็จสมบูรณ์ |
| `public_ip` | Origin server พร้อมที่อยู่ IP สาธารณะ | `ip` (IPv4) | ✅ เสร็จสมบูรณ์ |

### ตัวอย่าง: public_name

```json
{
  "origin_servers": [
    {
      "public_name": {
        "dns_name": "backend.example.com"
      }
    }
  ]
}
```

### ตัวอย่าง: public_ip

```json
{
  "origin_servers": [
    {
      "public_ip": {
        "ip": "8.8.8.8"
      }
    }
  ]
}
```

## สิ่งที่ต้องทำ: ประเภท Origin Server ที่ขึ้นอยู่กับ Site

ประเภท origin server ต่อไปนี้ต้องการการพัฒนาลอจิกทรัพยากร "site" ก่อนการใช้งาน:

| ประเภท | คำอธิบาย | ฟิลด์ที่จำเป็น | สถานะ |
|------|-------------|-----------------|--------|
| `private_ip` | Origin server พร้อม IP ส่วนตัว/สาธารณะและข้อมูล site | `ip`, `site_locator`, `network_choice`, `snat_pool` | 🔲 รอลอจิก site |
| `private_name` | Origin server พร้อมชื่อ DNS และข้อมูล site | `dns_name`, `site_locator`, `network_choice`, `snat_pool` | 🔲 รอลอจิก site |
| `k8s_service` | Kubernetes service พร้อมข้อมูล site | `service_name`, `site_locator`, `network_choice` | 🔲 รอลอจิก site |
| `consul_service` | HashiCorp Consul service พร้อมข้อมูล site | `service_name`, `site_locator`, `network_choice` | 🔲 รอลอจิก site |

### ข้อกำหนดการใช้งาน

ก่อนที่ประเภทเหล่านี้จะสามารถจัดทำเอกสารได้อย่างสมบูรณ์:

1. **การค้นหาทรัพยากร Site** - จัดทำเอกสาร schema ทรัพยากร `site` และรูปแบบ site_locator
2. **Enum ตัวเลือกเครือข่าย** - จัดทำเอกสารค่าที่จำกัดของ `network_choice` (site_local_inside, site_local_outside ฯลฯ)
3. **ตัวเลือก SNAT Pool** - จัดทำเอกสารรูปแบบการกำหนดค่า SNAT pool
4. **การอ้างอิงข้ามทรัพยากร** - กำหนดวิธีที่ origin_pool อ้างอิงทรัพยากร site

### การติดตาม

- ตัวยึดตำแหน่งการกำหนดค่า: `config/discovered_defaults.yaml` → `origin_pool.origin_server_types.enums`
- Sprint ที่เกี่ยวข้อง: การเสริม schema ทรัพยากร site
