เมทาดาต้าเสริมสำหรับสคีมาที่เกี่ยวข้องกับ origin_pool ดู [ส่วนขยาย OpenAPI](../development/#openapi-extensions) สำหรับคำจำกัดความของส่วนขยาย

## สคีมาที่ได้รับการเสริม

### รูปแบบ

สคีมาทั้งหมดที่ตรงกับ `origin_pool.*SpecType` จะได้รับการเสริม:

- `origin_poolCreateSpecType`
- `origin_poolReplaceSpecType`
- `origin_poolGetSpecType`

## ฟิลด์ที่จำเป็น

| ฟิลด์ | จำเป็นสำหรับ | หมายเหตุ |
|-------|--------------|-------|
| `metadata.name` | การดำเนินการทั้งหมด | 1-63 ตัวอักษร, ตัวพิมพ์เล็กและตัวเลข, รูปแบบ: `^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |
| `metadata.namespace` | การดำเนินการทั้งหมด | ตัวระบุ Namespace |
| `spec.origin_servers` | สร้าง | ขั้นต่ำ 1 รายการ |
| `spec.port` | สร้าง | 1-65535, ไม่มีค่าเริ่มต้นจากเซิร์ฟเวอร์ |

## การกำหนดค่าขั้นต่ำที่ใช้งานได้

```json
{
  "metadata": {
    "name": "my-origin-pool",
    "namespace": "default"
  },
  "spec": {
    "origin_servers": [
      {
        "public_name": {
          "dns_name": "backend.example.com"
        }
      }
    ],
    "port": 443
  }
}
```

## ค่าเริ่มต้นที่เซิร์ฟเวอร์กำหนด

ฟิลด์ที่ทำเครื่องหมายด้วย `x-f5xc-server-default: true` จะมีค่า `default` ถูกกำหนดโดยเซิร์ฟเวอร์ API ของ F5 XC เมื่อถูกละเว้นจากคำขอ

### ฟิลด์ระดับบนสุด

| ฟิลด์ | ค่าเริ่มต้น | ชนิด | คำอธิบาย |
|-------|---------------|------|-------------|
| `no_tls` | `{}` | object | ปิดใช้งาน TLS ไปยัง origin |
| `healthcheck` | `[]` | array | ไม่มีการกำหนดค่า health check |
| `loadbalancer_algorithm` | `ROUND_ROBIN` | enum | การกระจายโหลดแบบ round-robin |
| `endpoint_selection` | `DISTRIBUTED` | enum | ใช้ endpoint ทั้งหมด (ภายในและระยะไกล) |

### ค่าเริ่มต้นของ advanced_options

เมื่อไม่ได้ระบุ `advanced_options` เซิร์ฟเวอร์จะทำงานเสมือนว่าค่าเหล่านี้ถูกตั้งไว้:

| ฟิลด์ | ค่าเริ่มต้น | ชนิด | คำอธิบาย |
|-------|---------------|------|-------------|
| `connection_timeout` | `2000` | integer | ระยะเวลาหมดเวลาการเชื่อมต่อในหน่วยมิลลิวินาที |
| `http_idle_timeout` | `300000` | integer | ระยะเวลาหมดเวลา HTTP idle ในหน่วยมิลลิวินาที (5 นาที) |
| `same_as_endpoint_port` | `{}` | object | Health check ใช้พอร์ตของ endpoint |
| `default_circuit_breaker` | `{}` | object | การตั้งค่า circuit breaker เริ่มต้น |
| `disable_outlier_detection` | `{}` | object | ปิดใช้งานการตรวจจับ outlier |
| `no_panic_threshold` | `{}` | object | ไม่มี panic threshold |
| `disable_subsets` | `{}` | object | ปิดใช้งานการกระจายโหลดแบบ subset |
| `auto_http_config` | `{}` | object | การเจรจาโปรโตคอล HTTP อัตโนมัติ |
| `disable_proxy_protocol` | `{}` | object | ปิดใช้งาน proxy protocol |
| `disable_lb_source_ip_persistance` | `{}` | object | ปิดใช้งานการคงอยู่ของ IP ต้นทางของ LB |

### ค่าเริ่มต้นของออบเจ็กต์ที่ซ้อนกัน

| เส้นทาง | ค่าเริ่มต้น | คำอธิบาย |
|------|---------------|-------------|
| `origin_servers[].labels` | `{}` | ออบเจ็กต์ labels ว่าง |
| `origin_servers[].public_name.refresh_interval` | `0` | ใช้ค่าเริ่มต้นของระบบสำหรับการรีเฟรช DNS |

## ความไม่ตรงกันระหว่างค่าเริ่มต้นของ UI และเซิร์ฟเวอร์

เว็บ UI ของ F5 XC เลือกค่าล่วงหน้าที่แตกต่างจากค่าที่ API กำหนดเมื่อฟิลด์ถูกละเว้น

| ฟิลด์ | ค่าที่ UI เลือกล่วงหน้า | ค่าที่เซิร์ฟเวอร์กำหนด | หมายเหตุ |
|-------|-----------------|----------------|------|
| `loadbalancer_algorithm` | `LB_OVERRIDE` | `ROUND_ROBIN` | ทรัพยากรที่สร้างจาก UI และ API จะแตกต่างกันเมื่อละเว้นฟิลด์ |

## ค่า Enum

### loadbalancer_algorithm

| ค่า | คำอธิบาย | หมายเหตุ |
|-------|-------------|-------|
| `ROUND_ROBIN` | เลือก endpoint ที่มีสุขภาพดีแต่ละตัวตามลำดับ round-robin | ค่าเริ่มต้นของเซิร์ฟเวอร์ |
| `LEAST_REQUEST` | เลือก endpoint ที่มีคำขอที่ทำงานอยู่น้อยที่สุด | |
| `RING_HASH` | การแฮชที่สม่ำเสมอโดยใช้ ring hash ของชื่อ endpoint | |
| `RANDOM` | เลือก endpoint ที่มีสุขภาพดีแบบสุ่ม | |
| `LB_OVERRIDE` | นโยบายแฮชสืบทอดจาก load balancer หลัก | ค่าเริ่มต้นของ UI |

### endpoint_selection

| ค่า | คำอธิบาย | หมายเหตุ |
|-------|-------------|-------|
| `DISTRIBUTED` | พิจารณา endpoint ทั้งระยะไกลและภายใน | ค่าเริ่มต้นของเซิร์ฟเวอร์ |
| `LOCAL_ONLY` | ใช้เฉพาะ endpoint ภายในเท่านั้น | |
| `LOCAL_PREFERRED` | ใช้ภายในก่อน สำรองเป็นระยะไกลหากไม่พร้อมใช้งาน | |

## กลุ่มฟิลด์ OneOf

กลุ่มฟิลด์ที่แยกจากกัน สามารถระบุได้เพียงหนึ่งฟิลด์จากแต่ละกลุ่มเท่านั้น:

| กลุ่ม | ฟิลด์ | ค่าเริ่มต้นของเซิร์ฟเวอร์ |
|-------|--------|----------------|
| การกำหนดค่าพอร์ต | `port`, `automatic_port`, `lb_port` | `port` (ระบุชัดเจน) |
| TLS ไปยัง Origin | `no_tls`, `use_tls` | `no_tls` |
| พอร์ต Health Check | `same_as_endpoint_port`, `health_check_port` | `same_as_endpoint_port` |
| Circuit Breaker | `default_circuit_breaker`, `disable_circuit_breaker`, `circuit_breaker` | `default_circuit_breaker` |
| การตรวจจับ Outlier | `disable_outlier_detection`, `outlier_detection` | `disable_outlier_detection` |
| Panic Threshold | `no_panic_threshold`, `panic_threshold` | `no_panic_threshold` |
| Subset LB | `disable_subsets`, `enable_subsets` | `disable_subsets` |
| โปรโตคอล HTTP | `auto_http_config`, `http1_config`, `http2_options` | `auto_http_config` |
| Proxy Protocol | `disable_proxy_protocol`, `proxy_protocol_v1`, `proxy_protocol_v2` | `disable_proxy_protocol` |
| LB Source IP | `disable_lb_source_ip_persistance`, `enable_lb_source_ip_persistance` | `disable_lb_source_ip_persistance` |
| Connection Pool | `enable_conn_pool_reuse`, `disable_conn_pool_reuse` | `enable_conn_pool_reuse` |

## ข้อจำกัดของฟิลด์

| ฟิลด์ | ชนิด | ข้อจำกัด |
|-------|------|------------|
| `spec.port` | integer | 1-65535 |
| `spec.advanced_options.connection_timeout` | integer | 0-1,800,000 มิลลิวินาที |
| `spec.advanced_options.http_idle_timeout` | integer | 0-600,000 มิลลิวินาที |
| `spec.advanced_options.panic_threshold` | integer | 0-100 (เปอร์เซ็นต์) |
| `metadata.name` | string | 1-63 ตัวอักษร, รูปแบบ: `^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |

## ชนิด Origin Server

สามารถระบุ origin server ได้โดยใช้ชนิดที่แตกต่างกัน ชนิดจะกำหนดว่าต้องใช้ฟิลด์เพิ่มเติมใดบ้าง

### ชนิดที่พัฒนาแล้ว

| ชนิด | คำอธิบาย | ฟิลด์ที่จำเป็น | สถานะ |
|------|-------------|-----------------|--------|
| `public_name` | Origin server ที่มีชื่อ DNS สาธารณะ | `dns_name` | ✅ เสร็จสมบูรณ์ |
| `public_ip` | Origin server ที่มีที่อยู่ IP สาธารณะ | `ip` (IPv4) | ✅ เสร็จสมบูรณ์ |

### ตัวอย่าง: public_name

```json
{
  "origin_servers": [
    {
      "public_name": {
        "dns_name": "backend.example.com"
      }
    }
  ]
}
```

### ตัวอย่าง: public_ip

```json
{
  "origin_servers": [
    {
      "public_ip": {
        "ip": "8.8.8.8"
      }
    }
  ]
}
```

## สิ่งที่ต้องทำ: ชนิด Origin Server ที่ขึ้นอยู่กับ Site

ชนิด origin server ต่อไปนี้ต้องการการพัฒนาตรรกะทรัพยากร "site" ก่อนการนำไปใช้งาน:

| ชนิด | คำอธิบาย | ฟิลด์ที่จำเป็น | สถานะ |
|------|-------------|-----------------|--------|
| `private_ip` | Origin server ที่มี IP ส่วนตัว/สาธารณะและข้อมูล site | `ip`, `site_locator`, `network_choice`, `snat_pool` | 🔲 รอตรรกะ site |
| `private_name` | Origin server ที่มีชื่อ DNS และข้อมูล site | `dns_name`, `site_locator`, `network_choice`, `snat_pool` | 🔲 รอตรรกะ site |
| `k8s_service` | บริการ Kubernetes ที่มีข้อมูล site | `service_name`, `site_locator`, `network_choice` | 🔲 รอตรรกะ site |
| `consul_service` | บริการ HashiCorp Consul ที่มีข้อมูล site | `service_name`, `site_locator`, `network_choice` | 🔲 รอตรรกะ site |

### ข้อกำหนดในการพัฒนา

ก่อนที่ชนิดเหล่านี้จะถูกจัดทำเอกสารได้อย่างสมบูรณ์:

1. **การค้นหาทรัพยากร Site** - จัดทำเอกสารสคีมาทรัพยากร `site` และรูปแบบ site_locator
2. **Enum ของ Network Choice** - จัดทำเอกสารค่าที่จำกัดของ `network_choice` (site_local_inside, site_local_outside เป็นต้น)
3. **ตัวเลือก SNAT Pool** - จัดทำเอกสารรูปแบบการกำหนดค่า SNAT pool
4. **การอ้างอิงข้ามทรัพยากร** - กำหนดวิธีที่ origin_pool อ้างอิงทรัพยากร site

### การติดตาม

- ตัวยึดตำแหน่งการกำหนดค่า: `config/discovered_defaults.yaml` → `origin_pool.origin_server_types.enums`
- สปรินต์ที่เกี่ยวข้อง: การเสริมสคีมาทรัพยากร Site
