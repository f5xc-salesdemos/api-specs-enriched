การจัดการข้อมูลประจำตัวด้วยการผสานผู้ให้บริการ, นโยบายการเข้าถึง และการควบคุมวงจรชีวิตของข้อมูลรับรองตัวตน

- **หมวดหมู่**: แพลตฟอร์ม
- **ความซับซ้อน**: ง่าย
- **เส้นทาง**: 14 | **สกีมา**: 38
- **ระดับ**: มาตรฐาน
- **โดเมนที่เกี่ยวข้อง**: `system`, `users`

## กรณีการใช้งาน

- กำหนดค่ากลไกการรับรองความถูกต้อง
- จัดการผู้ให้บริการ OIDC และ OAuth
- กำหนดค่าการจัดสรรผู้ใช้ด้วย SCIM
- จัดการข้อมูลรับรอง API และการเข้าถึง
- กำหนดค่านโยบายการสมัครบัญชี

## ทรัพยากรหลัก

- **authentication_policy**: นโยบายการรับรองความถูกต้องสำหรับผู้ใช้และการเข้าถึง API
- **token**: โทเค็น API สำหรับการเข้าถึงแบบโปรแกรม
- **api_credential**: ข้อมูลรับรอง API สำหรับการรับรองความถูกต้องของบริการ
