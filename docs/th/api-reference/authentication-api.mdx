การจัดการ Identity ด้วยการผสานรวม Provider, นโยบายการเข้าถึง และการควบคุมวงจรชีวิตของข้อมูลประจำตัว

- **หมวดหมู่**: แพลตฟอร์ม
- **ความซับซ้อน**: ง่าย
- **Paths**: 14 | **Schemas**: 38
- **ระดับ**: มาตรฐาน
- **โดเมนที่เกี่ยวข้อง**: `system`, `users`

## กรณีการใช้งาน

- กำหนดค่ากลไกการพิสูจน์ตัวตน
- จัดการ OIDC และ OAuth Provider
- กำหนดค่าการจัดเตรียมผู้ใช้ SCIM
- จัดการข้อมูลประจำตัว API และการเข้าถึง
- กำหนดค่านโยบายการสมัครบัญชี

## ทรัพยากรหลัก

- **authentication_policy**: นโยบายการพิสูจน์ตัวตนสำหรับผู้ใช้และการเข้าถึง API
- **token**: API token สำหรับการเข้าถึงแบบโปรแกรม
- **api_credential**: ข้อมูลประจำตัว API สำหรับการพิสูจน์ตัวตนบริการ
