การจัดการตัวตนพร้อมการเชื่อมต่อผู้ให้บริการ นโยบายการเข้าถึง และการควบคุมวงจรชีวิตข้อมูลรับรอง

- **หมวดหมู่**: Platform
- **ความซับซ้อน**: simple
- **เส้นทาง**: 14 | **สคีมา**: 38
- **ระดับ**: Standard
- **โดเมนที่เกี่ยวข้อง**: `system`, `users`

## กรณีการใช้งาน

- กำหนดค่ากลไกการยืนยันตัวตน
- จัดการผู้ให้บริการ OIDC และ OAuth
- กำหนดค่าการจัดเตรียมผู้ใช้ SCIM
- จัดการข้อมูลรับรอง API และการเข้าถึง
- กำหนดค่านโยบายการลงทะเบียนบัญชี

## ทรัพยากรหลัก

- **authentication_policy**: นโยบายการยืนยันตัวตนสำหรับการเข้าถึงของผู้ใช้และ API
- **token**: โทเค็น API สำหรับการเข้าถึงแบบโปรแกรม
- **api_credential**: ข้อมูลรับรอง API สำหรับการยืนยันตัวตนของบริการ
