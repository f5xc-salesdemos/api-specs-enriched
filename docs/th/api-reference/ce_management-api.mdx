การจัดสรรทรัพยากรโดยใช้โทเค็นพร้อมการดาวน์โหลดอิมเมจและการตรวจสอบก่อนอัปเกรด การจัดกลุ่มฝูงช่วยให้สามารถดำเนินการจำนวนมากได้ทั่วทั้งสถานที่แบบกระจาย

- **หมวดหมู่**: โครงสร้างพื้นฐาน
- **ความซับซ้อน**: ปานกลาง
- **เส้นทาง**: 28 | **สคีมา**: 231
- **ระดับ**: มาตรฐาน
- **โดเมนที่เกี่ยวข้อง**: `customer_edge`, `sites`

## กรณีการใช้งาน

- จัดการวงจรชีวิตของไซต์ Customer Edge
- กำหนดค่าอินเทอร์เฟซเครือข่ายและการตั้งค่ากลุ่มฝูง
- จัดการการลงทะเบียนไซต์และขั้นตอนการทำงานของโทเค็น
- ดำเนินการอัปเกรดไซต์พร้อมการตรวจสอบก่อนอัปเกรด

## ทรัพยากรหลัก

- **site_config**: การกำหนดค่าไซต์สำหรับการตั้งค่า edge node
- **fleet_config**: การกำหนดค่ากลุ่มฝูงสำหรับการจัดการหลายไซต์
- **registration_token**: โทเค็นการลงทะเบียนสำหรับการเริ่มต้นใช้งานไซต์
