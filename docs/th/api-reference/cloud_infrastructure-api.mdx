การเชื่อมต่อผู้ให้บริการมัลติคลาวด์พร้อมการเชื่อมต่อ gateway และการกำหนดค่าเส้นทางเครือข่าย การผสานรวม credential vault และการระบุ subnet

- **หมวดหมู่**: โครงสร้างพื้นฐาน
- **ความซับซ้อน**: ปานกลาง
- **เส้นทาง**: 31 | **สกีมา**: 208
- **ระดับ**: มาตรฐาน
- **โดเมนที่เกี่ยวข้อง**: `sites`, `customer_edge`

## กรณีการใช้งาน

- เชื่อมต่อกับผู้ให้บริการคลาวด์ (AWS, Azure, GCP)
- จัดการข้อมูลรับรองและการยืนยันตัวตนบนคลาวด์
- กำหนดค่าการเชื่อมต่อคลาวด์และการจัดสรรแบบยืดหยุ่น
- เชื่อมโยงและจัดการภูมิภาคคลาวด์

## ทรัพยากรหลัก

- **aws_vpc_site**: การติดตั้ง AWS VPC site พร้อมการจัดสรร edge node
- **azure_vnet_site**: การติดตั้ง Azure VNet site พร้อมการจัดสรร edge node
- **gcp_vpc_site**: การติดตั้ง Google Cloud VPC site พร้อมการจัดสรร edge node
- **cloud_credentials**: ข้อมูลรับรองการยืนยันตัวตนของผู้ให้บริการคลาวด์สำหรับการติดตั้ง site
