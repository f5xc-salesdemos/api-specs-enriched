การเชื่อมต่อผู้ให้บริการมัลติคลาวด์พร้อมการเพียร์เกตเวย์และการกำหนดค่าเส้นทางเครือข่าย การรวมคลังข้อมูลรับรองและการแจงนับซับเน็ต

- **หมวดหมู่**: โครงสร้างพื้นฐาน
- **ความซับซ้อน**: ปานกลาง
- **เส้นทาง**: 31 | **สคีมา**: 208
- **ระดับ**: มาตรฐาน
- **โดเมนที่เกี่ยวข้อง**: `sites`, `customer_edge`

## กรณีการใช้งาน

- เชื่อมต่อกับผู้ให้บริการคลาวด์ (AWS, Azure, GCP)
- จัดการข้อมูลรับรองคลาวด์และการยืนยันตัวตน
- กำหนดค่าการเชื่อมต่อคลาวด์และการจัดสรรแบบยืดหยุ่น
- เชื่อมโยงและจัดการภูมิภาคคลาวด์

## ทรัพยากรหลัก

- **aws_vpc_site**: การปรับใช้ไซต์ AWS VPC พร้อมการจัดสรรโหนดเอดจ์
- **azure_vnet_site**: การปรับใช้ไซต์ Azure VNet พร้อมการจัดสรรโหนดเอดจ์
- **gcp_vpc_site**: การปรับใช้ไซต์ Google Cloud VPC พร้อมการจัดสรรโหนดเอดจ์
- **cloud_credentials**: ข้อมูลรับรองการยืนยันตัวตนของผู้ให้บริการคลาวด์สำหรับการปรับใช้ไซต์
