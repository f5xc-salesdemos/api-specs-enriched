การเชื่อมต่อผู้ให้บริการมัลติคลาวด์พร้อมการเชื่อมต่อ gateway แบบ peering และการกำหนดค่าเส้นทางเครือข่าย การรวมระบบ credential vault และการระบุ subnet

- **หมวดหมู่**: โครงสร้างพื้นฐาน
- **ความซับซ้อน**: ปานกลาง
- **เส้นทาง**: 31 | **สคีมา**: 208
- **ระดับ**: มาตรฐาน
- **โดเมนที่เกี่ยวข้อง**: `sites`, `customer_edge`

## กรณีการใช้งาน

- เชื่อมต่อกับผู้ให้บริการคลาวด์ (AWS, Azure, GCP)
- จัดการ credential และการยืนยันตัวตนของคลาวด์
- กำหนดค่าการเชื่อมต่อคลาวด์และการจัดสรรแบบยืดหยุ่น
- เชื่อมโยงและจัดการภูมิภาคคลาวด์

## ทรัพยากรหลัก

- **aws_vpc_site**: การปรับใช้ AWS VPC site พร้อมการจัดสรร edge node
- **azure_vnet_site**: การปรับใช้ Azure VNet site พร้อมการจัดสรร edge node
- **gcp_vpc_site**: การปรับใช้ Google Cloud VPC site พร้อมการจัดสรร edge node
- **cloud_credentials**: credential สำหรับยืนยันตัวตนกับผู้ให้บริการคลาวด์สำหรับการปรับใช้ site
