การจัดการ Pod โดยไม่ต้องมีความซับซ้อนของคลัสเตอร์เต็มรูปแบบ การทำงานที่ไซต์ Edge การบังคับใช้โควตา และโปรไฟล์คอมพิวต์มาตรฐานสำหรับแอปพลิเคชันแบบกระจาย

- **หมวดหมู่**: โครงสร้างพื้นฐาน
- **ความซับซ้อน**: ปานกลาง
- **เส้นทาง**: 13 | **สคีมา**: 159
- **ระดับ**: ขั้นสูง
- **โดเมนที่เกี่ยวข้อง**: `managed_kubernetes`, `sites`, `service_mesh`

## กรณีการใช้งาน

- ปรับใช้เนมสเปซ XCCS (Container Services) สำหรับเวิร์กโหลดแบบหลายผู้เช่า
- จัดการเวิร์กโหลดคอนเทนเนอร์ด้วยการจัดการแบบเรียบง่าย
- กำหนดค่าการปรับใช้คอนเทนเนอร์ Edge แบบกระจาย
- รันแอปพลิเคชันแบบคอนเทนเนอร์โดยไม่ต้องมีความซับซ้อนของ K8s เต็มรูปแบบ

## ทรัพยากรหลัก

- **virtual_k8s**: เนมสเปซ Kubernetes เสมือนสำหรับเวิร์กโหลดคอนเทนเนอร์
- **workload**: การกำหนดค่าการปรับใช้เวิร์กโหลดคอนเทนเนอร์
- **pod_security_policy**: นโยบายความปลอดภัยของ Pod สำหรับข้อจำกัดรันไทม์คอนเทนเนอร์
