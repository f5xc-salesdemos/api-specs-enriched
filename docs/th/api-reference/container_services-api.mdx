การจัดการ Pod โดยไม่ต้องมีความซับซ้อนของคลัสเตอร์เต็มรูปแบบ การรันที่ Edge site, การบังคับใช้โควตา และโปรไฟล์คอมพิวต์มาตรฐานสำหรับแอปพลิเคชันแบบกระจาย

- **หมวดหมู่**: โครงสร้างพื้นฐาน
- **ความซับซ้อน**: ปานกลาง
- **Paths**: 13 | **Schemas**: 159
- **ระดับ**: ขั้นสูง
- **โดเมนที่เกี่ยวข้อง**: `managed_kubernetes`, `sites`, `service_mesh`

## กรณีการใช้งาน

- ปรับใช้ XCCS (Container Services) namespaces สำหรับ workloads แบบ multi-tenant
- จัดการ container workloads ด้วยการจัดการแบบง่าย
- กำหนดค่าการปรับใช้ edge container แบบกระจาย
- รันแอปพลิเคชันแบบ containerized โดยไม่ต้องมีความซับซ้อนของ K8s เต็มรูปแบบ

## ทรัพยากรหลัก

- **virtual_k8s**: Virtual Kubernetes namespace สำหรับ container workloads
- **workload**: การกำหนดค่าการปรับใช้ container workload
- **pod_security_policy**: นโยบายความปลอดภัยของ Pod สำหรับข้อจำกัดของ container runtime
