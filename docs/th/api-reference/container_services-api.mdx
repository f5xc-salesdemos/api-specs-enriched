การประสานงาน Pod โดยไม่ต้องมีความซับซ้อนของคลัสเตอร์เต็มรูปแบบ การดำเนินการที่ไซต์ Edge การบังคับใช้โควตา และโปรไฟล์การประมวลผลมาตรฐานสำหรับแอปพลิเคชันแบบกระจาย

- **หมวดหมู่**: โครงสร้างพื้นฐาน
- **ความซับซ้อน**: ปานกลาง
- **Paths**: 13 | **Schemas**: 159
- **ระดับ**: ขั้นสูง
- **โดเมนที่เกี่ยวข้อง**: `managed_kubernetes`, `sites`, `service_mesh`

## กรณีการใช้งาน

- ปรับใช้เนมสเปซ XCCS (Container Services) สำหรับปริมาณงานแบบหลายผู้เช่า
- จัดการปริมาณงานคอนเทนเนอร์ด้วยการประสานงานที่เรียบง่าย
- กำหนดค่าการปรับใช้คอนเทนเนอร์ที่ Edge แบบกระจาย
- รันแอปพลิเคชันที่ใช้คอนเทนเนอร์โดยไม่ต้องมีความซับซ้อนของ K8s เต็มรูปแบบ

## ทรัพยากรหลัก

- **virtual_k8s**: เนมสเปซ Kubernetes เสมือนสำหรับปริมาณงานคอนเทนเนอร์
- **workload**: การกำหนดค่าการปรับใช้ปริมาณงานคอนเทนเนอร์
- **pod_security_policy**: นโยบายความปลอดภัย Pod สำหรับข้อจำกัดรันไทม์ของคอนเทนเนอร์
