Kubernetes role bindings และ admission policies การผสานรวม Registry สำหรับเวิร์กโหลด EKS, AKS และ GKE

- **หมวดหมู่**: โครงสร้างพื้นฐาน
- **ความซับซ้อน**: ปานกลาง
- **Paths**: 20 | **Schemas**: 95
- **ระดับ**: ขั้นสูง
- **โดเมนที่เกี่ยวข้อง**: `container_services`, `sites`, `service_mesh`

## กรณีการใช้งาน

- จัดการ RBAC และความปลอดภัยของคลัสเตอร์ XCKS (Managed Kubernetes)
- กำหนดค่านโยบายความปลอดภัยของ Pod และ Admission Controllers
- จัดการ Container Registry สำหรับการปรับใช้ระดับองค์กร
- ผสานรวมกับคลัสเตอร์ Kubernetes ภายนอก (EKS, AKS, GKE)

## ทรัพยากรหลัก

- **mk8s_cluster**: การกำหนดค่าคลัสเตอร์ Managed Kubernetes
- **k8s_cluster_role**: การกำหนดค่า RBAC Role ของคลัสเตอร์ Kubernetes
- **container_registry**: Container Image Registry สำหรับอิมเมจเวิร์กโหลด
