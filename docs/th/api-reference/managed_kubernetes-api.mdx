การผูก role ของ Kubernetes และนโยบาย admission ทะเบียนการผสานรวมสำหรับ workload ของ EKS, AKS และ GKE

- **หมวดหมู่**: โครงสร้างพื้นฐาน
- **ความซับซ้อน**: ปานกลาง
- **เส้นทาง**: 20 | **สคีมา**: 95
- **ระดับ**: ขั้นสูง
- **โดเมนที่เกี่ยวข้อง**: `container_services`, `sites`, `service_mesh`

## กรณีการใช้งาน

- จัดการ RBAC ของคลัสเตอร์ XCKS (Managed Kubernetes) และความปลอดภัย
- กำหนดค่านโยบายความปลอดภัยของ pod และตัวควบคุม admission
- จัดการ container registries สำหรับการปรับใช้งานระดับองค์กร
- ผสานรวมกับคลัสเตอร์ Kubernetes ภายนอก (EKS, AKS, GKE)

## ทรัพยากรหลัก

- **mk8s_cluster**: การกำหนดค่าคลัสเตอร์ Managed Kubernetes
- **k8s_cluster_role**: การกำหนดค่า RBAC role ของคลัสเตอร์ Kubernetes
- **container_registry**: Container image registry สำหรับ workload images
