กฎไฟร์วอลล์พร้อมการตัดสินใจเส้นทางตามต้นทาง ปลายทาง หรือโปรโตคอล การแบ่งเซกเมนต์แยกเวิร์กโหลดออกจากกัน ขณะที่พร็อกซีขาออกควบคุมการเข้าถึง

- **หมวดหมู่**: ความปลอดภัย
- **ความซับซ้อน**: ขั้นสูง
- **เส้นทาง**: 62 | **สคีมา**: 386
- **ระดับ**: ขั้นสูง
- **โดเมนที่เกี่ยวข้อง**: `virtual`, `api`, `network`

## กรณีการใช้งาน

- กำหนดค่านโยบายไฟร์วอลล์เครือข่ายและ ACL
- จัดการนโยบาย NAT และการส่งต่อพอร์ต
- กำหนดค่าการกำหนดเส้นทางตามนโยบาย
- กำหนดเซกเมนต์เครือข่ายและนโยบาย
- กำหนดค่านโยบายพร็อกซีขาออก

## ทรัพยากรหลัก

- **network_policy**: นโยบายความปลอดภัยเครือข่ายสำหรับการกรองทราฟฟิก
- **forward_proxy_policy**: นโยบายพร็อกซีขาออกสำหรับควบคุมทราฟฟิกขาออก
- **network_firewall**: ไฟร์วอลล์เครือข่ายสำหรับการป้องกันทราฟฟิกเลเยอร์ 3/4
