การรวม AWS, Azure, GCP VPC เข้ากับ Transit Gateway การเลือกตาม Label สำหรับการใช้นโยบายข้ามภูมิภาค

- **หมวดหมู่**: โครงสร้างพื้นฐาน
- **ความซับซ้อน**: ขั้นสูง
- **เส้นทาง**: 133 | **สคีมา**: 1015
- **ระดับ**: มาตรฐาน
- **โดเมนที่เกี่ยวข้อง**: `cloud_infrastructure`, `customer_edge`, `managed_kubernetes`

## กรณีการใช้งาน

- ปรับใช้ F5 XC ข้ามผู้ให้บริการคลาวด์ (AWS, Azure, GCP)
- จัดการการปรับใช้ไซต์ XCKS (Managed Kubernetes) (เดิมชื่อ AppStack)
- ปรับใช้ไซต์ Secure Mesh สำหรับการปรับใช้ Edge ที่เน้นระบบเครือข่าย
- รวม Kubernetes คลัสเตอร์ภายนอกเป็น Customer Edge
- กำหนดค่าไซต์ AWS VPC, Azure VNet และ GCP VPC
- จัดการ Virtual Site และนโยบายไซต์

## ทรัพยากรหลัก

- **site**: ตำแหน่งการปรับใช้ทางกายภาพหรือบนคลาวด์สำหรับบริการ Edge
- **virtual_site**: การจัดกลุ่มไซต์เชิงตรรกะโดยใช้ Label Selector
- **site_mesh_group**: การกำหนดค่าการเชื่อมต่อ Mesh ระหว่างหลายไซต์
