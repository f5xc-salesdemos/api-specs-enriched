การกระจายทราฟฟิกข้ามภูมิภาคด้วยกฎการเราท์ การตรวจสอบสุขภาพและนโยบาย failover

- **หมวดหมู่**: เครือข่าย
- **ความซับซ้อน**: ขั้นสูง
- **Paths**: 133 | **Schemas**: 903
- **ระดับ**: ขั้นสูง
- **โดเมนที่เกี่ยวข้อง**: `dns`, `service_policy`, `network`

## กรณีการใช้งาน

- กำหนดค่า HTTP/TCP/UDP load balancers
- จัดการ origin pools และบริการ
- กำหนดค่า virtual hosts และการเราท์
- กำหนด rate limiter และนโยบายบริการ
- จัดการการเราท์ตามตำแหน่งทางภูมิศาสตร์
- กำหนดค่านโยบาย proxy และ forwarding
- จัดการการป้องกันมัลแวร์และแคมเปญภัยคุกคาม
- กำหนดค่าการตรวจสอบสุขภาพและการตรวจสอบ endpoint

## ทรัพยากรหลัก

- **http_loadbalancer**: HTTP/HTTPS load balancer เลเยอร์ 7 สำหรับการกระจายทราฟฟิกแอปพลิเคชัน
- **tcp_loadbalancer**: TCP/UDP load balancer เลเยอร์ 4 สำหรับทราฟฟิกโปรโตคอลที่ไม่ใช่ HTTP
- **origin_pool**: กลุ่มเซิร์ฟเวอร์แบ็กเอนด์สำหรับการกระจายทราฟฟิก load balancer
- **healthcheck**: การกำหนดค่าการตรวจสอบสุขภาพสำหรับความพร้อมใช้งานของ origin server
- **app_firewall**: นโยบาย Web Application Firewall สำหรับการป้องกันทราฟฟิก HTTP
- **service_policy**: กฎการควบคุมการเข้าถึงระดับบริการและการจัดการทราฟฟิก
- **malicious_user_detection**: การตรวจจับและบรรเทาพฤติกรรมผู้ใช้ที่เป็นอันตรายโดยอัตโนมัติ
