การกระจายทราฟฟิกข้ามภูมิภาคด้วยกฎการกำหนดเส้นทาง การตรวจสอบสุขภาพและนโยบาย failover

- **หมวดหมู่**: เครือข่าย
- **ความซับซ้อน**: ขั้นสูง
- **Paths**: 133 | **Schemas**: 903
- **ระดับ**: ขั้นสูง
- **โดเมนที่เกี่ยวข้อง**: `dns`, `service_policy`, `network`

## กรณีการใช้งาน

- กำหนดค่า HTTP/TCP/UDP load balancers
- จัดการ origin pools และบริการ
- กำหนดค่า virtual hosts และการกำหนดเส้นทาง
- กำหนด rate limiter และนโยบายบริการ
- จัดการการกำหนดเส้นทางตามตำแหน่งทางภูมิศาสตร์
- กำหนดค่านโยบาย proxy และ forwarding
- จัดการการป้องกันมัลแวร์และแคมเปญภัยคุกคาม
- กำหนดค่าการตรวจสอบสุขภาพและการมอนิเตอร์ endpoint

## ทรัพยากรหลัก

- **http_loadbalancer**: Load balancer HTTP/HTTPS Layer 7 สำหรับการกระจายทราฟฟิกแอปพลิเคชัน
- **tcp_loadbalancer**: Load balancer TCP/UDP Layer 4 สำหรับทราฟฟิกโปรโตคอลที่ไม่ใช่ HTTP
- **origin_pool**: กลุ่มเซิร์ฟเวอร์แบ็กเอนด์สำหรับการกระจายทราฟฟิกของ load balancer
- **healthcheck**: การกำหนดค่าการมอนิเตอร์สุขภาพสำหรับความพร้อมใช้งานของ origin server
- **app_firewall**: นโยบาย Web Application Firewall สำหรับการป้องกันทราฟฟิก HTTP
- **service_policy**: กฎควบคุมการเข้าถึงระดับบริการและการจัดการทราฟฟิก
- **malicious_user_detection**: การตรวจจับและบรรเทาพฤติกรรมผู้ใช้ที่เป็นอันตรายโดยอัตโนมัติ
