คู่มือนี้อธิบายเวิร์กโฟลว์การพัฒนาสำหรับ F5 XC API Enrichment Pipeline

## เริ่มต้นอย่างรวดเร็ว

### การตั้งค่าครั้งแรก

```bash
# Clone the repository
git clone https://github.com/f5xc-salesdemos/api-specs-enriched.git
cd api-specs-enriched

# Create virtual environment and install dependencies
make install

# Install pre-commit hooks
make pre-commit-install

# Download specs and run pipeline
make build

# Preview documentation locally
make serve
```

### การพัฒนาประจำวัน

```bash
# Quick rebuild (uses cached specs)
make rebuild

# Run all quality checks
make pre-commit-run

# Preview documentation
make serve
```

## ภาพรวมสถาปัตยกรรม

### การออกแบบแบบสองโฟลเดอร์

```text
┌─────────────────────┐     ┌─────────────────────────────┐
│   specs/original/   │────▶│   docs/specifications/api/  │
│   (READ-ONLY)       │     │   (GENERATED)               │
│   - Downloaded      │     │   - Domain specs            │
│   - Gitignored      │     │   - Master spec             │
│   - ETag cached     │     │   - GitHub Pages            │
└─────────────────────┘     └─────────────────────────────┘
```

### ขั้นตอนการทำงานของ Pipeline

```text
Download (ETag) → Enrich → Normalize → Merge → Lint → Serve
     │              │          │         │       │
     ▼              ▼          ▼         ▼       ▼
 270 specs     Branding    Fix refs   23 domains  Spectral
             + Grammar    + Types    + Master     rules
```

### โครงสร้างไดเรกทอรี

| ไดเรกทอรี | วัตถุประสงค์ | สถานะใน Git |
|-----------|---------|------------|
| `specs/original/` | สเปคต้นทางจาก F5 | Gitignored |

## รูปแบบเวิร์กโฟลว์

### 1. เวิร์กโฟลว์การค้นพบ

**ข้อกำหนดเบื้องต้น**:
- การเชื่อมต่อ VPN ไปยังสภาพแวดล้อม F5 XC
- ข้อมูลประจำตัว API ที่ถูกต้อง

```bash
# Set credentials
export F5XC_API_URL="https://your-tenant.console.ves.volterra.io/api"
export F5XC_API_TOKEN="your-api-token"

# Run discovery
make discover

# View results
jq '.statistics' specs/discovered/session.json

# Commit for CI/CD
make push-discovery
```

**การค้นพบจะบันทึก**:
- ฟิลด์ที่จำเป็น/ไม่จำเป็นจริง
- ค่า Enum จากการตอบสนองจริง
- ค่าเริ่มต้น
- การตรวจสอบรูปแบบ
- ตัวอย่างการตอบสนอง

### 2. เวิร์กโฟลว์การปล่อยเวอร์ชัน

การปล่อยเวอร์ชันเป็นแบบอัตโนมัติ:

1. กำหนดการรายวัน (6 AM UTC) หรือการ push ไปยัง main จะเรียกใช้เวิร์กโฟลว์
2. การตรวจสอบ ETag จะกำหนดว่าสเปค F5 มีการเปลี่ยนแปลงหรือไม่
3. Pipeline ประมวลผลและเพิ่มประสิทธิภาพสเปค
4. คำนวณเวอร์ชันจาก git tags: `git describe --tags --abbrev=0`
5. สร้าง Direct commit + tag (ไม่มี PR สำหรับการเพิ่มเวอร์ชัน)
6. สร้าง GitHub Release พร้อม changelog
7. ปรับใช้เอกสารไปยัง GitHub Pages

**กฎการเพิ่มเวอร์ชัน**:

| เงื่อนไข | ประเภทการเพิ่ม | ตัวอย่าง |
|-----------|-----------|---------|
| `[major]` ใน commit | Major | 1.0.0 → 2.0.0 |
| `BREAKING CHANGE` ใน commit | Major | 1.0.0 → 2.0.0 |
| เพิ่มสเปคโดเมนใหม่ | Minor | 1.0.0 → 1.1.0 |
| การเปลี่ยนแปลงอื่นๆ | Patch | 1.0.0 → 1.0.1 |

### 3. เวิร์กโฟลว์การพัฒนา

```bash
# Create feature branch
git checkout -b feature/my-change

# Make changes to config or scripts

# Test locally
make pipeline
make lint

# Commit (pre-commit hooks run automatically)
git add .
git commit -m "feat: add new enrichment rule"

# Push and create PR
git push -u origin feature/my-change
gh pr create
```

## การศึกษาเชิงลึกเกี่ยวกับการค้นพบ

### การค้นพบคืออะไร?

การค้นพบจะสำรวจ F5 XC API จริงเพื่อค้นหา:

- **ข้อจำกัดที่ไม่ได้บันทึกไว้**: ฟิลด์ที่จำเป็นซึ่งไม่ได้ระบุไว้ใน OpenAPI
- **ค่า Enum จริง**: ค่าจริงที่พบในการใช้งานจริง
- **พฤติกรรมเริ่มต้น**: ค่าที่เซิร์ฟเวอร์กำหนดเมื่อไม่ได้ระบุฟิลด์
- **รูปแบบการตอบสนอง**: รูปร่างข้อมูลจริง

### การกำหนดค่าการค้นพบ

```yaml
# config/discovery.yaml
discovery:
  exploration:
    namespaces:
      - "system"
      - "shared"
    methods:
      - "GET"
      - "OPTIONS"
    max_endpoints_per_run: 500

  schema_inference:
    sample_size: 3
    detect_patterns: true
    detect_constraints: true
```

### ส่วนขยายการค้นพบ

การค้นพบจะเพิ่มส่วนขยาย `x-discovered-*` ให้กับสเปค:

```json
{
  "properties": {
    "name": {
      "type": "string",
      "x-discovered-required": true,
      "x-discovered-pattern": "^[a-z][a-z0-9-]*$",
      "x-discovered-examples": ["my-app", "prod-lb"]
    }
  }
}
```

## ส่วนขยาย OpenAPI

pipeline การเพิ่มประสิทธิภาพใช้ส่วนขยายของผู้ผลิตเพื่อฝังข้อมูลเมตาการตรวจสอบและค่าเริ่มต้น

### หมวดหมู่ส่วนขยาย

| ส่วนขยาย | ความหมาย | ผลกระทบ |
|-----------|---------|-------------|
| `x-f5xc-server-default` | เซิร์ฟเวอร์ F5 XC API ใช้ค่านี้เมื่อไม่ได้ระบุฟิลด์ | ฟิลด์เป็นแบบเลือกได้; การละเว้นจะสร้างพฤติกรรมเริ่มต้นที่บันทึกไว้ |
| `x-f5xc-recommended-value` | เว็บคอนโซล F5 XC กรอกค่านี้ล่วงหน้าสำหรับทรัพยากรใหม่ | ฟิลด์ไม่มีค่าเริ่มต้นจากเซิร์ฟเวอร์ แต่ค่านี้แสดงถึงการกำหนดค่าทั่วไป |
| `x-f5xc-recommended-oneof-variant` | คอนโซล F5 XC เลือกตัวแปร OneOf นี้ไว้ล่วงหน้า | ระบุตัวเลือกทั่วไปเมื่อมีตัวเลือกที่ไม่สามารถใช้ร่วมกันได้หลายรายการ |
| `x-f5xc-conflicts-with` | แสดงรายการคุณสมบัติอื่นๆ ที่ไม่สามารถใช้ร่วมกับฟิลด์นี้ได้ | คุณสมบัติเป็นส่วนหนึ่งของกลุ่ม OneOf; สามารถระบุได้เพียงหนึ่งในคุณสมบัติที่ขัดแย้งกัน |

### ส่วนขยายฟิลด์ที่จำเป็น

| ส่วนขยาย | แหล่งที่มา | วัตถุประสงค์ |
|-----------|--------|---------|
| `x-ves-required: "true"` | สเปคต้นฉบับ F5 XC | ฟิลด์ต้องการค่าที่ไม่ใช่ศูนย์/ไม่ว่างเปล่า |
| `x-f5xc-required-for` | pipeline การเพิ่มประสิทธิภาพ | สถานะที่จำเป็นตามบริบท |

**บริบทของ x-f5xc-required-for**:
- `create`: จำเป็นเมื่อสร้างทรัพยากร
- `update`: จำเป็นเมื่ออัปเดตทรัพยากร
- `minimum_config`: จำเป็นสำหรับการกำหนดค่าขั้นต่ำที่ใช้งานได้

### ส่วนขยายค่าเริ่มต้นและ OneOf

| ส่วนขยาย | วัตถุประสงค์ |
|-----------|---------|
| `x-f5xc-server-default: true` | ระบุค่าเริ่มต้นที่เซิร์ฟเวอร์กำหนด |
| `x-f5xc-recommended-value` | ค่าที่กรอกล่วงหน้าโดยคอนโซล F5 XC |
| `x-f5xc-recommended-oneof-variant` | ตัวแปร OneOf ที่แนะนำ |
| `x-f5xc-conflicts-with` | การยกเว้นร่วมกันกับคุณสมบัติ OneOf อื่นๆ |

#### x-f5xc-server-default

**ประเภท**: `boolean`

เมื่อเป็น `true` แสดงว่าค่า `default` ที่มาพร้อมกันนั้นถูกบังคับใช้โดยเซิร์ฟเวอร์ F5 XC API ฟิลด์ที่มีส่วนขยายนี้สามารถละเว้นจากคำขอ API ได้อย่างปลอดภัย โดยเซิร์ฟเวอร์จะใช้ค่าเริ่มต้นโดยอัตโนมัติ

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

#### x-f5xc-recommended-value

**ประเภท**: `any` (ตรงกับประเภทฟิลด์)

ระบุค่าที่เว็บคอนโซล F5 XC ใช้เป็นค่าเริ่มต้นที่กรอกไว้ล่วงหน้า ค่านี้ไม่ได้บังคับใช้โดยเซิร์ฟเวอร์ แต่แสดงถึงการกำหนดค่าเริ่มต้นทั่วไปสำหรับทรัพยากรใหม่ที่สร้างผ่านคอนโซล

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

#### x-f5xc-recommended-oneof-variant

**ประเภท**: `object` (แผนที่ชื่อกลุ่มไปยังชื่อตัวแปร)

สำหรับสคีมาที่มีกลุ่มฟิลด์ที่ไม่สามารถใช้ร่วมกันได้ จะระบุตัวแปรที่เป็นตัวเลือกเริ่มต้นหรือพบบ่อยที่สุด คีย์คือชื่อกลุ่ม OneOf และค่าคือชื่อฟิลด์ตัวแปรที่แนะนำ

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

#### x-f5xc-conflicts-with

**ประเภท**: `array` ของ strings

**เพิ่มใน**: v3.2.0 (Issue #494)

ประกาศความสัมพันธ์การยกเว้นร่วมกันระหว่างสมาชิกกลุ่ม OneOf ได้รับการสร้างโดยอัตโนมัติจากส่วนขยาย `x-ves-oneof-field-*` ซึ่งช่วยให้เครื่องมือปลายทาง (Terraform, CLI, MCP) ตรวจสอบความขัดแย้งในระดับสคีมาแทนที่จะเป็นขณะรันไทม์

```yaml
host_header:
  type: string
  x-f5xc-conflicts-with: ["use_origin_server_name"]

use_origin_server_name:
  type: object
  x-f5xc-conflicts-with: ["host_header"]
```

**กรณีการใช้งาน**:
- Terraform providers สามารถสร้างกฎการตรวจสอบ
- เครื่องมือ CLI สามารถแจ้งเตือนเกี่ยวกับการรวมฟิลด์ที่ขัดแย้งกัน
- ผู้ช่วย AI สามารถสร้างการกำหนดค่า OneOf ที่ถูกต้อง
- ส่วนขยาย IDE สามารถให้การเติมโค้ดอัตโนมัติที่รับรู้ความขัดแย้ง

**แหล่งที่มา**: สร้างโดยอัตโนมัติจากส่วนขยาย F5 native `x-ves-oneof-field-*` ในระหว่างการเพิ่มประสิทธิภาพ pipeline

### ผลกระทบของกฎการตรวจสอบ

ตัวเพิ่มประสิทธิภาพตรวจสอบ `x-ves-validation-rules` เพื่ออนุมานสถานะที่จำเป็น:

| กฎ | ผลกระทบ |
|------|-------------|
| `ves.io.schema.rules.message.required: "true"` | ฟิลด์จำเป็น |
| `ves.io.schema.rules.uint32.gte: N` | ถ้า N >= 1 และไม่มีค่าเริ่มต้นจากเซิร์ฟเวอร์ ฟิลด์จำเป็น |
| `ves.io.schema.rules.repeated.min_items: N` | ถ้า N >= 1 อาร์เรย์ต้องมีอย่างน้อย N รายการ |
| `ves.io.schema.rules.string.min_bytes: N` | ถ้า N >= 1 สตริงต้องมีอย่างน้อย N bytes |

### ทรัพยากรที่มีค่าเริ่มต้นที่เซิร์ฟเวอร์กำหนด

ทรัพยากรบางรายการยอมรับสเปคว่างเปล่าเนื่องจากเซิร์ฟเวอร์กำหนดค่าเริ่มต้น:

| ทรัพยากร | ค่าเริ่มต้นที่เซิร์ฟเวอร์กำหนด |
|----------|------------------------|
| `app_firewall` | `monitoring: \{\}`, `default_detection_settings: \{\}` |
| `rate_limiter` | `limits: []`, `user_identification: []` |
| `api_definition` | `swagger_specs: []`, default `api_groups` |
| `healthcheck` | `jitter: 0`, `jitter_percent: 0`, nested http_health_check defaults |

สำหรับทรัพยากรเหล่านี้ `x-f5xc-required-for.create` อาจเป็น `false` แม้ว่า `x-ves-required` จะเป็น `true`

### รูปแบบค่าเริ่มต้น

| รูปแบบ | ประเภท | ตัวอย่าง |
|---------|------|---------|
| `\{\}` | อ็อบเจกต์ว่างเปล่า (การเลือกตัวเลือก) | `monitoring: \{\}` |
| `[]` | อาร์เรย์ว่างเปล่า | `expected_status_codes: []` |
| `0` | ตัวเลข | `jitter: 0` |
| `""` | สตริง | `expected_response: ""` |
| `false` | Boolean | `use_http2: false` |

### การเพิ่มค่าเริ่มต้นที่ค้นพบ

1. สร้างทรัพยากรใน F5 XC ด้วยสเปคขั้นต่ำผ่าน API
2. อ่านทรัพยากรกลับมาเพื่อดูค่าที่เซิร์ฟเวอร์กำหนด
3. บันทึกค่าเริ่มต้นใน `config/discovered_defaults.yaml`
4. รัน pipeline: `make pipeline`
5. ตรวจสอบด้วย: `jq '.components.schemas | to_entries[] | select(.key | contains("resource_name"))'`

แก้ไขปัญหาในการกำหนดค่าการเพิ่มประสิทธิภาพ/การทำให้เป็นมาตรฐาน ไม่ใช่ไฟล์ผลลัพธ์

### ระบบเวอร์ชัน

เวอร์ชันได้รับมาจาก git tags (เช่น `v2.0.38`) ซึ่งขจัดสภาวะแข่งขันที่ทำให้เกิดความขัดแย้งในการ merge บน PR ที่ทำงานพร้อมกัน

```python
# Version is calculated dynamically from git tags
from scripts.utils.version_calculator import get_version_from_tags
version = get_version_from_tags()  # Returns "2.0.38"
```

### สเปคหายไปหลัง Clone

สเปคที่ถูกสร้างขึ้นถูก gitignored:

```bash
make build  # Downloads and generates everything
```

### ไฟล์ขนาดใหญ่ถูกบล็อก

เพิ่มการยกเว้นใน `.pre-commit-config.yaml`:

```yaml
- id: check-added-large-files
  exclude: ^path/to/large/file\.json$
```

## อ้างอิงด่วน

### เป้าหมาย Make

| เป้าหมาย | คำอธิบาย |
|--------|-------------|
| `make build` | Build แบบเต็ม (ดาวน์โหลด + pipeline) |
| `make rebuild` | Build ด่วน (ข้ามการดาวน์โหลด) |
| `make download` | ดาวน์โหลดสเปค (แคช ETag) |
| `make download-force` | บังคับดาวน์โหลด |
| `make pipeline` | รัน pipeline การเพิ่มประสิทธิภาพ |
| `make lint` | Spectral linting |
| `make serve` | เซิร์ฟเวอร์เอกสารในเครื่อง |
| `make discover` | การค้นพบ API (ต้องใช้ VPN) |
| `make push-discovery` | Commit ข้อมูลการค้นพบ |
| `make clean` | ลบไฟล์ที่ถูกสร้างขึ้น |
| `make pre-commit-run` | รันการตรวจสอบคุณภาพทั้งหมด |

### ตัวแปรสภาพแวดล้อม

| ตัวแปร | วัตถุประสงค์ |
|----------|---------|
| `F5XC_API_URL` | URL ของ API สำหรับ tenant F5 XC |
| `F5XC_API_TOKEN` | โทเค็นการยืนยันตัวตน API |

### ไฟล์สำคัญ

| ไฟล์ | วัตถุประสงค์ |
|------|---------|
| `.etag` | ETag ที่ดาวน์โหลดล่าสุด |
| `CHANGELOG.md` | Changelog ที่สร้างโดยอัตโนมัติ |
| `config/enrichment.yaml` | กฎการเพิ่มประสิทธิภาพ |
| `config/normalization.yaml` | กฎการทำให้เป็นมาตรฐาน |
| `config/discovery.yaml` | การตั้งค่าการค้นพบ |
| `config/spectral.yaml` | กฎ Linting |
| `scripts/utils/version_calculator.py` | การคำนวณเวอร์ชันจาก tag |

**หมายเหตุ**: เวอร์ชันได้รับมาจาก git tags (เช่น `v2.0.38`) ไม่ใช่จากไฟล์ `.version`
