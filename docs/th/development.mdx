คู่มือนี้อธิบายเวิร์กโฟลว์การพัฒนาสำหรับ F5 XC API Enrichment Pipeline

## เริ่มต้นอย่างรวดเร็ว

### การตั้งค่าครั้งแรก

```bash
# Clone the repository
git clone https://github.com/f5xc-salesdemos/api-specs-enriched.git
cd api-specs-enriched

# Create virtual environment and install dependencies
make install

# Install pre-commit hooks
make pre-commit-install

# Download specs and run pipeline
make build

# Preview documentation locally
make serve
```

### การพัฒนาประจำวัน

```bash
# Quick rebuild (uses cached specs)
make rebuild

# Run all quality checks
make pre-commit-run

# Preview documentation
make serve
```

## รูปแบบเวิร์กโฟลว์

### 1. เวิร์กโฟลว์การค้นหา

**ข้อกำหนดเบื้องต้น**:
- การเชื่อมต่อ VPN ไปยังสภาพแวดล้อม F5 XC
- ข้อมูลรับรอง API ที่ถูกต้อง

```bash
# Set credentials
export F5XC_API_URL="https://your-tenant.console.ves.volterra.io/api"
export F5XC_API_TOKEN="your-api-token"

# Run discovery
make discover

# View results
jq '.statistics' specs/discovered/session.json

# Commit for CI/CD
make push-discovery
```

**การค้นหาจะบันทึก**:
- ฟิลด์ที่จำเป็น/ไม่จำเป็นจริง
- ค่า enum จากการตอบสนองจริง
- ค่าเริ่มต้น
- การตรวจสอบรูปแบบ
- ตัวอย่างการตอบสนอง

### 2. เวิร์กโฟลว์การเผยแพร่

การเผยแพร่เป็นแบบอัตโนมัติ:

1. กำหนดการรายวัน (6 AM UTC) หรือ push ไปยัง main จะทริกเกอร์เวิร์กโฟลว์
2. การตรวจสอบ ETag จะพิจารณาว่าสเปค F5 มีการเปลี่ยนแปลงหรือไม่
3. Pipeline ประมวลผลและเพิ่มคุณค่าสเปค
4. เวอร์ชันคำนวณจาก git tags: `git describe --tags --abbrev=0`
5. สร้าง direct commit + tag (ไม่มี PR สำหรับ version bump)
6. สร้าง GitHub Release พร้อม changelog
7. ปรับใช้เอกสารไปยัง GitHub Pages

**กฎการ Bump เวอร์ชัน**:

| เงื่อนไข | ประเภท Bump | ตัวอย่าง |
|-----------|-----------|---------|
| `[major]` ใน commit | Major | 1.0.0 → 2.0.0 |
| `BREAKING CHANGE` ใน commit | Major | 1.0.0 → 2.0.0 |
| เพิ่มสเปคโดเมนใหม่ | Minor | 1.0.0 → 1.1.0 |
| การเปลี่ยนแปลงอื่น ๆ | Patch | 1.0.0 → 1.0.1 |

### 3. เวิร์กโฟลว์การพัฒนา

```bash
# Create feature branch
git checkout -b feature/my-change

# Make changes to config or scripts

# Test locally
make pipeline
make lint

# Commit (pre-commit hooks run automatically)
git add .
git commit -m "feat: add new enrichment rule"

# Push and create PR
git push -u origin feature/my-change
gh pr create
```

## การค้นหาเชิงลึก

### การค้นหาคืออะไร?

การค้นหาจะสำรวจ F5 XC API ที่ทำงานจริงเพื่อค้นหา:

- **ข้อจำกัดที่ไม่ได้ระบุในเอกสาร**: ฟิลด์ที่จำเป็นซึ่งไม่ได้ทำเครื่องหมายใน OpenAPI
- **ค่า enum จริง**: ค่าจริงที่พบในสภาพแวดล้อมการผลิต
- **พฤติกรรมค่าเริ่มต้น**: ค่าที่เซิร์ฟเวอร์กำหนดเมื่อละเว้นฟิลด์
- **รูปแบบการตอบสนอง**: รูปร่างข้อมูลจริง

### การตั้งค่าการค้นหา

```yaml
# config/discovery.yaml
discovery:
  exploration:
    namespaces:
      - "system"
      - "shared"
    methods:
      - "GET"
      - "OPTIONS"
    max_endpoints_per_run: 500

  schema_inference:
    sample_size: 3
    detect_patterns: true
    detect_constraints: true
```

### ส่วนขยายการค้นหา

การค้นหาจะเพิ่มส่วนขยาย `x-discovered-*` ลงในสเปค:

```json
{
  "properties": {
    "name": {
      "type": "string",
      "x-discovered-required": true,
      "x-discovered-pattern": "^[a-z][a-z0-9-]*$",
      "x-discovered-examples": ["my-app", "prod-lb"]
    }
  }
}
```

## ส่วนขยาย OpenAPI

Enrichment pipeline ใช้ส่วนขยายของผู้ผลิตเพื่อฝังข้อมูลเมตาสำหรับการตรวจสอบความถูกต้องและค่าเริ่มต้น

### หมวดหมู่ส่วนขยาย

| ส่วนขยาย | ความหมาย | ผลที่ตามมา |
|-----------|---------|-------------|
| `x-f5xc-server-default` | เซิร์ฟเวอร์ F5 XC API จะกำหนดค่านี้เมื่อละเว้นฟิลด์ | ฟิลด์เป็นทางเลือก; การละเว้นจะทำให้เกิดพฤติกรรมค่าเริ่มต้นตามที่ระบุไว้ |
| `x-f5xc-recommended-value` | คอนโซลเว็บ F5 XC จะเติมค่านี้ล่วงหน้าสำหรับทรัพยากรใหม่ | ฟิลด์ไม่มีค่าเริ่มต้นจากเซิร์ฟเวอร์แต่ค่านี้แสดงถึงการตั้งค่าทั่วไป |
| `x-f5xc-recommended-oneof-variant` | คอนโซล F5 XC จะเลือก OneOf variant นี้ล่วงหน้า | ระบุตัวเลือกทั่วไปเมื่อมีตัวเลือกที่แยกกันหลายตัว |
| `x-f5xc-conflicts-with` | แสดงรายการคุณสมบัติอื่นที่ไม่สามารถใช้ร่วมกับฟิลด์นี้ได้ | คุณสมบัติเป็นส่วนหนึ่งของกลุ่ม OneOf; สามารถระบุได้เพียงหนึ่งในคุณสมบัติที่ขัดแย้งกัน |

### ส่วนขยายฟิลด์ที่จำเป็น

| ส่วนขยาย | แหล่งที่มา | วัตถุประสงค์ |
|-----------|--------|---------|
| `x-ves-required: "true"` | สเปคต้นฉบับ F5 XC | ฟิลด์ต้องมีค่าที่ไม่ใช่ศูนย์/ไม่ว่าง |
| `x-f5xc-required-for` | Enrichment pipeline | สถานะจำเป็นตามบริบท |

**บริบทของ x-f5xc-required-for**:
- `create`: จำเป็นเมื่อสร้างทรัพยากร
- `update`: จำเป็นเมื่ออัปเดตทรัพยากร
- `minimum_config`: จำเป็นสำหรับการตั้งค่าขั้นต่ำที่ใช้งานได้

### ส่วนขยายค่าเริ่มต้นและ OneOf

| ส่วนขยาย | วัตถุประสงค์ |
|-----------|---------|
| `x-f5xc-server-default: true` | ทำเครื่องหมายค่าเริ่มต้นที่เซิร์ฟเวอร์กำหนด |
| `x-f5xc-recommended-value` | ค่าที่เติมล่วงหน้าโดยคอนโซล F5 XC |
| `x-f5xc-recommended-oneof-variant` | OneOf variant ที่แนะนำ |
| `x-f5xc-conflicts-with` | ความเป็นเอกสิทธิ์ร่วมกันกับคุณสมบัติ OneOf อื่น |

#### x-f5xc-server-default

**ชนิด**: `boolean`

เมื่อเป็น `true` หมายความว่าค่า `default` ที่แนบมาถูกบังคับใช้โดยเซิร์ฟเวอร์ F5 XC API ฟิลด์ที่มีส่วนขยายนี้สามารถละเว้นจากคำขอ API ได้อย่างปลอดภัย — เซิร์ฟเวอร์จะกำหนดค่าเริ่มต้นโดยอัตโนมัติ

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

#### x-f5xc-recommended-value

**ชนิด**: `any` (ตรงกับชนิดของฟิลด์)

ระบุค่าที่คอนโซลเว็บ F5 XC ใช้เป็นค่าเริ่มต้นที่เติมล่วงหน้า ค่านี้ไม่ได้ถูกบังคับใช้โดยเซิร์ฟเวอร์แต่แสดงถึงการตั้งค่าเริ่มต้นทั่วไปสำหรับทรัพยากรใหม่ที่สร้างผ่านคอนโซล

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

#### x-f5xc-recommended-oneof-variant

**ชนิด**: `object` (แมปของชื่อกลุ่มไปยังชื่อ variant)

สำหรับสคีมาที่มีกลุ่มฟิลด์ที่แยกกัน ระบุว่า variant ใดเป็นค่าเริ่มต้นหรือเป็นตัวเลือกที่พบบ่อยที่สุด คีย์คือชื่อกลุ่ม OneOf และค่าคือชื่อฟิลด์ variant ที่แนะนำ

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

#### x-f5xc-conflicts-with

**ชนิด**: `array` ของ strings

**เพิ่มเมื่อ**: v3.2.0 (Issue #494)

ประกาศความสัมพันธ์แบบเอกสิทธิ์ร่วมกันระหว่างสมาชิกกลุ่ม OneOf สร้างโดยอัตโนมัติจากส่วนขยาย `x-ves-oneof-field-*` ซึ่งช่วยให้เครื่องมือปลายทาง (Terraform, CLI, MCP) สามารถตรวจสอบความขัดแย้งที่ระดับสคีมาแทนที่จะเป็นขณะรันไทม์

```yaml
host_header:
  type: string
  x-f5xc-conflicts-with: ["use_origin_server_name"]

use_origin_server_name:
  type: object
  x-f5xc-conflicts-with: ["host_header"]
```

**กรณีการใช้งาน**:
- Terraform providers สามารถสร้างกฎการตรวจสอบได้
- เครื่องมือ CLI สามารถเตือนเกี่ยวกับการรวมฟิลด์ที่ขัดแย้งกัน
- ผู้ช่วย AI สามารถสร้างการตั้งค่า OneOf ที่ถูกต้อง
- ส่วนขยาย IDE สามารถให้การเติมข้อมูลอัตโนมัติที่ตระหนักถึงความขัดแย้ง

**แหล่งที่มา**: สร้างโดยอัตโนมัติจากส่วนขยายดั้งเดิมของ F5 `x-ves-oneof-field-*` ในระหว่างการเพิ่มคุณค่าของ pipeline

### ผลที่ตามมาของกฎการตรวจสอบ

Enricher ตรวจสอบ `x-ves-validation-rules` เพื่ออนุมานสถานะจำเป็น:

| กฎ | ผลที่ตามมา |
|------|-------------|
| `ves.io.schema.rules.message.required: "true"` | ฟิลด์เป็นฟิลด์จำเป็น |
| `ves.io.schema.rules.uint32.gte: N` | หาก N >= 1 และไม่มีค่าเริ่มต้นจากเซิร์ฟเวอร์ ฟิลด์เป็นฟิลด์จำเป็น |
| `ves.io.schema.rules.repeated.min_items: N` | หาก N >= 1 อาร์เรย์ต้องมีอย่างน้อย N รายการ |
| `ves.io.schema.rules.string.min_bytes: N` | หาก N >= 1 สตริงต้องมีอย่างน้อย N ไบต์ |

### ทรัพยากรที่มีค่าเริ่มต้นจากเซิร์ฟเวอร์

ทรัพยากรบางอย่างยอมรับสเปคว่างเพราะเซิร์ฟเวอร์จะกำหนดค่าเริ่มต้น:

| ทรัพยากร | ค่าเริ่มต้นที่เซิร์ฟเวอร์กำหนด |
|----------|------------------------|
| `app_firewall` | `monitoring: \{\}`, `default_detection_settings: \{\}` |
| `rate_limiter` | `limits: []`, `user_identification: []` |
| `api_definition` | `swagger_specs: []`, `api_groups` เริ่มต้น |
| `healthcheck` | `jitter: 0`, `jitter_percent: 0`, ค่าเริ่มต้น http_health_check ที่ซ้อนกัน |

สำหรับทรัพยากรเหล่านี้ `x-f5xc-required-for.create` อาจเป็น `false` แม้ว่า `x-ves-required` จะเป็น `true`

### รูปแบบค่าเริ่มต้น

| รูปแบบ | ชนิด | ตัวอย่าง |
|---------|------|---------|
| `\{\}` | ออบเจกต์ว่าง (การเลือกตัวเลือก) | `monitoring: \{\}` |
| `[]` | อาร์เรย์ว่าง | `expected_status_codes: []` |
| `0` | ตัวเลข | `jitter: 0` |
| `""` | สตริง | `expected_response: ""` |
| `false` | บูลีน | `use_http2: false` |

### การเพิ่มค่าเริ่มต้นที่ค้นพบ

1. สร้างทรัพยากรใน F5 XC ด้วยสเปคขั้นต่ำผ่าน API
2. อ่านทรัพยากรกลับมาเพื่อดูค่าที่เซิร์ฟเวอร์กำหนด
3. บันทึกค่าเริ่มต้นใน `config/discovered_defaults.yaml`
4. รัน pipeline: `make pipeline`
5. ตรวจสอบด้วย: `jq '.components.schemas | to_entries[] | select(.key | contains("resource_name"))'`

### ระบบเวอร์ชัน

เวอร์ชันถูกดึงมาจาก git tags (เช่น `v2.0.38`) ซึ่งขจัด race conditions ที่ทำให้เกิด merge conflicts บน PR ที่ทำงานพร้อมกัน

```python
# Version is calculated dynamically from git tags
from scripts.utils.version_calculator import get_version_from_tags
version = get_version_from_tags()  # Returns "2.0.38"
```

### สเปคหายไปหลังจาก Clone

สเปคที่สร้างขึ้นถูก gitignore:

```bash
make build  # Downloads and generates everything
```

### ไฟล์ขนาดใหญ่ถูกบล็อก

เพิ่มข้อยกเว้นใน `.pre-commit-config.yaml`:

```yaml
- id: check-added-large-files
  exclude: ^path/to/large/file\.json$
```

## ข้อมูลอ้างอิงด่วน

### เป้าหมาย Make

| เป้าหมาย | คำอธิบาย |
|--------|-------------|
| `make build` | สร้างทั้งหมด (ดาวน์โหลด + pipeline) |
| `make rebuild` | สร้างใหม่อย่างรวดเร็ว (ข้ามการดาวน์โหลด) |
| `make download` | ดาวน์โหลดสเปค (แคช ETag) |
| `make download-force` | บังคับดาวน์โหลด |
| `make pipeline` | รัน enrichment pipeline |
| `make lint` | Spectral linting |
| `make serve` | เซิร์ฟเวอร์เอกสารในเครื่อง |
| `make discover` | การค้นหา API (ต้องใช้ VPN) |
| `make push-discovery` | Commit ข้อมูลการค้นหา |
| `make clean` | ลบไฟล์ที่สร้างขึ้น |
| `make pre-commit-run` | รันการตรวจสอบคุณภาพทั้งหมด |

### ตัวแปรสภาพแวดล้อม

| ตัวแปร | วัตถุประสงค์ |
|----------|---------|
| `F5XC_API_URL` | URL ของ F5 XC tenant API |
| `F5XC_API_TOKEN` | โทเค็นการยืนยันตัวตน API |

### ไฟล์สำคัญ

| ไฟล์ | วัตถุประสงค์ |
|------|---------|
| `.etag` | ETag ที่ดาวน์โหลดล่าสุด |
