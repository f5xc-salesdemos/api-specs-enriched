origin_pool 相关模式的富化元数据。扩展定义请参阅 [OpenAPI 扩展](../development/#openapi-extensions)。

## 富化模式

### 匹配模式

所有匹配 `origin_pool.*SpecType` 的模式都会接收富化信息：

- `origin_poolCreateSpecType`
- `origin_poolReplaceSpecType`
- `origin_poolGetSpecType`

## 必填字段

| 字段 | 适用操作 | 备注 |
|-------|--------------|-------|
| `metadata.name` | 所有操作 | 1-63 个字符，小写字母数字，模式：`^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |
| `metadata.namespace` | 所有操作 | 命名空间标识符 |
| `spec.origin_servers` | 创建 | 最少 1 项 |
| `spec.port` | 创建 | 1-65535，无服务器默认值 |

## 最小可用配置

```json
{
  "metadata": {
    "name": "my-origin-pool",
    "namespace": "default"
  },
  "spec": {
    "origin_servers": [
      {
        "public_name": {
          "dns_name": "backend.example.com"
        }
      }
    ],
    "port": 443
  }
}
```

## 服务器应用的默认值

标记为 `x-f5xc-server-default: true` 的字段，当请求中省略时，其 `default` 值由 F5 XC API 服务器应用。

### 顶层字段

| 字段 | 默认值 | 类型 | 描述 |
|-------|---------------|------|-------------|
| `no_tls` | `{}` | object | 禁用到源服务器的 TLS |
| `healthcheck` | `[]` | array | 未配置健康检查 |
| `loadbalancer_algorithm` | `ROUND_ROBIN` | enum | 轮询负载均衡 |
| `endpoint_selection` | `DISTRIBUTED` | enum | 使用所有端点（本地 + 远程） |

### advanced_options 默认值

当未指定 `advanced_options` 时，服务器的行为如同设置了以下值：

| 字段 | 默认值 | 类型 | 描述 |
|-------|---------------|------|-------------|
| `connection_timeout` | `2000` | integer | 连接超时时间（毫秒） |
| `http_idle_timeout` | `300000` | integer | HTTP 空闲超时时间（毫秒，5 分钟） |
| `same_as_endpoint_port` | `{}` | object | 健康检查使用端点端口 |
| `default_circuit_breaker` | `{}` | object | 默认熔断器设置 |
| `disable_outlier_detection` | `{}` | object | 禁用异常检测 |
| `no_panic_threshold` | `{}` | object | 无恐慌阈值 |
| `disable_subsets` | `{}` | object | 禁用子集负载均衡 |
| `auto_http_config` | `{}` | object | 自动 HTTP 协议协商 |
| `disable_proxy_protocol` | `{}` | object | 禁用代理协议 |
| `disable_lb_source_ip_persistance` | `{}` | object | 禁用负载均衡器源 IP 持久性 |

### 嵌套对象默认值

| 路径 | 默认值 | 描述 |
|------|---------------|-------------|
| `origin_servers[].labels` | `{}` | 空标签对象 |
| `origin_servers[].public_name.refresh_interval` | `0` | 使用系统默认 DNS 刷新 |

## UI 与服务器默认值的差异

F5 XC Web UI 预选的值与省略字段时 API 应用的值不同。

| 字段 | UI 预选值 | 服务器应用值 | 备注 |
|-------|-----------------|----------------|------|
| `loadbalancer_algorithm` | `LB_OVERRIDE` | `ROUND_ROBIN` | 当字段被省略时，UI 创建和 API 创建的资源有所不同 |

## 枚举值

### loadbalancer_algorithm

| 值 | 描述 | 备注 |
|-------|-------------|-------|
| `ROUND_ROBIN` | 按轮询顺序选择每个健康端点 | 服务器默认值 |
| `LEAST_REQUEST` | 选择活跃请求最少的端点 | |
| `RING_HASH` | 使用端点名称的环形哈希进行一致性哈希 | |
| `RANDOM` | 随机选择健康端点 | |
| `LB_OVERRIDE` | 从父级负载均衡器继承哈希策略 | UI 默认值 |

### endpoint_selection

| 值 | 描述 | 备注 |
|-------|-------------|-------|
| `DISTRIBUTED` | 同时考虑远程和本地端点 | 服务器默认值 |
| `LOCAL_ONLY` | 仅使用本地端点 | |
| `LOCAL_PREFERRED` | 优先本地，不可用时回退到远程 | |

## OneOf 字段组

互斥字段组。每个组中只能指定一个字段：

| 组 | 字段 | 服务器默认值 |
|-------|--------|----------------|
| 端口配置 | `port`, `automatic_port`, `lb_port` | `port`（显式） |
| 到源服务器的 TLS | `no_tls`, `use_tls` | `no_tls` |
| 健康检查端口 | `same_as_endpoint_port`, `health_check_port` | `same_as_endpoint_port` |
| 熔断器 | `default_circuit_breaker`, `disable_circuit_breaker`, `circuit_breaker` | `default_circuit_breaker` |
| 异常检测 | `disable_outlier_detection`, `outlier_detection` | `disable_outlier_detection` |
| 恐慌阈值 | `no_panic_threshold`, `panic_threshold` | `no_panic_threshold` |
| 子集负载均衡 | `disable_subsets`, `enable_subsets` | `disable_subsets` |
| HTTP 协议 | `auto_http_config`, `http1_config`, `http2_options` | `auto_http_config` |
| 代理协议 | `disable_proxy_protocol`, `proxy_protocol_v1`, `proxy_protocol_v2` | `disable_proxy_protocol` |
| 负载均衡器源 IP | `disable_lb_source_ip_persistance`, `enable_lb_source_ip_persistance` | `disable_lb_source_ip_persistance` |
| 连接池 | `enable_conn_pool_reuse`, `disable_conn_pool_reuse` | `enable_conn_pool_reuse` |

## 字段约束

| 字段 | 类型 | 约束 |
|-------|------|------------|
| `spec.port` | integer | 1-65535 |
| `spec.advanced_options.connection_timeout` | integer | 0-1,800,000 毫秒 |
| `spec.advanced_options.http_idle_timeout` | integer | 0-600,000 毫秒 |
| `spec.advanced_options.panic_threshold` | integer | 0-100（百分比） |
| `metadata.name` | string | 1-63 个字符，模式：`^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |


## 源服务器类型

源服务器可以使用不同的类型变体来指定。类型决定了需要哪些额外字段。

### 已实现的类型

| 类型 | 描述 | 必填字段 | 状态 |
|------|-------------|-----------------|--------|
| `public_name` | 具有公共 DNS 名称的源服务器 | `dns_name` | ✅ 已完成 |
| `public_ip` | 具有公共 IP 地址的源服务器 | `ip`（IPv4） | ✅ 已完成 |

### 示例：public_name

```json
{
  "origin_servers": [
    {
      "public_name": {
        "dns_name": "backend.example.com"
      }
    }
  ]
}
```

### 示例：public_ip

```json
{
  "origin_servers": [
    {
      "public_ip": {
        "ip": "8.8.8.8"
      }
    }
  ]
}
```

## 待办：依赖站点的源服务器类型

以下源服务器类型需要在实现之前完成"站点"资源逻辑开发：

| 类型 | 描述 | 必填字段 | 状态 |
|------|-------------|-----------------|--------|
| `private_ip` | 具有私有/公共 IP 和站点信息的源服务器 | `ip`、`site_locator`、`network_choice`、`snat_pool` | 🔲 等待站点逻辑 |
| `private_name` | 具有 DNS 名称和站点信息的源服务器 | `dns_name`、`site_locator`、`network_choice`、`snat_pool` | 🔲 等待站点逻辑 |
| `k8s_service` | 具有站点信息的 Kubernetes 服务 | `service_name`、`site_locator`、`network_choice` | 🔲 等待站点逻辑 |
| `consul_service` | 具有站点信息的 HashiCorp Consul 服务 | `service_name`、`site_locator`、`network_choice` | 🔲 等待站点逻辑 |

### 实现要求

在完整记录这些类型之前，需要：

1. **站点资源发现** - 记录 `site` 资源模式和 site_locator 模式
2. **网络选择枚举** - 记录 `network_choice` 受约束的值（site_local_inside、site_local_outside 等）
3. **SNAT 池选项** - 记录 SNAT 池配置模式
4. **跨资源引用** - 定义 origin_pool 如何引用站点资源
