具有自定义分类规则的敏感数据策略。LMA 区域配置和基于地理位置的合规控制。

- **类别**：安全
- **复杂度**：简单
- **路径数**：11 | **模式数**：65
- **层级**：高级
- **相关领域**：`blindfold`、`client_side_defense`

## 使用场景

- 配置敏感数据检测策略
- 定义用于 PII 分类的自定义数据类型
- 管理 LMA 区域配置
- 集成地理配置以实现合规

## 主要资源

- **sensitive_data_policy**：用于 PII 保护的敏感数据策略
- **data_classification**：用于内容分类的数据分类

## 端点

| 方法 | 路径 | 描述 |
|--------|------|-------------|
| POST | `/api/config/namespaces/&#123;metadata.namespace&#125;/data_types` | 创建数据类型。 |
| PUT | `/api/config/namespaces/&#123;metadata.namespace&#125;/data_types/&#123;metadata.name&#125;` | 替换数据类型。 |
| GET | `/api/config/namespaces/&#123;namespace&#125;/data_types` | 列出数据类型。 |
| GET | `/api/config/namespaces/&#123;namespace&#125;/data_types/&#123;name&#125;` | 获取数据类型。 |
| DELETE | `/api/config/namespaces/&#123;namespace&#125;/data_types/&#123;name&#125;` | 删除数据类型。 |
| GET | `/api/config/namespaces/&#123;namespace&#125;/geo_configs/&#123;name&#125;` | 获取地理配置。 |

| GET | `/api/config/namespaces/&#123;namespace&#125;/lma_regions/&#123;name&#125;` | 获取 LMA 区域。 |
