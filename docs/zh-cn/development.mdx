本指南描述了 F5 XC API 丰富化管道的开发工作流程。

## 快速入门

### 首次安装配置

```bash
# Clone the repository
git clone https://github.com/f5xc-salesdemos/api-specs-enriched.git
cd api-specs-enriched

# Create virtual environment and install dependencies
make install

# Install pre-commit hooks
make pre-commit-install

# Download specs and run pipeline
make build

# Preview documentation locally
make serve
```

### 日常开发

```bash
# Quick rebuild (uses cached specs)
make rebuild

# Run all quality checks
make pre-commit-run

# Preview documentation
make serve
```

## 工作流程模式

### 1. 发现工作流程

**前置条件**：
- 已连接 VPN 至 F5 XC 环境
- 有效的 API 凭证

```bash
# Set credentials
export F5XC_API_URL="https://your-tenant.console.ves.volterra.io/api"
export F5XC_API_TOKEN="your-api-token"

# Run discovery
make discover

# View results
jq '.statistics' specs/discovered/session.json

# Commit for CI/CD
make push-discovery
```

**发现功能捕获以下信息**：
- 实际的必填/可选字段
- 来自实时响应的枚举值
- 默认值
- 模式验证规则
- 响应示例

### 2. 发布工作流程

发布流程已自动化：

1. 每日定时（UTC 早上 6 点）或推送至主分支时触发工作流
2. ETag 检查判断 F5 规范是否已变更
3. 管道处理并丰富化规范
4. 版本号通过 git 标签计算：`git describe --tags --abbrev=0`
5. 直接提交并创建标签（无版本号升级 PR）
6. 创建包含变更日志的 GitHub Release
7. 文档部署至 GitHub Pages

**版本号升级规则**：

| 条件 | 升级类型 | 示例 |
|-----------|-----------|---------|
| 提交中包含 `[major]` | 主版本 | 1.0.0 → 2.0.0 |
| 提交中包含 `BREAKING CHANGE` | 主版本 | 1.0.0 → 2.0.0 |
| 新增域规范 | 次版本 | 1.0.0 → 1.1.0 |
| 其他任何变更 | 补丁版本 | 1.0.0 → 1.0.1 |

### 3. 开发工作流程

```bash
# Create feature branch
git checkout -b feature/my-change

# Make changes to config or scripts

# Test locally
make pipeline
make lint

# Commit (pre-commit hooks run automatically)
git add .
git commit -m "feat: add new enrichment rule"

# Push and create PR
git push -u origin feature/my-change
gh pr create
```

## 发现功能深度解析

### 什么是发现功能？

发现功能通过探索实时 F5 XC API 来获取以下信息：

- **未记录的约束**：未在 OpenAPI 中标注的必填字段
- **实际枚举值**：生产环境中实际出现的值
- **默认行为**：字段省略时服务器自动应用的值
- **响应模式**：实际数据结构

### 发现功能配置

```yaml
# config/discovery.yaml
discovery:
  exploration:
    namespaces:
      - "system"
      - "shared"
    methods:
      - "GET"
      - "OPTIONS"
    max_endpoints_per_run: 500

  schema_inference:
    sample_size: 3
    detect_patterns: true
    detect_constraints: true
```

### 发现功能扩展字段

发现功能会向规范中添加 `x-discovered-*` 扩展字段：

```json
{
  "properties": {
    "name": {
      "type": "string",
      "x-discovered-required": true,
      "x-discovered-pattern": "^[a-z][a-z0-9-]*$",
      "x-discovered-examples": ["my-app", "prod-lb"]
    }
  }
}
```

## OpenAPI 扩展字段

丰富化管道使用供应商扩展字段来嵌入验证规则和默认值元数据。

### 扩展字段分类

| 扩展字段 | 含义 | 使用说明 |
|-----------|---------|-------------|
| `x-f5xc-server-default` | F5 XC API 服务器在字段省略时自动应用该值 | 字段可选；省略后将产生文档中记录的默认行为 |
| `x-f5xc-recommended-value` | F5 XC Web 控制台在创建新资源时预填充该值 | 字段无服务器默认值，但该值代表典型配置 |
| `x-f5xc-recommended-oneof-variant` | F5 XC 控制台预选该 OneOf 变体 | 当存在多个互斥选项时，标识最常用的选择 |
| `x-f5xc-conflicts-with` | 列出不能与此字段同时使用的其他属性 | 属性属于 OneOf 分组；互斥属性中只能指定一个 |

### 必填字段扩展

| 扩展字段 | 来源 | 用途 |
|-----------|--------|---------|
| `x-ves-required: "true"` | F5 XC 原始规范 | 字段需要非零/非空值 |
| `x-f5xc-required-for` | 丰富化管道 | 特定上下文的必填状态 |

**x-f5xc-required-for 上下文说明**：
- `create`：创建资源时必填
- `update`：更新资源时必填
- `minimum_config`：最小可用配置时必填

### 默认值与 OneOf 扩展字段

| 扩展字段 | 用途 |
|-----------|---------|
| `x-f5xc-server-default: true` | 标记服务器应用的默认值 |
| `x-f5xc-recommended-value` | F5 XC 控制台预填充的值 |
| `x-f5xc-recommended-oneof-variant` | 推荐的 OneOf 变体 |
| `x-f5xc-conflicts-with` | 与其他 OneOf 属性的互斥关系 |

#### x-f5xc-server-default

**类型**：`boolean`

当值为 `true` 时，表示附带的 `default` 值由 F5 XC API 服务器强制应用。带有此扩展字段的字段可以安全地从 API 请求中省略——服务器会自动应用默认值。

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

#### x-f5xc-recommended-value

**类型**：`any`（与字段类型匹配）

指定 F5 XC Web 控制台用作预填充默认值的值。该值不由服务器强制执行，但代表通过控制台创建新资源时的典型初始配置。

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

#### x-f5xc-recommended-oneof-variant

**类型**：`object`（组名到变体名的映射）

对于包含互斥字段组的模式，标识哪个变体是默认或最常用的选择。键为 OneOf 组名，值为推荐变体的字段名。

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

#### x-f5xc-conflicts-with

**类型**：字符串 `array`

**新增版本**：v3.2.0（Issue #494）

声明 OneOf 组成员之间的互斥关系。自动从 `x-ves-oneof-field-*` 扩展字段派生而来。这使下游工具（Terraform、CLI、MCP）能够在模式层面验证冲突，而不必等到运行时。

```yaml
host_header:
  type: string
  x-f5xc-conflicts-with: ["use_origin_server_name"]

use_origin_server_name:
  type: object
  x-f5xc-conflicts-with: ["host_header"]
```

**使用场景**：
- Terraform 提供程序可生成验证规则
- CLI 工具可对冲突字段组合发出警告
- AI 助手可生成正确的 OneOf 配置
- IDE 扩展可提供感知冲突的自动补全

**来源**：在管道丰富化过程中，自动从 F5 原生 `x-ves-oneof-field-*` 扩展字段派生。

### 验证规则的含义

丰富化程序会检查 `x-ves-validation-rules` 以推断必填状态：

| 规则 | 含义 |
|------|-------------|
| `ves.io.schema.rules.message.required: "true"` | 字段为必填项 |
| `ves.io.schema.rules.uint32.gte: N` | 若 N >= 1 且无服务器默认值，则字段为必填项 |
| `ves.io.schema.rules.repeated.min_items: N` | 若 N >= 1，数组至少需要 N 个元素 |
| `ves.io.schema.rules.string.min_bytes: N` | 若 N >= 1，字符串至少需要 N 个字节 |

### 具有服务器应用默认值的资源

某些资源可以接受空规范，因为服务器会应用默认值：

| 资源 | 服务器应用的默认值 |
|----------|------------------------|
| `app_firewall` | `monitoring: \{\}`，`default_detection_settings: \{\}` |
| `rate_limiter` | `limits: []`，`user_identification: []` |
| `api_definition` | `swagger_specs: []`，默认 `api_groups` |
| `healthcheck` | `jitter: 0`，`jitter_percent: 0`，嵌套 http_health_check 默认值 |

对于这些资源，即使 `x-ves-required` 为 `true`，`x-f5xc-required-for.create` 也可能为 `false`。

### 默认值模式

| 模式 | 类型 | 示例 |
|---------|------|---------|
| `\{\}` | 空对象（选项选择） | `monitoring: \{\}` |
| `[]` | 空数组 | `expected_status_codes: []` |
| `0` | 数值 | `jitter: 0` |
| `""` | 字符串 | `expected_response: ""` |
| `false` | 布尔值 | `use_http2: false` |

### 添加已发现的默认值

1. 通过 API 使用最小规范在 F5 XC 中创建资源
2. 读取资源以查看服务器应用的值
3. 在 `config/discovered_defaults.yaml` 中记录默认值
4. 运行管道：`make pipeline`
5. 使用以下命令验证：`jq '.components.schemas | to_entries[] | select(.key | contains("resource_name"))'`

## 故障排查

### pre-commit 执行时间过长

管道在每次提交时运行以确保一致性。对于进行中的提交：

```bash
git commit --no-verify -m "WIP: work in progress"
# Run before final commit: make pre-commit-run
```

### 发现功能失败

```bash
# Check VPN
ping your-tenant.console.ves.volterra.io

# Check credentials
echo $F5XC_API_TOKEN | head -c 10

# Check API URL format
echo $F5XC_API_URL
# Format: https://tenant.console.ves.volterra.io/api
```

### 生成规范出现代码检查错误

```bash
# Check lint report
cat reports/lint-report.json | jq '.errors'
```

请在丰富化/规范化配置中修复问题，而不是直接修改输出文件。

### 版本系统

版本号从 git 标签派生（例如 `v2.0.38`），消除了并发 PR 中因竞争条件导致的合并冲突。

```python
# Version is calculated dynamically from git tags
from scripts.utils.version_calculator import get_version_from_tags
version = get_version_from_tags()  # Returns "2.0.38"
```

### 克隆后规范文件缺失

生成的规范文件已被 git 忽略：

```bash
make build  # Downloads and generates everything
```

### 大文件被拦截

在 `.pre-commit-config.yaml` 中添加排除规则：

```yaml
- id: check-added-large-files
  exclude: ^path/to/large/file\.json$
```
