健康檢查相關結構描述的 enrichment metadata。請參閱 [OpenAPI 擴展](../development/#openapi-extensions) 以了解擴展定義。

## 已增強的結構描述

### 模式

所有匹配 `healthcheck.*SpecType` 的結構描述都會接收增強功能：

- `healthcheckCreateSpecType`
- `healthcheckReplaceSpecType`
- `healthcheckGetSpecType`

此外，巢狀結構描述 `healthcheckHttpHealthCheck` 包含 HTTP 健康檢查配置的增強功能。

## 伺服器套用的預設值

標記為 `x-f5xc-server-default: true` 的欄位，其 `default` 值會在請求中省略時由 F5 XC API 伺服器套用。

### 頂層欄位

| 欄位 | 預設值 | 類型 | 說明 |
|-------|--------|------|------|
| `jitter` | `0` | integer | 用於時間隨機化的絕對抖動值 |
| `jitter_percent` | `0` | integer | 用於時間隨機化的百分比抖動值 |

### healthcheckHttpHealthCheck 結構描述

| 欄位 | 預設值 | 類型 | 說明 |
|-------|--------|------|------|
| `use_origin_server_name` | `{}` | object | 用於 Host 標頭的來源伺服器名稱 |
| `headers` | `{}` | object | 自訂標頭 |
| `request_headers_to_remove` | `[]` | array | 從請求中移除的標頭 |
| `use_http2` | `false` | boolean | HTTP/2 支援 |
| `expected_status_codes` | `[]` | array | 接受的狀態碼（空值 = 200-299） |

## 建議值

標記為 `x-f5xc-recommended-value` 的欄位表示 F5 XC 網頁主控台在建立新資源時預先填入的值。

### 頂層欄位

| 欄位 | 建議值 | 類型 | 說明 |
|-------|--------|------|------|
| `timeout` | `3` | integer | 健康檢查逾時時間（秒） |
| `interval` | `15` | integer | 健康檢查之間的間隔時間（秒） |
| `unhealthy_threshold` | `1` | integer | 標記為不健康前的連續失敗次數 |
| `healthy_threshold` | `3` | integer | 標記為健康前的連續成功次數 |
| `jitter_percent` | `30` | integer | 用於生產環境的抖動百分比 |

### healthcheckHttpHealthCheck 結構描述

| 欄位 | 建議值 | 類型 | 說明 |
|-------|--------|------|------|
| `path` | `"/"` | string | 健康檢查端點路徑 |
| `use_http2` | `false` | boolean | HTTP/2 支援設定 |
| `expected_status_codes` | `["200"]` | array | 表示來源伺服器健康的狀態碼 |
| `use_origin_server_name` | `{}` | object | 用於 Host 標頭的來源伺服器名稱 |

## OneOf 變體建議

包含互斥欄位群組（OneOf）的結構描述包括 `x-f5xc-recommended-oneof-variant`，以指示最常使用的變體。此 metadata 根據 F5 XC 主控台預設值和常見使用模式，識別存在多個選項時的典型選擇。

### 頂層 OneOf 群組

| 結構描述 | OneOf 群組 | 建議變體 | 說明 |
|----------|------------|----------|------|
| `healthcheckCreateSpecType` | `health_check` | `http_health_check` | HTTP 健康檢查類型 |
| `healthcheckReplaceSpecType` | `health_check` | `http_health_check` | HTTP 健康檢查類型 |

### healthcheckHttpHealthCheck OneOf 群組

| 結構描述 | OneOf 群組 | 建議變體 | 說明 |
|----------|------------|----------|------|
| `healthcheckHttpHealthCheck` | `host_header_choice` | `use_origin_server_name` | 健康檢查請求的 Host 標頭 |

#### Host 標頭選擇

`host_header_choice` OneOf 群組控制在健康檢查 HTTP 請求中如何指定 Host 標頭。

| 變體 | 類型 | 說明 |
|------|------|------|
| `use_origin_server_name` | `object`（空） | 使用來源伺服器名稱作為 Host 標頭。這是 F5 XC 主控台中的預設選擇。 |
| `host_header` | `string` | 為健康檢查請求指定自訂的 Host 標頭值。 |

**API 結構描述參考**：`x-ves-oneof-field-host_header_choice: ["host_header", "use_origin_server_name"]`

## OpenAPI 擴展參考

這些供應商擴展被加入到標準 OpenAPI 結構描述中，以傳達 F5 XC 特定的預設行為。

### x-f5xc-server-default

**類型**：`boolean`

當值為 `true` 時，表示隨附的 `default` 值由 F5 XC API 伺服器強制執行。具有此擴展的欄位可以安全地從 API 請求中省略——伺服器會自動套用預設值。

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

### x-f5xc-recommended-value

**類型**：`any`（與欄位類型匹配）

指定 F5 XC 網頁主控台用作預先填入預設值的值。此值並非由伺服器強制執行，但代表透過主控台建立新資源時的典型初始配置。

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

### x-f5xc-recommended-oneof-variant

**類型**：`object`（群組名稱到變體名稱的映射）

對於具有互斥欄位群組的結構描述，識別哪個變體是預設或最常見的選擇。鍵為 OneOf 群組名稱，值為建議的變體欄位名稱。

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```
## 資料存取

### OpenAPI 規格

| 檔案 | 內容 |
|------|------|
| `docs/specifications/api/virtual.json` | `healthcheckHttpHealthCheck`、`healthcheckCreateSpecType`、`healthcheckReplaceSpecType`、`healthcheckGetSpecType` |
| `docs/specifications/api/openapi.json` | 包含所有結構描述的合併規格 |

### validation.json 結構

```text
defaults.resources.healthcheck
├── server_applied      # 具有 x-f5xc-server-default: true 的欄位
├── recommended         # 具有 x-f5xc-recommended-value 的欄位
├── oneof_recommended   # OneOf 變體建議
└── nested_recommended  # 巢狀結構描述建議值
```
