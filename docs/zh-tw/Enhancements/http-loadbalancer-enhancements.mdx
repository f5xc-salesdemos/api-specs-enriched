HTTP 負載平衡器結構描述的擴充中繼資料。請參閱 [OpenAPI 擴充](../development/#openapi-extensions) 以取得擴充定義。

## 已擴充的結構描述

### 模式

所有符合 `viewshttp_loadbalancer.*SpecType` 的結構描述都會接收擴充：

- `viewshttp_loadbalancerCreateSpecType`
- `viewshttp_loadbalancerReplaceSpecType`
- `viewshttp_loadbalancerGetSpecType`

## 最小組態中繼資料

`x-f5xc-minimum-configuration` 擴充提供全面的 CLI 中繼資料，用於建立最小可行的 HTTP 負載平衡器。

### 必要欄位

| 欄位 | 限制條件 | 描述 |
|-------|-----------|-------------|
| `metadata.name` | DNS 標籤格式：`[a-z0-9]([-a-z0-9]*[a-z0-9])?` | 資源名稱 |
| `metadata.namespace` | DNS 標籤格式 | 命名空間 |
| `spec.domains` | 陣列，min_items: 1 | 要服務的網域 |

### 負載平衡器類型（必須指定其一）

必須指定下列其中一種負載平衡器類型：

| 變體 | 描述 |
|---------|-------------|
| `spec.http` | 僅 HTTP |
| `spec.https` | HTTPS 搭配手動憑證 |
| `spec.https_auto_cert` | HTTPS 搭配自動憑證 |
| `spec.http_https` | 同時支援 HTTP 和 HTTPS |

**注意**：負載平衡器類型 OneOf 群組被稱為：
- 組態檔案中的 `lb_type`（`config/minimum_configs.yaml` 中的簡短形式）
- OpenAPI 規格原生擴充中的 `loadbalancer_type`（原生欄位名稱）

**API 結構描述參考**：`x-ves-oneof-field-loadbalancer_type: ["http", "https", "https_auto_cert", "http_https"]`

### 最小可行組態

```json
{
  "metadata": {
    "name": "example-app",
    "namespace": "default"
  },
  "spec": {
    "domains": ["example.com"],
    "https_auto_cert": {
      "port": 443,
      "tls_config": {"default_security": {}}
    },
    "advertise_on_public_default_vip": {},
    "routes": [{"prefix": "/", "origin_pool": {"pool_name": "backend-pool"}}]
  }
}
```

## 伺服器套用的預設值

標記為 `x-f5xc-server-default: true` 的欄位，其 `default` 值會在請求中省略時由 F5 XC API 伺服器套用。HTTP 負載平衡器具有大量伺服器套用的預設值，因為大多數安全功能和通訊協定選項預設為停用或安全狀態。

### HTTPS 自動憑證預設值

當指定 `https_auto_cert` 時，伺服器會對省略的欄位套用以下預設值：

| 欄位 | 預設值 | 類型 | 描述 |
|-------|---------------|------|-------------|
| `port` | `443` | integer | HTTPS 監聽埠 |
| `http_redirect` | `false` | boolean | HTTP 到 HTTPS 重新導向 |
| `add_hsts` | `false` | boolean | HTTP 嚴格傳輸安全標頭 |
| `tls_config.default_security` | `{}` | object | TLS 1.2+ 搭配強加密套件 |
| `no_mtls` | `{}` | object | 雙向 TLS 已停用 |
| `default_header` | `{}` | object | 預設伺服器名稱標頭處理 |
| `enable_path_normalize` | `{}` | object | 路徑正規化已啟用 |
| `default_loadbalancer` | `{}` | object | 預設負載平衡器設定 |
| `header_transformation_type.legacy_header_transformation` | `{}` | object | 舊版標頭轉換 |
| `connection_idle_timeout` | `120000` | integer | 連線閒置逾時（毫秒，2 分鐘） |
| `http_protocol_options.http_protocol_enable_v1_v2` | `{}` | object | 已啟用 HTTP/1.1 和 HTTP/2 |
| `coalescing_options.default_coalescing` | `{}` | object | 預設 HTTP/2 連線合併 |

### 安全功能預設值

所有安全功能在省略時預設為停用：

| 欄位 | 預設值 | 描述 |
|-------|---------------|-------------|
| `disable_waf` | `{}` | Web 應用程式防火牆已停用 |
| `disable_bot_defense` | `{}` | 機器人防禦已停用 |
| `disable_rate_limit` | `{}` | 速率限制已停用 |
| `disable_api_discovery` | `{}` | API 探索已停用 |
| `disable_api_testing` | `{}` | API 測試已停用 |
| `disable_api_definition` | `{}` | API 定義已停用 |
| `disable_malware_protection` | `{}` | 惡意軟體防護已停用 |
| `disable_client_side_defense` | `{}` | 用戶端防禦已停用 |
| `disable_ip_reputation` | `{}` | IP 信譽已停用 |
| `disable_threat_mesh` | `{}` | 威脅網格已停用 |
| `disable_malicious_user_detection` | `{}` | 惡意使用者偵測已停用 |

### DDoS 和存取控制預設值

| 欄位 | 預設值 | 描述 |
|-------|---------------|-------------|
| `l7_ddos_protection.mitigation_block` | `{}` | 阻擋 DDoS 流量（預設緩解動作） |
| `l7_ddos_protection.default_rps_threshold` | `{}` | 使用預設 RPS 閾值 |
| `l7_ddos_protection.clientside_action_none` | `{}` | 無用戶端 DDoS 驗證 |
| `l7_ddos_protection.ddos_policy_none` | `{}` | 無 DDoS 原則參考 |
| `no_challenge` | `{}` | 無用戶端挑戰 |
| `user_id_client_ip` | `{}` | 透過用戶端 IP 識別使用者 |
| `disable_trust_client_ip_headers` | `{}` | 不信任用戶端 IP 標頭 |

### 其他伺服器套用的預設值

| 欄位 | 預設值 | 類型 | 描述 |
|-------|---------------|------|-------------|
| `advertise_on_public_default_vip` | `{}` | object | 在公開預設 VIP 上發佈 |
| `round_robin` | `{}` | object | 輪詢負載平衡演算法 |
| `add_location` | `true` | boolean | 在回應中新增位置標頭 |
| `system_default_timeouts` | `{}` | object | 使用系統預設逾時 |
| `service_policies_from_namespace` | `{}` | object | 從命名空間繼承服務原則 |
| `default_sensitive_data_policy` | `{}` | object | 使用預設敏感資料原則 |

## 互斥欄位群組

標記為 `x-f5xc-conflicts-with` 的欄位表示 OneOf 模式。每個群組中只能指定一個欄位。

### 核心組態群組

| 群組名稱 | 欄位 | 描述 |
|-----------|--------|-------------|
| `lb_type` | `http`, `https`, `https_auto_cert`, `http_https` | 負載平衡器通訊協定類型 |

## OneOf 變體建議（未來擴充）

**狀態**：尚未套用至 http_loadbalancer 結構描述。

與包含 `x-f5xc-recommended-oneof-variant` 來指示 OneOf 群組最常見選擇的 healthcheck 不同，http_loadbalancer 目前缺少此擴充。

### 與 Healthcheck 的比較

**Healthcheck**（已實作）：
```yaml
viewshealthcheckCreateSpecType:
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

**HTTP 負載平衡器**（尚未實作）：
```yaml
viewshttp_loadbalancerCreateSpecType:
  x-f5xc-recommended-oneof-variant: null  # Would indicate recommended lb_type variant
```

### 未來實作

要為 http_loadbalancer 新增此擴充：
1. 觀察 F5 XC 主控台的預設選擇（例如，UI 中預先選取的 lb_type）
2. 將組態新增至 `config/discovered_defaults.yaml`
3. 重新執行擴充管線以套用擴充

**新增後的預期結構**：
```yaml
viewshttp_loadbalancerCreateSpecType:
  x-f5xc-recommended-oneof-variant:
    loadbalancer_type: "https_auto_cert"  # Example - requires verification
```

此擴充將使下游工具能夠在向使用者呈現組態選項時預先選取最常用的變體。

## 引導式工作流程整合

擴充管線在規格層級新增 `x-f5xc-guided-workflows` 中繼資料。這些工作流程提供逐步部署指引，供 AI 助理和 CLI 工具使用。

### 部署 HTTP 負載平衡器

一個 5 步驟的工作流程，用於建立具有後端來源集區的完整組態 HTTP 負載平衡器（定義於 `config/guided_workflows.yaml`）：

| 步驟 | 動作 | 資源 | 必要欄位 | 選用 |
|------|--------|----------|-----------------|----------|
| 1 | 建立來源集區 | `origin_pool` | `name`, `origin_servers`, `port` | 否 |
| 2 | 組態健康檢查 | `healthcheck` | `name`, `http_health_check` | 是 |
| 3 | 附加健康檢查 | - | - | 是 |
| 4 | 建立 HTTP 負載平衡器 | `http_loadbalancer` | `name`, `domains`, `http.port` | 否 |
| 5 | 驗證部署 | - | - | 否 |

**先決條件**：目標租戶中的有效命名空間、可透過 IP 或 DNS 連線的後端應用程式、SSL 憑證（HTTPS 選用）。

### 部署含 TLS 的 HTTPS 負載平衡器

一個 7 步驟的工作流程，用於建立具有 SSL/TLS 終止的 HTTPS 負載平衡器：

| 步驟 | 動作 | 資源 | 必要欄位 | 選用 |
|------|--------|----------|-----------------|----------|
| 1 | 上傳 SSL 憑證 | `certificate` | `name`, `certificate_chain`, `private_key` | 否 |
| 2 | 建立來源集區 | `origin_pool` | `name`, `origin_servers` | 否 |
| 3 | 組態健康檢查 | `healthcheck` | - | 是 |
| 4 | 組態 WAF 原則 | `app_firewall` | - | 是 |
| 5 | 建立 HTTPS 負載平衡器 | `http_loadbalancer` | `name`, `domains`, `https.tls_parameters` | 否 |
| 6 | 附加 WAF 原則 | - | - | 是 |
| 7 | 驗證 HTTPS 部署 | - | - | 否 |

**先決條件**：目標租戶中的有效命名空間、SSL 憑證和私密金鑰、可連線的後端應用程式。

### 工作流程使用

下游工具使用這些工作流程引導使用者完成多步驟部署：

- **xcsh CLI**：將工作流程呈現為互動式引導命令（例如 `xcsh deploy http-lb`）
- **VS Code 擴充功能**：在側邊欄面板中顯示工作流程步驟和進度追蹤
- **AI 助理**：依序遵循工作流程步驟，建立每個資源並驗證相依性

## OpenAPI 擴充參考

這些供應商擴充被新增至標準 OpenAPI 結構描述，用以傳達 F5 XC 特定的中繼資料。

### x-f5xc-cli-domain

**類型**：`string`

指示 CLI 和工具組織的領域分類。

```yaml
viewshttp_loadbalancerCreateSpecType:
  type: object
  x-f5xc-cli-domain: "virtual"
```

### x-f5xc-minimum-configuration

**類型**：`object`

提供全面的中繼資料，用於建立最小可行組態。包含：
- 描述
- 具有限制條件的必要欄位
- 互斥群組
- 範例組態（YAML、JSON）
- curl 命令範例

```yaml
viewshttp_loadbalancerCreateSpecType:
  type: object
  x-f5xc-minimum-configuration:
    description: "HTTP/HTTPS load balancer for distributing traffic across origin pools"
    required_fields:
      - "metadata.name"
      - "metadata.namespace"
      - "spec.domains"
    mutually_exclusive_groups:
      - name: "lb_type"
        fields: ["spec.http", "spec.https", "spec.https_auto_cert", "spec.http_https"]
        reason: "Choose exactly one load balancer type"
    example_yaml: |
      ...
    example_json: |
      ...
    example_curl: |
      ...
```

### x-f5xc-server-default

**類型**：`boolean`

當值為 `true` 時，表示伴隨的 `default` 值由 F5 XC API 伺服器強制執行。具有此擴充的欄位可以安全地從 API 請求中省略——伺服器會自動套用預設值。

```yaml
disable_waf:
  type: object
  default: {}
  x-f5xc-server-default: true
```

### x-f5xc-conflicts-with

**類型**：`array of strings`

列出與目前欄位互斥的欄位名稱，表示 OneOf 模式。

```yaml
advertise_custom:
  type: object
  x-f5xc-conflicts-with:
    - advertise_on_public
    - advertise_on_public_default_vip
    - do_not_advertise
```

### x-f5xc-constraints

**類型**：`object`

提供驗證限制條件，包括陣列大小限制、整數範圍、唯一性要求和探索中繼資料。

```yaml
routes:
  type: array
  x-f5xc-constraints:
    constraintType: "array"
    minItems: 1
    maxItems: 256
    uniqueItems: false
    metadata:
      source: "discovery"
      confidence: 0.99
      validatedAt: "2026-01-19T12:00:00Z"
```

```yaml
port:
  type: integer
  x-f5xc-constraints:
    constraintType: "number"
    minimum: 1
    maximum: 65535
    metadata:
      source: "discovery"
      confidence: 0.99
```

### x-f5xc-description-short 和 x-f5xc-description-medium

**類型**：`string`

為不同使用情境提供替代描述長度：
- `x-f5xc-description-short`：最多 60 個字元（CLI 欄位、標記）
- `x-f5xc-description-medium`：最多 150 個字元（工具提示、摘要）

```yaml
domains:
  type: array
  description: "List of domains the load balancer will serve"
  x-f5xc-description-short: "Domains to serve"
  x-f5xc-description-medium: "List of domains for which the load balancer accepts traffic"
```

### x-f5xc-example

**類型**：`any`（符合欄位類型）

為文件和工具提供具體的欄位範例。

```yaml
name:
  type: string
  x-f5xc-example: "example-app"

domains:
  type: array
  x-f5xc-example: ["example.com", "www.example.com"]
```

### x-f5xc-required-for

**類型**：`array of strings`

使用旗標指示特定情境的要求：`minimum_config`、`create`、`update`、`read`。

```yaml
metadata.name:
  type: string
  x-f5xc-required-for: ["minimum_config", "create"]

metadata.uid:
  type: string
  x-f5xc-required-for: ["read"]
```
