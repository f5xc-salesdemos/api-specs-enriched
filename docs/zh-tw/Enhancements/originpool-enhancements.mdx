origin_pool 相關結構描述的富化中繼資料。請參閱 [OpenAPI 擴充](../development/#openapi-extensions) 以了解擴充定義。

## 富化結構描述

### 模式

所有符合 `origin_pool.*SpecType` 的結構描述都會接收富化資料：

- `origin_poolCreateSpecType`
- `origin_poolReplaceSpecType`
- `origin_poolGetSpecType`

## 必填欄位

| 欄位 | 適用操作 | 備註 |
|-------|--------------|-------|
| `metadata.name` | 所有操作 | 1-63 字元，小寫英數字，模式：`^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |
| `metadata.namespace` | 所有操作 | 命名空間識別碼 |
| `spec.origin_servers` | 建立 | 最少 1 個項目 |
| `spec.port` | 建立 | 1-65535，無伺服器預設值 |

## 最小可行組態

```json
{
  "metadata": {
    "name": "my-origin-pool",
    "namespace": "default"
  },
  "spec": {
    "origin_servers": [
      {
        "public_name": {
          "dns_name": "backend.example.com"
        }
      }
    ],
    "port": 443
  }
}
```

## 伺服器套用的預設值

標記為 `x-f5xc-server-default: true` 的欄位，當請求中省略時，其 `default` 值會由 F5 XC API 伺服器套用。

### 頂層欄位

| 欄位 | 預設值 | 類型 | 描述 |
|-------|---------------|------|-------------|
| `no_tls` | `{}` | 物件 | 停用至來源的 TLS |
| `healthcheck` | `[]` | 陣列 | 未設定健康檢查 |
| `loadbalancer_algorithm` | `ROUND_ROBIN` | 列舉 | 輪詢負載平衡 |
| `endpoint_selection` | `DISTRIBUTED` | 列舉 | 使用所有端點（本地 + 遠端） |

### advanced_options 預設值

當未指定 `advanced_options` 時，伺服器的行為如同設定了以下值：

| 欄位 | 預設值 | 類型 | 描述 |
|-------|---------------|------|-------------|
| `connection_timeout` | `2000` | 整數 | 連線逾時（毫秒） |
| `http_idle_timeout` | `300000` | 整數 | HTTP 閒置逾時（毫秒，5 分鐘） |
| `same_as_endpoint_port` | `{}` | 物件 | 健康檢查使用端點連接埠 |
| `default_circuit_breaker` | `{}` | 物件 | 預設斷路器設定 |
| `disable_outlier_detection` | `{}` | 物件 | 停用異常值偵測 |
| `no_panic_threshold` | `{}` | 物件 | 無恐慌閾值 |
| `disable_subsets` | `{}` | 物件 | 停用子集負載平衡 |
| `auto_http_config` | `{}` | 物件 | 自動 HTTP 協定協商 |
| `disable_proxy_protocol` | `{}` | 物件 | 停用代理協定 |
| `disable_lb_source_ip_persistance` | `{}` | 物件 | 停用 LB 來源 IP 持久性 |

### 巢狀物件預設值

| 路徑 | 預設值 | 描述 |
|------|---------------|-------------|
| `origin_servers[].labels` | `{}` | 空標籤物件 |
| `origin_servers[].public_name.refresh_interval` | `0` | 使用系統預設 DNS 重新整理 |

## UI 與伺服器預設值差異

F5 XC 網頁 UI 預先選取的值與省略欄位時 API 套用的值不同。

| 欄位 | UI 預先選取 | 伺服器套用 | 備註 |
|-------|-----------------|----------------|------|
| `loadbalancer_algorithm` | `LB_OVERRIDE` | `ROUND_ROBIN` | 當欄位被省略時，UI 建立和 API 建立的資源會不同 |

## 列舉值

### loadbalancer_algorithm

| 值 | 描述 | 備註 |
|-------|-------------|-------|
| `ROUND_ROBIN` | 以輪詢順序選取每個健康端點 | 伺服器預設值 |
| `LEAST_REQUEST` | 選取活躍請求最少的端點 | |
| `RING_HASH` | 使用端點名稱的環形雜湊進行一致性雜湊 | |
| `RANDOM` | 隨機選取健康端點 | |
| `LB_OVERRIDE` | 從父負載平衡器繼承雜湊策略 | UI 預設值 |

### endpoint_selection

| 值 | 描述 | 備註 |
|-------|-------------|-------|
| `DISTRIBUTED` | 同時考慮遠端和本地端點 | 伺服器預設值 |
| `LOCAL_ONLY` | 僅使用本地端點 | |
| `LOCAL_PREFERRED` | 優先使用本地，不可用時退回遠端 | |

## OneOf 欄位群組

互斥欄位群組。每個群組只能指定其中一個欄位：

| 群組 | 欄位 | 伺服器預設值 |
|-------|--------|----------------|
| 連接埠組態 | `port`, `automatic_port`, `lb_port` | `port`（明確指定） |
| 至來源的 TLS | `no_tls`, `use_tls` | `no_tls` |
| 健康檢查連接埠 | `same_as_endpoint_port`, `health_check_port` | `same_as_endpoint_port` |
| 斷路器 | `default_circuit_breaker`, `disable_circuit_breaker`, `circuit_breaker` | `default_circuit_breaker` |
| 異常值偵測 | `disable_outlier_detection`, `outlier_detection` | `disable_outlier_detection` |
| 恐慌閾值 | `no_panic_threshold`, `panic_threshold` | `no_panic_threshold` |
| 子集 LB | `disable_subsets`, `enable_subsets` | `disable_subsets` |
| HTTP 協定 | `auto_http_config`, `http1_config`, `http2_options` | `auto_http_config` |
| 代理協定 | `disable_proxy_protocol`, `proxy_protocol_v1`, `proxy_protocol_v2` | `disable_proxy_protocol` |
| LB 來源 IP | `disable_lb_source_ip_persistance`, `enable_lb_source_ip_persistance` | `disable_lb_source_ip_persistance` |
| 連線池 | `enable_conn_pool_reuse`, `disable_conn_pool_reuse` | `enable_conn_pool_reuse` |

## 欄位約束

| 欄位 | 類型 | 約束 |
|-------|------|------------|
| `spec.port` | 整數 | 1-65535 |
| `spec.advanced_options.connection_timeout` | 整數 | 0-1,800,000 毫秒 |
| `spec.advanced_options.http_idle_timeout` | 整數 | 0-600,000 毫秒 |
| `spec.advanced_options.panic_threshold` | 整數 | 0-100（百分比） |
| `metadata.name` | 字串 | 1-63 字元，模式：`^[a-z0-9]([-a-z0-9]*[a-z0-9])?$` |

## Origin Server 類型

Origin Server 可使用不同的類型變體來指定。類型決定了需要哪些額外欄位。

### 已實作的類型

| 類型 | 描述 | 必填欄位 | 狀態 |
|------|-------------|-----------------|--------|
| `public_name` | 具有公開 DNS 名稱的 Origin Server | `dns_name` | ✅ 完成 |
| `public_ip` | 具有公開 IP 位址的 Origin Server | `ip`（IPv4） | ✅ 完成 |

### 範例：public_name

```json
{
  "origin_servers": [
    {
      "public_name": {
        "dns_name": "backend.example.com"
      }
    }
  ]
}
```

### 範例：public_ip

```json
{
  "origin_servers": [
    {
      "public_ip": {
        "ip": "8.8.8.8"
      }
    }
  ]
}
```

## 待辦：依賴站點的 Origin Server 類型

以下 Origin Server 類型需要在實作前先開發「站點」資源邏輯：

| 類型 | 描述 | 必填欄位 | 狀態 |
|------|-------------|-----------------|--------|
| `private_ip` | 具有私有/公開 IP 和站點資訊的 Origin Server | `ip`、`site_locator`、`network_choice`、`snat_pool` | 🔲 等待站點邏輯 |
| `private_name` | 具有 DNS 名稱和站點資訊的 Origin Server | `dns_name`、`site_locator`、`network_choice`、`snat_pool` | 🔲 等待站點邏輯 |
| `k8s_service` | 具有站點資訊的 Kubernetes 服務 | `service_name`、`site_locator`、`network_choice` | 🔲 等待站點邏輯 |
| `consul_service` | 具有站點資訊的 HashiCorp Consul 服務 | `service_name`、`site_locator`、`network_choice` | 🔲 等待站點邏輯 |

### 實作需求

在完整記錄這些類型之前：

1. **站點資源探索** - 記錄 `site` 資源結構描述和 site_locator 模式
2. **Network Choice 列舉** - 記錄 `network_choice` 受約束的值（site_local_inside、site_local_outside 等）
3. **SNAT Pool 選項** - 記錄 SNAT Pool 組態模式
4. **跨資源參考** - 定義 origin_pool 如何參考 site 資源

### 追蹤

- 組態預留位置：`config/discovered_defaults.yaml` → `origin_pool.origin_server_types.enums`
- 相關衝刺：站點資源結構描述富化
