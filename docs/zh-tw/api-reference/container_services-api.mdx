無需完整叢集複雜性的 Pod 編排。邊緣站點執行、配額管控，以及為分散式應用程式提供標準化的運算設定檔。

- **類別**：基礎架構
- **複雜度**：中等
- **路徑數**：13 | **結構描述數**：159
- **層級**：進階
- **相關領域**：`managed_kubernetes`、`sites`、`service_mesh`

## 使用案例

- 部署 XCCS（容器服務）命名空間以支援多租戶工作負載
- 透過簡化的編排管理容器工作負載
- 配置分散式邊緣容器部署
- 在無需完整 K8s 複雜性的情況下執行容器化應用程式

## 主要資源

- **virtual_k8s**：用於容器工作負載的虛擬 Kubernetes 命名空間
- **workload**：容器工作負載部署配置
- **pod_security_policy**：用於容器執行時期約束的 Pod 安全性原則
