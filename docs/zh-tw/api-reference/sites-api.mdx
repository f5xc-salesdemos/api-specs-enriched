AWS、Azure、GCP VPC 與 Transit Gateway 的整合。基於標籤的選擇機制，用於跨區域的策略應用。

- **類別**：基礎架構
- **複雜度**：進階
- **路徑**：133 | **結構描述**：1015
- **層級**：標準
- **相關領域**：`cloud_infrastructure`、`customer_edge`、`managed_kubernetes`

## 使用案例

- 跨雲端供應商（AWS、Azure、GCP）部署 F5 XC
- 管理 XCKS（受管 Kubernetes）站點部署（前身為 AppStack）
- 部署以網路為重點的 Secure Mesh 邊緣站點
- 將外部 Kubernetes 叢集整合為 Customer Edge
- 設定 AWS VPC、Azure VNet 和 GCP VPC 站點
- 管理虛擬站點與站點策略

## 主要資源

- **site**：用於邊緣服務的實體或雲端部署位置
- **virtual_site**：使用標籤選擇器進行站點的邏輯分組
- **site_mesh_group**：多個站點之間的 Mesh 連線設定
