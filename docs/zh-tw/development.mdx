本指南描述 F5 XC API 擴充管線的開發工作流程。

## 快速開始

### 首次設定

```bash
# Clone the repository
git clone https://github.com/f5xc-salesdemos/api-specs-enriched.git
cd api-specs-enriched

# Create virtual environment and install dependencies
make install

# Install pre-commit hooks
make pre-commit-install

# Download specs and run pipeline
make build

# Preview documentation locally
make serve
```

### 日常開發

```bash
# Quick rebuild (uses cached specs)
make rebuild

# Run all quality checks
make pre-commit-run

# Preview documentation
make serve
```

## 工作流程模式

### 1. 探索工作流程

**前置條件**：
- 連線至 F5 XC 環境的 VPN
- 有效的 API 憑證

```bash
# Set credentials
export F5XC_API_URL="https://your-tenant.console.ves.volterra.io/api"
export F5XC_API_TOKEN="your-api-token"

# Run discovery
make discover

# View results
jq '.statistics' specs/discovered/session.json

# Commit for CI/CD
make push-discovery
```

**探索會擷取**：
- 實際的必填/選填欄位
- 來自即時回應的列舉值
- 預設值
- 模式驗證
- 回應範例

### 2. 發佈工作流程

發佈是自動化的：

1. 每日排程（UTC 6 AM）或推送至 main 會觸發工作流程
2. ETag 檢查判斷 F5 規格是否有變更
3. 管線處理並擴充規格
4. 版本從 git 標籤計算：`git describe --tags --abbrev=0`
5. 直接提交 + 建立標籤（無版本升級 PR）
6. 建立 GitHub Release 與變更日誌
7. 文件部署至 GitHub Pages

**版本升級規則**：

| 條件 | 升級類型 | 範例 |
|-----------|-----------|---------|
| 提交訊息中包含 `[major]` | 主要版本 | 1.0.0 → 2.0.0 |
| 提交訊息中包含 `BREAKING CHANGE` | 主要版本 | 1.0.0 → 2.0.0 |
| 新增領域規格 | 次要版本 | 1.0.0 → 1.1.0 |
| 其他任何變更 | 修補版本 | 1.0.0 → 1.0.1 |

### 3. 開發工作流程

```bash
# Create feature branch
git checkout -b feature/my-change

# Make changes to config or scripts

# Test locally
make pipeline
make lint

# Commit (pre-commit hooks run automatically)
git add .
git commit -m "feat: add new enrichment rule"

# Push and create PR
git push -u origin feature/my-change
gh pr create
```

## 探索深入解析

### 什麼是探索？

探索會檢視即時的 F5 XC API 以尋找：

- **未記錄的約束**：OpenAPI 中未標記的必填欄位
- **實際列舉值**：在生產環境中實際出現的值
- **預設行為**：欄位省略時伺服器套用的值
- **回應模式**：實際的資料形狀

### 探索設定

```yaml
# config/discovery.yaml
discovery:
  exploration:
    namespaces:
      - "system"
      - "shared"
    methods:
      - "GET"
      - "OPTIONS"
    max_endpoints_per_run: 500

  schema_inference:
    sample_size: 3
    detect_patterns: true
    detect_constraints: true
```

### 探索擴充

探索會將 `x-discovered-*` 擴充加入規格中：

```json
{
  "properties": {
    "name": {
      "type": "string",
      "x-discovered-required": true,
      "x-discovered-pattern": "^[a-z][a-z0-9-]*$",
      "x-discovered-examples": ["my-app", "prod-lb"]
    }
  }
}
```

### 約束分析報告

```bash
make constraint-report
# Output: reports/constraint-analysis.md
```

## OpenAPI 擴充

擴充管線使用供應商擴充來嵌入驗證和預設值中繼資料。

### 擴充類別

| 擴充 | 含義 | 影響 |
|-----------|---------|-------------|
| `x-f5xc-server-default` | F5 XC API 伺服器在欄位省略時套用此值 | 欄位為選填；省略時會產生文件記載的預設行為 |
| `x-f5xc-recommended-value` | F5 XC 網頁主控台為新資源預先填入此值 | 欄位沒有伺服器預設值，但此值代表典型設定 |
| `x-f5xc-recommended-oneof-variant` | F5 XC 主控台預先選取此 OneOf 變體 | 當存在多個互斥選項時，識別典型的選擇 |
| `x-f5xc-conflicts-with` | 列出不能與此欄位同時使用的其他屬性 | 屬性是 OneOf 群組的一部分；只能指定衝突屬性中的一個 |

### 必填欄位擴充

| 擴充 | 來源 | 用途 |
|-----------|--------|---------|
| `x-ves-required: "true"` | F5 XC 原始規格 | 欄位需要非零/非空值 |
| `x-f5xc-required-for` | 擴充管線 | 特定情境的必填狀態 |

**x-f5xc-required-for 情境**：
- `create`：建立資源時為必填
- `update`：更新資源時為必填
- `minimum_config`：最小可行設定時為必填

### 預設值與 OneOf 擴充

| 擴充 | 用途 |
|-----------|---------|
| `x-f5xc-server-default: true` | 標記伺服器套用的預設值 |
| `x-f5xc-recommended-value` | F5 XC 主控台預先填入的值 |
| `x-f5xc-recommended-oneof-variant` | 建議的 OneOf 變體 |
| `x-f5xc-conflicts-with` | 與其他 OneOf 屬性的互斥關係 |

#### x-f5xc-server-default

**類型**：`boolean`

當為 `true` 時，表示附帶的 `default` 值由 F5 XC API 伺服器強制執行。具有此擴充的欄位可以安全地從 API 請求中省略 — 伺服器會自動套用預設值。

```yaml
use_http2:
  type: boolean
  default: false
  x-f5xc-server-default: true
```

#### x-f5xc-recommended-value

**類型**：`any`（與欄位類型匹配）

指定 F5 XC 網頁主控台用作預先填入預設值的值。此值不由伺服器強制執行，但代表透過主控台建立新資源時的典型起始設定。

```yaml
timeout:
  type: integer
  x-f5xc-recommended-value: 3
```

#### x-f5xc-recommended-oneof-variant

**類型**：`object`（群組名稱到變體名稱的映射）

對於具有互斥欄位群組的結構描述，識別哪個變體是預設或最常見的選擇。鍵是 OneOf 群組名稱，值是建議的變體欄位名稱。

```yaml
healthcheckCreateSpecType:
  type: object
  x-f5xc-recommended-oneof-variant:
    health_check: "http_health_check"
```

#### x-f5xc-conflicts-with

**類型**：字串 `array`

**新增於**：v3.2.0 (Issue #494)

宣告 OneOf 群組成員之間的互斥關係。自動從 `x-ves-oneof-field-*` 擴充衍生而來。這使下游工具（Terraform、CLI、MCP）能夠在結構描述層級而非執行階段驗證衝突。

```yaml
host_header:
  type: string
  x-f5xc-conflicts-with: ["use_origin_server_name"]

use_origin_server_name:
  type: object
  x-f5xc-conflicts-with: ["host_header"]
```

**使用案例**：
- Terraform 提供者可以產生驗證規則
- CLI 工具可以警告衝突的欄位組合
- AI 助理可以產生正確的 OneOf 設定
- IDE 擴充套件可以提供衝突感知的自動完成

**來源**：在管線擴充期間自動從 F5 原生 `x-ves-oneof-field-*` 擴充衍生。

### 驗證規則影響

擴充器會檢查 `x-ves-validation-rules` 以推斷必填狀態：

| 規則 | 影響 |
|------|-------------|
| `ves.io.schema.rules.message.required: "true"` | 欄位為必填 |
| `ves.io.schema.rules.uint32.gte: N` | 若 N >= 1 且無伺服器預設值，欄位為必填 |
| `ves.io.schema.rules.repeated.min_items: N` | 若 N >= 1，陣列至少需要 N 個項目 |
| `ves.io.schema.rules.string.min_bytes: N` | 若 N >= 1，字串至少需要 N 個位元組 |

### 具有伺服器套用預設值的資源

部分資源接受空的規格，因為伺服器會套用預設值：

| 資源 | 伺服器套用的預設值 |
|----------|------------------------|
| `app_firewall` | `monitoring: \{\}`、`default_detection_settings: \{\}` |
| `rate_limiter` | `limits: []`、`user_identification: []` |
| `api_definition` | `swagger_specs: []`、預設 `api_groups` |
| `healthcheck` | `jitter: 0`、`jitter_percent: 0`、巢狀 http_health_check 預設值 |

對於這些資源，即使 `x-ves-required` 為 `true`，`x-f5xc-required-for.create` 也可能為 `false`。

### 預設值模式

| 模式 | 類型 | 範例 |
|---------|------|---------|
| `\{\}` | 空物件（選擇選取） | `monitoring: \{\}` |
| `[]` | 空陣列 | `expected_status_codes: []` |
| `0` | 數值 | `jitter: 0` |
| `""` | 字串 | `expected_response: ""` |
| `false` | 布林值 | `use_http2: false` |

### 新增探索到的預設值

1. 透過 API 以最小規格在 F5 XC 中建立資源
2. 讀回資源以查看伺服器套用的值
3. 將預設值記錄在 `config/discovered_defaults.yaml` 中
4. 執行管線：`make pipeline`
5. 驗證：`jq '.components.schemas | to_entries[] | select(.key | contains("resource_name"))'`

## 疑難排解

### Pre-commit 耗時過長

管線在每次提交時執行以確保一致性。對於進行中的工作提交：

```bash
git commit --no-verify -m "WIP: work in progress"
# Run before final commit: make pre-commit-run
```

### 探索失敗

```bash
# Check VPN
ping your-tenant.console.ves.volterra.io

# Check credentials
echo $F5XC_API_TOKEN | head -c 10

# Check API URL format
echo $F5XC_API_URL
# Format: https://tenant.console.ves.volterra.io/api
```

### 產生的規格出現檢查錯誤

```bash
# Check lint report
cat reports/lint-report.json | jq '.errors'
```

在擴充/正規化設定中修正問題，而非修改輸出檔案。

### 版本系統

版本從 git 標籤衍生（例如 `v2.0.38`），消除了因並行 PR 造成合併衝突的競態條件。

```python
# Version is calculated dynamically from git tags
from scripts.utils.version_calculator import get_version_from_tags
version = get_version_from_tags()  # Returns "2.0.38"
```

### Clone 後缺少規格

產生的規格已被 gitignore：

```bash
make build  # Downloads and generates everything
```

### 大檔案被阻擋

在 `.pre-commit-config.yaml` 中新增排除項：

```yaml
- id: check-added-large-files
  exclude: ^path/to/large/file\.json$
```
